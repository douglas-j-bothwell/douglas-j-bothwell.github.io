Database Performance Insight
This insight is useful when you want to
Identify time windows when database clients are experiencing high response times
Identify specific database servers whose performance is contributing to high response times.
When you identify a database server that you want to troubleshoot in detail, you can view details about that server in the Database Troubleshooting Insight.
Figure 2 Database Performance Insight
 
The following procedure describes the suggested workflow for this Insight.
Procedure 2 Database Performance: Suggested Workflow
1.
Locate and select a DB Client in the Top Servers connected to DBs table (top left).
This table lists all hosts that have sent SQL data requests during the current time window. A DB client might initiate data requests on its own, or forward requests from other hosts (for example, an application server forwarding requests from its clients).
A breakdown of traffic for the selected client appears in the Response Time Composition Chart (top right).
2.
In the Response Time Composition Chart (top right), look for periods of high response times where Server Response Time (red) is a significant factor.
3.
Select a server in the DB Servers table (bottom left).
This table lists all servers that have received SQL data requests from the selected DB client (top left).
4.
Examine the Database Performance time chart (bottom right). Do you see an increase in database response time that corresponds to the increase in Server Response Time (red area) in the top chart? If so, this indicates that you might want to troubleshoot the selected database server.
 
5.
To troubleshoot a database server, you can
• 
Select a time window in the Response Time Composition Chart (top right) by dragging horizontally. The Database Server table and Database Performance chart are updated with results for the selected time window.
• 
Right-click on the selected server and choose Troubleshoot Selected Databases. This opens the server in the Database Troubleshooting Insight.
End of Procedure 2
