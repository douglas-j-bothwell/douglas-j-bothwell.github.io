Collecting NetFlow Data
AppResponse appliances can collect NetFlow traffic data from routers and other NetFlow-enabled devices. When NetFlow collection is enabled, the appliance automatically performs the following:
Collects all NetFlow data that is sent to its management port
Configures its own internal settings in response to changes in external NetFlow data sources (requires SNMP polling)
Scales any sampled NetFlow data by its sampling rate (optional)
Merges and de-duplicates NetFlow and AppResponse data
NetFlow collection enables you to extend AppResponse visibility into all corners of an enterprise network. With NetFlow, AppResponse can account for traffic whose packets are not seen by the AppResponse appliance’s monitoring ports, such as Internet traffic from branch office locations and traffic between remote offices.
NetFlow collection also enables you to decompose the throughput on a remote office’s link or interface into its contributing applications, users, and protocols. You can therefore manage links more effectively throughout an enterprise network. This capability is especially useful when you need to analyze why a specific device interface is overloaded.
Note: To use this functionality, you must have a valid NetFlow Collection module license.
