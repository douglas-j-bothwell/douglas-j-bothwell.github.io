Supported VoIP Signaling Protocols
VoIP signaling protocols support the commands and codes to set up and tear down VoIP calls. In some respects, they can be thought of as comparable to how TCP sets up and controls a TCP session. However, unlike TCP sessions, VoIP signaling protocols might or might not be in-line with the RTP streams. In addition to basic call set-up and tear-down information, these families of protocols also contain information on the caller and the called party (phone numbers) and any associated error codes. These error codes can be useful in determining the actual devices involved in a failure.
It should be noted that placement of the AppResponse Appliance can dramatically limit which error codes, or even calling party information (i.e., trunking), are available.
The Module for VoIP Performance supports the following VoIP signaling protocols:
SIP
H.323 (and associated protocols)
SCCP (aka Skinny)
MGCP
Some VoIP equipment vendors have chosen to implement proprietary versions or use proprietary extensions of these standards that might not be compatible with signaling decode. Many of these VoIP equipment vendors also have configuration options that enable standards-compliant implementations to be used, enabling full protocol decode. Regardless of the protocol compatibility, the corresponding RTP flows with a setup call are unaffected and are reported appropriately within the system. For further information on protocol compatibility, contact Riverbed and your VoIP equipment vendor.
