Configuring NetFlow Collection on an AppResponse Appliance
The Administration > System web interface has a set of options for enabling/configuring NetFlow data collection.
To configure NetFlow collection on the appliance, go to the Administration > System > Advanced page.
To turn on data collection, select the “Collect NetFlow data (on/off)” checkbox. All options are listed in the following table.
 
Table 2
NetFlow Data Collection Options in Administration > System Web UI 
Option
Description
Collect NetFlow data
When this option is enabled, the AppResponse Appliance collects the NetFlow data that it receives on its Management Interface.
Collect Monitoring Interface(s) data
Turn this option off to collect only NetFlow data.
Collect data for Interfaces
When this option is enabled, the Appliance collects and groups NetFlow data by interface, including drill-downs to applications, IPs, and IP conversations.
Collect data for Destination Interface=Null (DstIf:Null)
When this option is enabled, the Appliance collects and groups traffic routed by devices to the Null destination interface (that is, dropped packets).
Collect and merge data with Monitoring Interface(s) groups/drilldowns
When this option is enabled, the Appliance merges NetFlow and AppResponse data and removes all duplicate traffic. (See Correcting Sampled NetFlow Data for details on using this option with NetFlow sampled data.)
Collect data for ToS
When this option is enabled, the Appliance collects and groups NetFlow data by ToS (DSCP values) and ToS Groups.
Correct for Sampled Data
When this option is enabled, the Appliance scales sampled NetFlow data by the sampling rate. See Correcting Sampled NetFlow Data for details.
NetFlow Port(s)
The ports on which the Appliance will be listening for incoming NetFlow packets.
Forwarding NetFlow Data from an Appliance
You can configure an appliance to forward NetFlow data to other network devices. This enables you to collect your NetFlow data using one appliance and then distribute that data as needed (for example, to devices running other NetFlow-related solutions).
To configure an AppResponse appliance to forward NetFlow data, log into the appliance using the CLI and enter the following command:
nfconfig FLOW_RECEIVER <ip1>:<port1>,<ip2>:<port2>,...<ipN>:<portN>
Note: Enabling this feature will reduce the maximum possible rate (flows per minute) at which the appliance can receive NetFlow packets, because the management port is used to both receive and forward packets. You should assume a reduction of ~50% in the normal rate. If an appliance is rated for a maximum of 1.8M flows per minute, for example, you should expect an effective maximum of ~900k flows per minute if you enable forwarding.
WARNING: If you have multiple AppResponse appliances, do not set up your appliances so that they forward NetFlow data to each other in a circular or round-trip configuration (for example, appliance1==>appliance2 and appliance2==>appliance1). This will cause the NetFlow data to be collected and re-collected in an “infinite loop.” This type of configuration can result in extremely high resource consumption on your appliances and might cause one or more appliances to shut down.
