VoIP Terms and Definitions
Use this glossary to assist you in determining meaning for key terms used in this manual.
agent
A program that performs some information gathering or processing task in the background. Typically, an agent is given a very small and well-defined task.
burst
Two or more packets losses occurring within a 16 packet sliding window.
cache
A special high-speed storage mechanism. It can be either a reserved section of main memory or an independent high-speed storage device. Two types of caching are commonly used in personal computers: memory caching and disk caching.
A memory cache, sometimes called a cache store or RAM cache, is a portion of memory made of high-speed static RAM (SRAM) instead of the slower and cheaper dynamic RAM (DRAM) used for main memory. Memory caching is effective because most programs access the same data or instructions over and over. By keeping as much of this information as possible in SRAM, the computer avoids accessing the slower DRAM.
call ID
A unique number assigned by AVMM to a call.
channel
Refers the RTP connection(s) used to carry the voice conversation(s).
codec
The codec that was used on an RTP channel.
codec loss concealment method
The channel codec packet loss concealment method.
data transfer rate
Memory space that is set aside specifically for either storing a packet that is awaiting transmission over a network or storing a packet that has been received over a network. The memory space is either located in the network interface card or in the computer that holds the card.
dest
The node that received RTP packets.
dest (call context)
The phone number as delivered by the signaling protocol that received the call. If the data in the signaling protocol is insufficient, this field is blank. If the signaling data can deliver additional information (such as caller id data), this field contains that data. This can also be referred to as “To”.
dest (channel context)
The IP address of the node that received the RTP packets.
flow
A flow is a statistical aggregation, analysis and alerting group. A flow is defined according to a set of rules such as source IP subnet, destination IP subnet, codec type, call duration, etc. The flow administration window will present a list of currently-defined flows and allow new flows to be defined or modify currently-defined flows.
heap
In programming, an area of memory reserved for data that is created at runtime -- that is, when the program actually executes. In contrast, the stack is an area of memory used for data whose size can be determined when the program is compiled.
key
A unique number assigned to a channel.
IP
Internet Protocol. IP specifies the format of packets, also called datagrams, and the addressing scheme. Most networks combine IP with a higher-level protocol called Transmission Control Protocol (TCP), which establishes a virtual connection between a destination and a source.
IP by itself is something like the postal system. It allows you to address a package and drop it in the system, but there's no direct link between you and the recipient. TCP/IP, on the other hand, establishes a connection between two hosts so that they can send messages back and forth for a period of time.
network
A group of two or more computer systems linked together.
MAPDV
Mean absolute packet delay variation.
MIB (Management Information Base)
A database of objects that can be monitored by a network management system. SNMP uses standardized MIB formats that allow any SNMP tool to monitor any device defined by a MIB.
MOS (mean opinion score)
A standard derived by having a panel of trained listeners listen to both impaired and non-impaired audio files and rating those on an absolute scale of 1 to 5 - the MOS is the mean of the reported scores.
MOS-CQ
Mean Opinion Score - Conversational Quality.
MOS-LQ
Mean Opinion Score - Listening Quality.
network packet loss rate (call context)
The call context maintains the minimum and maximum of the RTP Packet loss rate on all of the channels associated to this call.
octet
An octet is 8 bits. It is equivalent to a byte, as long as the byte is also 8 bits. Bytes range from 4 - 10 bits, but octets are always 8 bits.
OID
Object identifier. In SNMP, a value string that uniquely identifies a particular manageable object within an extensive binary tree of objects.
originator
The IP address of the component which initiated the signaling
packet buffer
Memory space that is set aside specifically for either storing a packet that is awaiting transmission over a network or storing a packet that has been received over a network. The memory space is either located in the network interface card or in the computer that holds the card.
PPDV
Packet-to-packet delay variation.
PDU
Protocol Data Units.
PSTN
Public Switched Telephone Network.
RTP (Real Time Transport Protocol)
Real-time transport protocol provides end-to-end network transport functions suitable for applications transmitting real-time data, such as audio, video or simulation data, over multicast or unicast network services.
R-factor
R-factor is a transmission quality estimation/rating model that is standardized according to ITU G.107; R-factors are inherently oriented around conversational quality because they take into account impairments that can occur during a call that impact conversational difficulty -e.g. double-talk situations due to large delays like with a satellite calls, echo effects, etc.
R-CQ
Conversational Quality R-factor that takes into account impairments that can occur during a call that impact conversational difficulty -e.g., double-talk situations due to large delays like with a satellite calls, echo effects, etc.
R-LQ
Similar to Conversational Quality R-factor, Listening Quality R-factor removes the impairments introduced by delay and echo, and only considers listening impairments not conversational.
ringing time
The duration in the signaling protocol from when the destination phone starts ringing until RTP packets begin to flow. If the signaling is insufficient to generate this number, the value appears “N/A”.
RTP packet payload type
The received RTP packet payload types.
sample rate, samples/sec
The sampling rate defined for this codec.
setup time
The time from the first observed signaling packet until the RTP packets begin flowing. If the signaling was insufficient to generate the number, the value appears as “N/A”.
signaling type
An acronym for the signaling protocol used to setup the RTP conversation. The following values are used:
–
SIP
–
MGCP-SSCP (which means that the MCGP protocol and SCCP were both used to setup the call)
–
SCCP
–
H323 (includes H245)
SNMP (Simple Network Management Protocol)
An internet protocol that assists in the flow of management information between network devices.
source (channel context).
The IP address of the node that sent the RTP packets
source (call context).
The phone number as delivered by the signaling protocol that initiated the call. If the data in the signaling protocol is insufficient, this field is blank. If the signaling data can deliver additional information (such as caller id data), this field contains that data. This may also be referred to as "From".
source RTP port
The port number on the source node used to send the RTP packets
start time
The time of the first observed signaling packet or (if no signaling is observed) the first observed RTP packet.
status
This field contains a character version of the completions status used by the signaling protocol. If the signaling protocol does not support completion status, this field is blank.
terminator
The IP address of the component which responded to the signaling
thread
A part of a program that can execute independently of other parts. Operating systems that support multithreading enable programmers to design programs whose threaded parts can execute concurrently.
VAD/Silence Suppression
A technique in which the voice encoder attempts to detect whether the user is speaking or not before encoding and transmitting packets. The goal of VAD/silence suppression is to minimize network bandwidth usage when the user is not talking and only transmit packets when the user is talking.
VoIP
Voice over Internet Protocol.
