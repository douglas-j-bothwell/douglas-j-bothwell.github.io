Using the Database Performance Module
This section contains procedures for these tasks:
Access the Management Console
Define Monitoring Rules
Access the Management Console
You perform all configuration via the Database Performance Module Management Console. Use the following procedure to access the Management Console.
Procedure 3 Accessing the Database Performance Module Management Console
1.
From the AppResponse Web Console, select Administration > System > Launch > Database Performance Module Configuration. Alternatively, open a Web browser and browse to https://<appresponse-xpert-appliance-ip>:2780.
2.
When prompted, enter the user name and password for your system, then click OK to log in.
Note—The default user name and password are both Admin (case-sensitive).
3.
(Recommended) If you have not already done so, change the default password:
3.1.
From the main screen, choose Manage this Console > Manage Users > Change.
3.2.
Enter a new password, then click Save.
End of Procedure 3
 
 
Define Monitoring Rules
The Database Performance Module works by collecting real-time data on SQL queries and database requests submitted by end users and applications. Use the following procedure to define monitoring rules that specify which database traffic you are interested in tracking. In most cases, you will need only a single auto-discovery rule that is configured to automatically find all supported databases.
Procedure 4 Adding New Monitoring Rules
1.
If necessary, access the Management Console (as described in Access the Management Console).
2.
Click the “Manage software instances” button.
The “Manage software instances” page opens.
Figure 1
Manage Software Instances Page
 
3.
On the “Manage software instances” page, click the Manage button in the Action column for the “asx” instance.
The “Manage instance” page opens.
4.
Click the “Manage monitoring rules” button.
The “Manage monitoring rules” page opens. From this page, you can create new rules and change or delete existing rules.
5.
Click the “New rule” button.
The “Enter values for the rule” page opens.
Figure 2
Enter Values for New Rule Page
 
6.
Enter a name for the rule and information defining the DBMS servers to be monitored.
• 
“Monitor via collector” should be selected.
• 
Most users should keep the default value of “Segment size”.
For auto-discovery, use the following settings:
• 
Auto Discovery: selected
• 
DB type: Any
• 
DB node: 0.0.0.0
• 
NetMask: 0.0.0.0
• 
First port: 1025
• 
Last Port: 65535
7.
After entering all values for the rule, click Save.
The “Manage monitoring rules” page opens again, showing the newly defined rule.
8.
If you need to define additional monitoring rules, repeat steps 5–7. You might need additional rules if
• 
you are not using auto-discovery to find the supported databases on the network
• 
the IP addresses of the databases to be monitored cannot be covered by one set of DB node and NetMask values.
9.
When you have defined all monitoring rules, click the “Save changes” button to save the configuration.
10.
When you have finished making changes to the monitoring rules, click the “Done” button.
11.
When prompted, click “Restart”.
End of Procedure 4
