Configuring NetFlow on Network Devices
When you select the interfaces to collect on and the traffic to collect, there are two objectives:
Collect relevant NetFlow data from all devices and interfaces of interest.
Avoid collecting data for the same traffic at multiple interfaces (for example, collecting both ingress and egress traffic at both ends of a link). Collecting redundant data from multiple interfaces will consume resources on the AppResponse Appliance unnecessarily.
For the appliance to process NetFlow data, you must configure your network devices as follows:
The devices should generate data at 1-minute intervals.
For inactive flows, specify timeouts of 15 seconds (or the shortest interval supported by the device)
For data templates, specify timeouts of 300 seconds.
If sampling is enabled:
–
Export sample data at 1-minute intervals or less
–
Specify sample option templates with timeouts of 300 seconds
The NetFlow data must include all of the following fields:
–
OUT_BYTES
–
OUT_PKTS
–
IN_BYTES
–
IN_PKTS
You can include only one pair of these fields if desired—that is, OUT_BYTES and OUT_PKTS only, or IN_BYTES or IN_BYTES only.
–
LAST_SWITCHED
–
FIRST_SWITCHED
–
SRC_TOS
–
DST_TOS
–
DIRECTION
–
FLOW_SAMPLER_ID
–
FLOW_SAMPLER_RANDOM_INTERVAL
–
LAST_SWITCHED
–
FIRST_SWITCHED
–
DIRECTION
Configure the device to send NetFlow data to the IP address/UDP port of the appliance.
By default, the appliance is configured to listen on port 9996. To specify other ports, change the NetFlow Port(s) preference (described in NetFlow Data Collection Options in Administration > System Web UI).
