Administration Tools
Administration Tools has the following categories:
Status (Status Page)
License (Accessing the Module for VoIP Performance Web UI)
Protocol Settings (Protocol Settings Page)
Backup (Backup Page)
Data Purge (Data Purge Page)
Diagnostics (Diagnostics Page)
Some of these categories are described in Configuring the Module for VoIP Performance. See the corresponding page number for information.
Status Page
The Status page verifies that all critical VoIP processes are operating normally. The process should not be stopped manually unless under the direction of Riverbed technical support. To go to the Status page, choose Administration Tools > Status from the Module for VoIP Performance menu.
Figure 11 Administration Tools, Status Page
 
Backup Page
The Backup page is used to back up and recover VoIP data. For complete information on how to configure backup servers, see the AppResponse Operations Guide.
Figure 12 Administration Tools, Backup Page
 
To back up data:
Procedure 5 Backing Up Data
1.
Choose Administration Tools > Backup.
2.
Select a backup server from the drop-down menu.
3.
If you want to compress the data, select the “compressed” checkbox.
4.
Click Backup.
End of Procedure 5
 
 
To find backup data for recovery:
Procedure 6 Finding Backup Data for Recovery
1.
Choose Administration Tools > Backup.
2.
Select a backup server from the drop-down menu.
3.
Click Find.
End of Procedure 6
 
 
Diagnostics Page
The Diagnostics page is for advanced system troubleshooting. Riverbed technical support might request that you generate a raw database dump or log bundle and send it in for further analysis. To go to the Diagnostics page, choose Administration Tools > Diagnostics from the Module for VoIP Performance menu.
Figure 13 Diagnostics Page
 
Contact Riverbed technical support for instructions on uploading the diagnostic information for further analysis.
Note: In existing systems, you should gather and process a log bundle prior to installation of the Module for VoIP Performance. The existing load and packet discard rates need to be reviewed.
