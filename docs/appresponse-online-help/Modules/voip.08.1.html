Configuring and Analyzing VoIP
The AppResponse Module for VoIP Performance adds Voice over IP support to an existing AppResponse platform. This support includes
Deep packet analysis and management information for common VoIP signaling (SIP, H.323, and SCCP) and Real Time Protocol (RTP) protocols
Automated alerts when VoIP call quality falls outside specified limits
Monitoring is accomplished from an “asymmetrical deployment” (similar to the existing AppResponse appliance) methodology, without the need for additional software, agents, additional hardware, SNMP polling, changes in the existing network configuration, or the presence of VoIP management protocols (RTCP or RTCP-XR).
Note: This manual assumes you have basic working knowledge of VoIP (including the vernacular used with it) and the AppResponse Appliance. Selected terms are defined in Chapter 1 VoIP Terms and Definitions.
The Module for VoIP Performance provides industry-standard voice quality metrics and creates an automatic view into underlying conditions responsible for call degradation. It allows for measurement of the effect of other applications on VoIP and, conversely, of how other applications are affected by VoIP.
Module for VoIP Performance Overview
The Module for VoIP Performance is an advanced software solution that passively monitors packet streams via the AppResponse Appliance monitoring ports. It automatically detects visible VoIP signaling and calls to produce call signaling and call quality metrics. When measuring a VoIP call, there are four basic categories studied:
Signaling Information – Refers to the setup, potential error, and tear-down information used to place a VoIP call.
Listening Quality—Refers to how users rate what they “hear” during a call.
Conversational Quality—Refers to how users rate the overall quality of a call, based on listening quality and their ability to converse during a call. This includes any echo- or delay-related difficulties that might affect the conversation.
Transmission Quality—Refers to the quality of the network connection used to carry the voice signal. This is a measure of network service quality as opposed to the specific call quality.
The Module for VoIP Performance displays audio channels, typically used by conference equipment for video playback, the same way it displays the voice channels for a VoIP call. There is, however, a difference in which metrics are computed for audio channels vs voice channels. This difference is mainly because the Audio codecs, like MP4, do not have standard Nominal R-Factors or Nominal MOS scores, nor do they typically have the same characteristics for end system measurements for echo, signal and delay.
This difference manifests itself in some measurements for an audio channel being reported as N/A when examined in detail. These metrics are:
Conversational R-factor
Listening R-factor
Nominal R-factor
Degradation Due to Excessive Delay
Degradation Due to Low/High Input Analog Signal Level
Degradation Due to High Input Analog Noise Level
Degradation Due to Low Echo Cancellation
Average Jitter (Calculated MAPDV), ms
Originator Average End System Delay, ms
Terminator Average End System Delay, ms
Average Burst Duration, ms
Average Gap Duration, ms
Average R-Factor during Burst Periods
For more information on metrics, see Chapter 4 VoIP/Video Metric Definitions.
