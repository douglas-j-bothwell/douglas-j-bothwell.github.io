Creating Custom Insights: Introduction to Insight Development Kit
This section presents a quick introduction to Insight Development Kit (IDK) and its use. The topics covered are
How Insights Work—an overview of the building blocks that comprise an Insight
Overview of the Insight Development Kit UI—an introduction to the IDK window
Creating an Insight—a tutorial covering how to design and build an Insight that implements a workflow
How Insights Work
An Insight is a synergistic collection of tables and charts from the AppResponse user interface. These elements can be linked by various automatic analyses and supplemented with controls to provide interactivity. Once saved, an Insight can be opened as a distinct window within the AppResponse Desktop Console. Insights provide a way of customizing how data is presented and how you interact with it to get specific summary information or to solve specific problems.
Insights can serve a variety of purposes, including:
Information summaries—predefined displays of commonly viewed information. The displays are automatically populated and generally require limited or no user input. Insights of this type provide a convenient way to monitor performance or provide high-level views to management.
Workflows—interactive “templates” that allow custom views of information, for which users can select data sources as needed. Workflow Insights can capture best practices and simplify the process of solving complex tasks by guiding the user through analyses appropriate to a particular task.
Five Building Blocks
Insights are created using five building blocks collectively called Elements. These building blocks are
Visual Components
Analysis Components
Controls
Interactions
Layers
Visual Components and Analysis Components communicate via Interactions. Controls enable users to set parameters for Visual Objects. Layers, which can be thought of as different “pages”, are a way of specifying the inheritance of Components and Interactions between different parts of an Insight.
Visual Components
Visual Components are just what the name implies: the parts of an Insight that a user can see and interact with, such as charts, topology maps, selectors, tables, text labels, and toolbars. Almost every element of the AppResponse user interface is available as a Visual Component for an Insight.
Some Visual Components have toolbars with Controls that permit data entry or enable the component’s operation or appearance to be modified by the user. For instance, Figure 1 shows a Table Component with a variety of toolbar controls.
Figure 1 Table Component with Toolbar
 
There is also a Toolbar Component that adds similar Controls that affect the Insight as a whole. Figure 2 shows such a toolbar with seven Controls, one of which (a spacer between the Clear button and the Print button) is invisible.
Figure 2 Toolbar with Seven Controls
 
Analysis Components
Analysis Components are behind-the-scenes components that transform data in a variety of ways. For instance, the Drilldown Analysis Component takes a Business Group as an input and outputs the top members of a drilldown from that group.
Controls
Controls are the various elements of Visual Components (such as buttons, text fields, and combo boxes) that enable users to enter information and change settings. For instance, the “By” control on a Chart or Table lets the user specify the metric by which displayed groups are ranked.
Note: There are also controls within the IDK that you use to configure the appearance and behavior of the Insight Elements. When we discuss Controls (with an upper-case “C”), we mean the ones available to a user within an Insight.
Interactions
As noted above, Interactions enable Components to communicate among themselves. An Interaction automatically gets information of a specified type from one Component (its Producer) and transfers that information to another Component (its Consumer).
For instance, in Figure 1, selecting a new limit using the Top Control in the Toolbar causes a Value Interaction to get that limit and send it to the table (the Consumer of that Interaction). This allows the table to reflect the new limit in the number of items it displays.
Layers
Layers, which can be thought of as different “pages” in an Insight, offer a means of constructing multi-part Insights using a variety of inheritance methods that determine which elements of each Layer are available in other Layers. They are one of the primary means of preserving screen real estate (which is important because, for efficiency, all parts of an Insight should be visible without scrolling). Layers can be configured for sequential access or can be non-sequential.

Creating an Insight
This topic shows one possible approach to building an Insight. It describes all steps, from start to finish, for building a simple Insight.
The general workflow for creating an Insight is
State the high-level problem you want the Insight to solve.
–
Describe the problem in detail.
–
Determine what data is needed and how to get it.
Sketch what the Insight should look like
–
Visual Components
–
Analysis Components
–
Interactions
Build the Insight with Insight Development Kit.
Test the Insight.
The build and test stages are likely to be iterative, as you modify your Insight to provide exactly what you want to see.
State the Problem
The first step to building an Insight is to determine what the Insight is intended to do. For this example, let’s assume you want to develop an Insight that will provide a workflow to help answer these questions:
Which applications are using the most bandwidth on the local network?
Which business groups (BGs) are using these applications?
Who in each business group is using these applications?
To answer these questions, the Insight must perform the following tasks:
Identify the most-used applications over a certain time frame (by throughput)
For a given application, determine which BGs on the network are using that application (by throughput)
For a BG using a given application, determine which IPs in that BG are using the application (by throughput)
In addition, you want to display throughput data for the top applications in a time-series chart.
Lay Out the Insight
After defining the problem, it is time to determine which components are needed for the Insight and how they should be arranged (both visually and functionally).
Visual Components
To decide which Visual Components are needed, consider each of the tasks identified when stating the problem. For example, the first task was
Identify the most-used applications over a certain time frame (by throughput)
The Component used to display top items by some criterion is called a favorites table. You can use this Component to display the list of top applications required by the first task. Likewise, favorites tables are suitable for the two remaining tasks (displaying BGs and IPs) as well.
So far, you know that three favorites tables are needed. You also know that you want to display throughput for an application over some time frame, so a time-series chart is needed. IDK provides several of these; we’ll use a multi-group time series chart for this Insight. (A multi-metric time series chart would also work.)
Finally, to make the Insight easier to understand, you can add an Insight title (a banner) and labels to identify the chart and each table.
To plan the layout of the Insight, make a sketch showing all of the Visual Components you have identified and how they should be arranged. Figure 8 shows what this sketch might look like.
Note: Riverbed recommends putting the first selector (the list or table from which the user first selects something) in the upper left-hand corner. Typically, the workflow then proceeds top-to-bottom and left-to-right. This arrangement is used in Figure 8.
Figure 8 Insight Design - Visual Components
 
Analysis Components
Thinking about which Analysis Components to add requires that you also think about information flow, because you need to consider how the information moves from one Component to the next. Analysis Components transform data as it moves between Components.
In this Insight, there are four information flows to consider:
into favorites table 1
(to populate the table when the Insight is first opened)
from favorites table 1 to the time series chart
(to provide throughput data for display)
from favorites table 1 to favorites table 2
(to populate the BGs table when an application is selected)
from favorites table 2 to favorites table 3
(to populate the IPs table when a BG is selected)
These flows are shown by dashed lines in Figure 9.
Figure 9 Insight Design - Information Flow
 
Now that you have planned how the information will flow, you can add Analysis Components. In looking at the information flows in Figure 9, only three of them need Analysis Components:
Favorites table 1 needs an Analysis Component to provide it with data on top applications. This will be a Drilldown analysis that runs automatically when the Insight is opened.
To move information from favorites table 1 to 2, an Application Member BGs analysis will obtain the top business groups for the selected application.
To move information from favorites table 2 to 3, a Drilldown analysis will obtain the top IPs using the selected application in the business group.
The information going from favorites table 1 to the time-series chart does not need an Analysis Component because the table is simply passing information to the chart.
Figure 10 shows the Insight with Analysis Components added next to the information flows they will operate on. These Components are drawn with dashed boxes as a reminder that they are hidden. Notice that each Analysis Component will be selecting top items based on throughput.
Figure 10 Insight Design - Analyses
 
Interactions
Finally, tie everything together by adding Interactions to implement the information flow through the Insight. In Figure 11, each arrow represents a Group Interaction between components. Group Interactions allow a group of objects (such as applications or BGs) to be selected in one Component and passed to another Component.
Figure 11 Insight Design - Interactions
 
Implement the Insight
After identifying the necessary data, analyses, and displays, you can implement the design with Insight Development Kit. You’ll begin by creating an empty Insight with a banner.
Note: In most cases, there are several ways to create or edit a Component. This tutorial will use different methods at various times to introduce them to you. After you have seen the possibilities, you can use whatever methods you prefer.
Procedure 1 Creating the Insight
1.
Log into AppResponse using an account with Administrator privileges.
2.
From the AppResponse menu, choose Insights > Insight Development Kit…
The Insight Development Kit window opens.
3.
Review the best practices shown in the Insight Builder Tips window (if it appears), then click OK.
4.
From the IDK menu, choose View > Layout Grid and make sure that Workflow is selected.
Note—Unless specified otherwise, all further steps will use the IDK menu.
5.
Create a banner that identifies the Insight:
5.1.
Right-click in the workspace and choose Add New > Visual Component from the pop-up menu.
The Select Component window opens.
5.2.
Expand the Text node in the treeview and click on BannerBox.
The right-hand pane displays a description of the BannerBox Component.
5.3.
Click OK to place a BannerBox Component in the workspace.
6.
It’s a best practice to give components a meaningful name, so do that now:
6.1.
Double-click on the BannerBox1 node in the Navigation pane.
6.2.
Change the name to Banner:title.
Notice that we keep the Component type (“Banner”) as part of the name. This practice is especially useful for Analysis Components, which all use the same icon.
7.
Edit the content of the banner:
7.1.
Right-click on the Banner:title icon in the workspace and choose Edit Component from the pop-up menu.
The Edit the Component dialog box opens.
7.2.
Click Add Cell…
The Cell Editor dialog box opens. In this window, you can add text and an image to the banner.
7.3.
In the Text area, enter My First Insight in the Text: field.
7.4.
Optionally, set the color, font, and size. (To set the color, first unselect the Default checkbox.)
7.5.
When finished, click OK twice to close both dialog boxes.
8.
Save your work so far:
8.1.
Choose File > Save As…
The Save Insight File dialog box opens.
8.2.
Navigate to your Local Insights directory and save the file as <your_initials>_quickstart.xml.
Note—To identify or change the Local Insights directory, choose Tools > Preferences from the AppResponse menu, then click Insights.
Note—If a warning about rows or columns with no components appears, click OK.
End of Procedure 1
 
 
Next, add the three Favorites tables and the time-series chart, along with the associated labels.
Procedure 2 Adding the Table and Chart Components
1.
Drag three instances of a Favorites Table Component () from the Tables toolbar into the workspace. Placement is not critical, but it is convenient to follow the general layout shown in Figure 11.
2.
Configure the favorites tables:
2.1.
Right-click on one of the table icons, choose Rename Component, and change the name to Table:favorite:apps.
2.2.
In the same way, rename the other two tables to Table:favorite:BGs and Table:favorite:IPs.
Other than the names, the default settings will do what you need, so these components require no further configuration.
3.
Drag three instances of a Label Component () from the Text toolbar into the workspace and rename them as follows:
• 
Label:apps
• 
Label:BGs
• 
Label:IPs
4.
Configure the labels:
4.1.
Right-click on the Label:apps component and choose Edit Component. Set the first text field to Top Applications (by Throughput) and click OK.
4.2.
In the same way, set the label text for the other two labels:
• 
Set the Label:BGs text to Top BGs for Application (${group})
• 
Set the Label:IPs text to Top IPs for BG (${group})
${group} is a variable that will be replaced with the name of the application or BG being passed to the table each time the table is updated.
5.
Now add the time-series chart and its label:
5.1.
Drag one Multi-Group Time Series Chart Component () from the Chart toolbar and one Label Component into the workspace.
5.2.
Rename the Chart Component to Chart:multigrp:apps.
5.3.
Rename the Label Component to Label:apps:chart and set its text to Throughput of Selected Application.
6.
Click Save to save the Insight. Your Insight should now look similar to Figure 12 (icon positions might vary).
Figure 12
Insight with Visual Components Added
 
End of Procedure 2
 
 
With the Visual Components in place, it is time to define the Analysis Components that will process the data for each table.
Procedure 3 Adding the Analysis Components
1.
As shown by the dashed boxes in Figure 11, this Insight will need three Analysis Components. The first one is a drilldown that will automatically populate the Applications table when the Insight is opened. Add it as follows:
1.1.
Select Drilldown from the Analysis pull-down menu, then click Add Analysis.
The Set Analysis Parameters dialog box opens.
1.2.
Select the Run Analysis on Startup checkbox. This will allow the Drilldown Component to populate the Applications table automatically.
1.3.
Specify what information the drilldown is collecting. To do this, click on the Drilldown To pull-down menu, then click Browse Groups…
1.4.
You want the drilldown to gather top applications. In the Group/Drilldown Chooser, click on the Defined Apps node and select any application from the pull-down menu. (It doesn’t matter which application you select; this is just a placeholder.) Click OK twice.
The Drilldown Component appears in the workspace.
1.5.
Rename the Drilldown to Drilldown:apps.
2.
The next Analysis Component collects top business groups for an application:
2.1.
Select Application Member BGs from the Analysis pull-down menu, then click Add Analysis.
The Set Analysis Parameters dialog box opens.
2.2.
This analysis should run on whatever application is passed to it, so you don’t need to select one. Click OK to use the default settings.
The Analysis Component appears in the workspace.
2.3.
Rename the Analysis Component to Application Member BGs.
3.
The final Analysis Component will drill down to IPs for a business group:
3.1.
Select Drilldown from the Analysis pull-down menu, then click Add Analysis.
The Set Analysis Parameters dialog box opens.
3.2.
Specify what information this drilldown collects. To do this, click on the Drilldown To pull-down menu, then click Browse Groups…
The Group/Drilldown Chooser opens.
3.3.
This drilldown needs to gather IPs for a business group. To specify this, begin by clicking on the Business Group node and select any BG from the pull-down menu. (It doesn’t matter which BG you select; this is just a placeholder.)
3.4.
Next, expand the Business Group node to Applications > Defined Apps. Click on the Defined Apps node and select any application from the pull-down menu. (Again, it doesn’t matter which one you select.)
3.5.
Finally, expand the Defined Apps node to Member IPs > IP Address. Click on the IP Address node and enter any IP address (for example, 10.0.0.0) in the entry field.
The Group/Drilldown Chooser should appear as shown in Figure 13.
Figure 13
Drilldown for IP Analysis
 
3.6.
Click OK twice.
3.7.
Rename the drilldown to Drilldown:IPs.
4.
Click Save to save the Insight.
End of Procedure 3
 
 
Next, define the flow of data through the Insight by connecting the Visual and Analysis Components with Interactions.
Procedure 4 Connecting the Visual and Analysis Components with Interactions
1.
Add an interaction to load the applications table with data:
1.1.
Click Add Interaction.
1.2.
In the Component Interactions dialog box, use the pull-down menus to specify the interaction type, its producer (source component) and its consumer (destination component):
• 
Interaction: group
• 
Producer component: Drilldown:apps
• 
Consumer component -> Table:favorite:apps
1.3.
Click OK.
2.
The next interaction sends a selected application to the chart for display. You can use a different method to add this interaction:
2.1.
In the workspace toolbar, click on the arrow to enter Add Interaction Mode.
Figure 14
Add Interaction Mode
 
2.2.
From the Interaction pull-down menu, select group.
2.3.
Click and drag an Interaction arrow from the Table:favorite:apps icon to the Chart:multigrp:apps icon.
3.
Using the arrows in Figure 11 as a guide, add the remaining three interactions. Use whichever method you prefer.
4.
Save the Insight.
End of Procedure 4
 
 
Now that the Insight is defined logically, it’s time to specify its visual layout.
Procedure 5 Specifying the Layout
1.
In the workspace toolbar, click the Layout Grid icon ().
The Layout Components dialog box opens.
2.
In this dialog box, you’ll arrange the Visual Components by dragging them from the treeview to the grid. Follow the arrangement shown in Figure 15.
Figure 15
Visual Components Layout
 
2.1.
Drag Banner:title to the grid square in row 1, column 1 (1:1).
2.2.
The banner should span the entire width of the Insight. Click the Banner:title icon in the grid and drag the right side so that the blue box extends across columns 1 and 2.
2.3.
Drag Label:apps to grid square 2:1.
2.4.
Continue until all nine Visual Components are in the grid, then click OK.
3.
Save the Insight.

Now it is time to see what the Insight looks like, adjust the sizes of each component, and make sure it works as planned.
 
 
Procedure 6 Testing the Insight
1.
Click Test/Layout.
The Preview Insight Content window opens.
2.
To start, adjust the Insight and its components to convenient sizes. These sizes will be saved with the Insight and used each time the Insight is opened.
To resize a component, drag any of its sides to the desired location. When finished, click Apply. (Optionally, you can readjust the sizes later after there is data in each component.)
3.
Select an application in the Top Applications table.
The chart and Top BGs table populate with data for the selected application. (This might take a short time; check the progress bar in the lower right corner.)
4.
Select a business group in in the Top BGs table.
The Top IPs table populates with data for the selected BG.
5.
Optionally, resize the Insight to match the displayed data and click Apply.
6.
The Insight is now working, but there are three things you might want to change (as shown in Figure 16):
1 – In the Group column of the Top BGs table, each row shows the same name for the selected application. This is not very useful, but you can change it.
2 – The labels for the Top BGs and Top IPs tables don’t show the application and BG names as expected.
3 – Each table has a toolbar with two icons taking up space in the top right corner. You can remove these if you don’t want them.
Figure 16
Insight Test Layout
 
7.
You must fix these problems from the Workflow view. For now, click OK to close the Preview Insight Content window.
8.
Save the Insight.
End of Procedure 6
 
 
All that is left now is to fix the problems you saw in the test layout.
Procedure 7 Making Final Adjustments
1.
(Problem 1) By default, the Groups column of a Favorites table displays the last part of the path element. To see more useful information, you must change this to display the first part of the path:
1.1.
Right-click on the Table:favorite:BGs icon and choose Edit Component.
The Edit the Component dialog box opens.
1.2.
Click the Table Settings icon ().
The Table Settings dialog box opens.
1.3.
Click the Composition tab.
1.4.
In the Group Path Display Options area, select the Show First Path Element radio button.
1.5.
Click OK twice to close both dialog boxes.
2.
(Problem 2) You defined the labels for the Top BGs and Top IPs tables to include variable information (using the ${group} token), but it is not appearing. This is because the labels need an interaction to send them the group information to display. The needed information is produced by the Analysis Components that feed the tables.
To fix this, enter Add Interaction Mode and draw two group interactions:
• 
From the Application Member BGs icon to the Label:BGs icon
• 
From the Drilldown:IPs icon to the Label:IPs icon
3.
(Problem 3) To remove the table toolbars:
3.1.
Right-click on the Table:favorite:BGs icon and choose Edit Component.
The Edit the Component dialog box opens.
3.2.
Click the Table Settings icon ().
The Table Settings dialog box opens.
3.3.
Click the Toolbar tab.
3.4.
Unselect the Show Toolbar checkbox.
3.5.
Click OK twice to close both dialog boxes.
3.6.
Repeat steps 1–5 for each of the other two tables.
4.
Save the Insight.
5.
Click Test/Layout to confirm that everything works as desired.
6.
Save the Insight.
7.
Close Insight Development Kit.
End of Procedure 7
 
 
You now have a new Insight available to you. To see it in AppResponse, choose the <your_initials>_quickstart Insight from the Insights > Local Insights menu. You can share your Insight with others by uploading it to the Update Center.
