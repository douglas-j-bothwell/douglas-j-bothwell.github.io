Advanced Data Analysis
This tutorial illustrates some advanced data analysis techniques such as drilling down on granular data, using custom tables and charts, and visualizing network devices and traffic flows.
Note: All AppResponse data is based on live network traffic. The data that appears in your Desktop Console will be different from the data shown in these Tutorials. The screen shots and other traffic-specific results shown in this tutorial are intended as examples only.
Prerequisites
This tutorial assumes that you have completed the following tutorials:
AppResponse Tutorial: Getting Started
AppResponse Tutorial: Trending, Alerts, Reporting, and SLAs
This tutorial assumes that you, or someone in your organization, has already done the following:
1)
Racked and powered up the AppResponse appliance
2)
Set up the appliance using the Command Line Interface and the Web User Interface
3)
Created a user account with administrative privileges on the appliance (you will need these privileges to perform some steps)
4)
Verified that the appliance is receiving traffic:
–
(AppResponse appliance) The span ports are configured and are feeding traffic into at least one of the monitoring interfaces
–
(AppResponse Rover) The Rover can see traffic on the virtual interfaces
5)
Defined a set of Business Groups that reflect the areas of interest in your network (as described in the first AppResponse “Getting Started” tutorial)
Favorites Tables
AppResponse enables you to generate many different tables such as Business Group Tables, Application Tables, and Total Traffic Tables. One of the most useful types of table is the Favorites Table.
Favorites Tables are useful when you want to take a subset of specific data and examine it in more detail. You can fully customize the data in a Favorites Table, which makes these tables useful for manipulating and analyzing any combination of metrics. You can use a Favorites Table to
Drill down into metrics for a specific object such as a site, user/host, or IP conversation
Select specific metrics that you want to look at
Correlate and analyze data by sorting on different metrics in the table
In this section, you will create a new Favorites Table and use it to analyze the Business Groups and applications that have the longest response times in your network.
Perform the following steps:
1)
Log in to the AppResponse appliance and open a Desktop Console if you have not already done so (as described in Procedure 1-1 Logging in to the AppResponse Appliance” in the “AppResponse: Getting Started” tutorial).
Creating a New Favorites Table
2)
Click the “Favorites Table” button in the toolbar at the top of the Desktop Console window (this is the button that shows a table with a star in front).
Figure 1 “Favorites Table” Toolbar Button
 
–> A new, empty Favorites Table appears.
Figure 2
New Favorites Table
 
Note—You can also create a new Favorites Table directly from an Insight. Select a Business Group, IP Address, or Conversation; then right-click and choose Send To > Favorites Table. This creates a new table with data for the selected group, address, or conversation.
In the following steps, you will identify the Business Groups with the longest Server Response Times and then add these groups to the Favorites Table.
Adding Business Groups to the Favorites Table
3)
Open the Slow Server Insight: choose Insights > Troubleshoot > Slow Servers.
You will use this Insight to identify data of interest and then export this data to a Favorites Table.
4)
In the Slow Servers Insight, identify the first Business Group listed in the Top Business Groups table (top left).
This is the group whose hosts have the highest overall Server Response Time for the project time.
5)
Add this Business Group to the Favorites Table:
a)
Click the Add Group/Drilldown button n the top-right corner of the Favorites Table.
 
b)
In the Group/Drilldown Chooser, select the group in the Business Group pull-down menu. Then click OK.
 
–> This group is included in the Favorites Table.
 
Note—You can also add an object (such as a Business Group or Application) by dragging it from a table in another window into the Favorites Table.
–> The Favorites Table now includes the top Business Group in the Slow Servers Insight. Because we are interested in analyzing application response times, we will now configure the Favorites Table to highlight relevant metrics.
Configuring the Favorites Table
6)
Click the Edit Columns button in the top-right corner of the Favorites Table window.
 
7)
In the Edit Columns dialog box, set the options so that only the following metrics are selected, and that all other metrics are not selected:
–
Informational > Group
–
Informational > Information
–
Application Performance > Server Response Time (Clients) (msec)
–
Application Performance > Server Response Time (Servers) (msec)
–
Application Performance > User Response Time (Clients) (msec)
–
Application Performance > User Response Time (Servers) (msec)
–
Utilization > Throughput (Inbound and Outbound) (kbits/sec)
8)
Click OK to close the Edit Columns dialog box.
–> The Favorites Table now shows only these metrics.
In some cases, you might want to arrange the columns in a way that makes logical sense to you in the context of the data that you want to analyze. In the following sections, we are going to focus on the Server Response Time and User Response Time metrics. To make the table easier to work with, we will now move two less-relevant columns “out of the way.”
9)
Rearrange the columns by performing the following steps:
a)
In the Favorites Table, click in the header of the Throughput (Inbound and Outbound) column and drag it to the right edge of the table.
b)
Click the header of the Information column and drag it to the right edge of the table.
–> Now, all the response-time metrics appear immediately to the right of the Group column (far left).
 
10)
Sort the Applications for this Business Group according to each of the response-time metrics, as outlined in the following steps:
a)
Using the Expand (+) buttons in the Favorites Table, drill down to and expand the Applications list for the group that you added.
 
b)
Sort the Applications list according to each response-time metric: click on the bullet or arrow in the top-right corner of the header in each column. This sorts the data according to the metric in that column.
 
In this section, you learned how to customize a Favorites table to show network objects and data metrics of interest. Although we added Business Groups to this specific table, remember that you can create Favorites Tables for other object types such as Applications, IP Addresses, IP Conversations, and Interfaces.

Customized Charts
Although Insights provide many different charts for many purposes, you can create custom charts based on any set of data. One of the best uses for custom charts is to display specific data of interest from a Favorites Table. Together, Favorites Tables and custom charts offer complete flexibility in displaying data in AppResponse.
You can use custom charts to
Create ad-hoc, targeted views of specific data
Provide a view or report that is not readily available or accessible from the default Insights
Analyze and correlate any combination of AppResponse metrics (over sixty metrics are available)
Now that we have some data in the Favorites Table, we can visually represent that data in a number of ways using different charts.
Multi-Group Time Series Chart
This chart is useful for tracking one metric over time for multiple objects. In the following steps, you will create a chart of top applications according to the User Response Time (Servers) metric.
1)
Sort the Favorites Table according to the User Response Time (Servers) metric, so that the applications with the highest response time appear first.
2)
In the User Response Time (Servers) column, use Ctrl-click to select the top five applications.
Note—The initial metric used in the chart is determined by the metric column in which you select the objects. If you select the objects in a column that has no corresponding metric—in the Information or Group column, for example—the default metric is Throughput (Inbound and Outbound).
3)
Click the Multi-Group Time Series Chart button in the Desktop Console toolbar.
Figure 3 “Multi-Group Time Series Chart” Toolbar Button
 
–> A chart appears for the selected objects and the metric of interest.
Figure 4
Multi-Group Time Series Chart (EXAMPLE)
 
4)
Close (or minimize) the chart window.
Multi-Metric Time Series Chart
This chart enables you to view multiple metrics for one and only one object. This chart type is useful when you want to compare or correlate several metrics for one object.
In the following steps, you will create a chart that shows the User Response Time (Clients) and User Response Time (Servers) metrics for one application.
1)
In the Favorites Table, select an application by clicking in the User Response Time (Servers) column.
NOTE—In some cases, the AppResponse appliance might not have sufficient data to calculate a specific response-time metric for a specific group. When this happens, the table entry will be blank. For the purposes of this section, select an application that also shows a value in the User Response Time (Clients) column.
2)
Click the Multi-Metric Time Series Chart button in the Desktop Console toolbar.
Figure 5 “Multi-Metric Time Series Chart” Toolbar Button
 
–> A Multi-Metric Time Series Chart appears with the selected object and metric.
Figure 6
Multi-Metric Time Series Chart with One Metric (EXAMPLE)
 
3)
Click in the Chart Metrics field (in the top-left corner of the chart window).
Figure 7
“Metrics...” Pull-Down menu
 
4)
In the Metric Selection window, set the options so that only the following metrics are selected:
–
Application Performance > User Response Time (Clients)
–
Application Performance > User Response Time (Servers)
(This metric should already be selected.)
5)
Click OK in the Metric Selection dialog box.
–> The Multi-Metric Time Series chart now shows both User Response Time metrics for the selected application.
Figure 8
Multi-Metric Time Series Chart with Two Metrics (EXAMPLE)
 
6)
Close (or minimize) the chart window.
Pie Charts
A Pie Chart shows the relative frequency or distribution of one metric value as it applies to a set of related objects (such as Business Groups, Applications, or IP Conversations).
In the following steps, you will create a pie chart of the five Connected Groups that have the highest bidirectional throughput.
1)
In the Favorites Table, collapse the Applications list if it is still expanded.
2)
Click in the heading of the Throughput (Inbound and Outbound) column and drag the heading leftward so that it is immediately to the right of the Group column.
 
3)
Drill down to and expand the Connected Groups list.
4)
Verify that the table is sorted according to the Throughput (Inbound and Outbound) metric, from highest throughput to lowest.
5)
Select the top five Connected Groups by Ctrl-clicking in the Throughput (Inbound and Outbound) column.
6)
Click the Pie Chart button in the Desktop Console toolbar.
Figure 9 “Pie Chart” Toolbar Button
 
–> A Pie Chart appears for the selected groups and Throughput metric.
Figure 10
Pie Chart (EXAMPLE)
 
7)
Close (or minimize) the chart window.
Response Time Composition Charts
Although the Response Time Composition Chart is available in several Insights, you might find it useful to create a highly targeted, granular RTCC for a specific element in a Favorites Table—for example, a single client-server conversation for a single application. You will create this type of chart in the following steps.
1)
In the Favorites Table, collapse the Connected Groups list.
2)
Expand the Applications list.
3)
Expand the first Application list under the group.
–> The table expands to show drilldowns for the IP Conversations, Connected IPs, and Member IPs for the application (as shown in the following figure).
Figure 11
Expanded List for Individual Application (EXAMPLE)
 
4)
Expand the IP Conversations sub-list.
5)
Sort the table by the User Response Time (Servers) metric in descending order.
6)
Select the first conversation in the [business_group_name] > Applications > [application_name] > IP Conversations list.
NOTE—The Response Time Composition Chart uses a fixed set of metrics; therefore, the column in which you select an object has no effect on the RTCC that gets generated.
7)
Click the Response Time Composition Chart button in the Desktop Console toolbar.
Figure 12 “Response Time Composition Chart” Toolbar Button
 
–> A Response Time Composition Chart appears for the application between the hosts specified in the IP Conversation.
Figure 13
Response Time Composition Chart (EXAMPLE)
 
8)
Close (or minimize) the chart window.
9)
Close (or minimize) the Favorites Table window.

Topology Views
One of the challenges of network management is making sense of the vast quantities of data carried over the network. AppResponse collects many detailed metrics for each group. Group-to-group data transfers can sometimes be difficult to understand through tables and charts only. Topology views enable you to
View graphical representations of the relationships between network elements—Business Groups, IP Addresses, or ASs—and important metrics for each element
Visualize group-to-group relationships
Clearly understand how different network elements interact
Business Group Topology View
This is the most commonly used topology view. This view enables you to see which Business Groups are exchanging—and are not exchanging—traffic. In the following steps, you will use this view to explore the relationship of different Business Groups in your network.
1)
Click the BG Topology button in the Desktop Console toolbar. (You can also choose the View > Topology > BG Topology menu item.)
Figure 14
“BG Topology” Toolbar Button
 
–> A new Business Group Topology window appears. Note that the topology pane (right) is initially empty.
Figure 15
“Business Group Topology” Window (Initial View)
 
2)
In the treeview (left pane), expand the Business Group Containers or the All Business Groups subtree so that the list of containers/groups is visible.
3)
Click on a Business Group of interest to select it.
4)
Click on the selected Business Group again and drag it into the topology (right pane).
–> The topology pane shows the Business Group you included and all BGs that exchange traffic with that group.
Figure 16
Business Group Topology, One Group Added (EXAMPLE)
 
Note the following:
–
Each BG-BG pair that exchanges traffic is connected by an arrow.
–
Each arrow has a numerical label that shows the value for the metric and units specified in the pull-down menus above the pane. The default settings are “Throughput (Inbound and Outbound)” and “kbits/sec”.
–
The thickness of each arrow represents the relative value of the metric (in this case, bidirectional throughput).
5)
Select a different metric in the pull-down menu above the topology pane. Note that the arrow thicknesses and labels are updated to reflect the selected metric.
6)
Drag another BG or BG Container from the treeview into the topology pane. Note how the topology is updated to show both BGs and all connected BGs.
Figure 17
Business Group Topology, Two Groups Added (EXAMPLE)
 
7)
Select a different layout in the Layout menu (top-right corner) and see how you can view your BG-to-BG traffic patterns using different layouts.
8)
Close (or minimize) the Business Group Topology window.
IP Topology View
You can use the IP topology view to
Examine layer-3 connectivity to a specific IP address, based on data collected through traceroute
Manually initiate and visualize traceroutes
Examine historic automated traceroutes that are stored in the appliance database

1)
Click the IP Topology button in the Desktop Console toolbar. (You can also choose the View > Topology > IP Topology menu item.)

Take-Away Points
Favorites Tables
You can use the Favorites Table to look at subsets of data for closer inspection.
Favorites Tables are especially useful for looking at metrics for a particular Business Group, Application, Host, Interface, or IP-to-IP Conversation.
You can include any combination of metrics in a Favorites Table.
Customized Charts
Use the Custom Charts in combination with the Favorites Table for specific data analysis and visualization.
Multi-Group Time Series Charts are useful for graphing one metric over time for multiple groups, hosts, applications, or interfaces.
Multi-Metric Time Series Charts are useful for comparing and correlating multiple metrics for a single group, host, application, conversation, or interface.
Pie Charts are useful for evaluating the relationship of a metric across multiple groups, applications, hosts, conversations or interfaces.
The Response Time Composition Chart is useful for looking at the major components of response time for very specific times and applications (single client to server conversation for a specific application).
The workflow of splitting data off into a Favorites Table and graphing it in a chart is almost identical to creating a new custom report in the Report Manager. If you want the graph you created to be automatically generated according to a schedule, open the Report Manager and drop the data you’ve already created into the Content section.
Topology Views
Use the Topology Views to visually represent data flowing across the network.
Business Group Topology Views are useful for examining how data flows between many different groups.
IP Topology Views are useful for visualizing the network layout and devices between the AppResponse Appliance and a specific IP address or group of address.
Topology Views can be exported as image files so that you can include the images in reports.
