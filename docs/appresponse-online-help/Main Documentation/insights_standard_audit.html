Audit Insights
Audit Insights enable you to profile the performance and usage of your network, discovering which groups are using which applications, and the pattern of communication between groups. These Insights can help you prepare for network transitions such as server or data center consolidation, establishing new offices, moving existing installations, and the like.
Audit Insights include:
Application Dashboard
Application Discovery
Business Group Audit
Other Application Discovery
Subnet Audit
Total Traffic Dashboard
Application Dashboard
The Application Dashboard is located on the Audit tab. This Insight helps you see which applications are running on your network and what their performance is. You may rank applications for topping by any metric. For instance, you could choose to explore the top five applications ranked by server response time to check for slow servers. The tabs in the detail window provide a more specific view of that application traffic on the network: Internal IPs, External IPs, IP Conversations, Business Groups, Response Time Composition Chart and Time Series.
Figure 1
Application Dashboard Insight
 
Procedure 1
Using the Application Dashboard Insight
1.
Select an application in the pie chart.
As in the Total Traffic Dashboard, you can set thresholds for the Severity indicators in this table.
2.
Select a metric in the Key Performance Metric Table (upper right). The default is Throughput (Inbound and Outbound).
The selection of this metric the topping in all the tables in the lower panes.
End of Procedure 1
 
 
Application Discovery
The Application Discovery is located on the Audit tab. This Insight answers questions like “What applications are running on my network?” This is especially helpful during initial set-up. Application Discovery listens to every port. The appliance comes with a default set of defined applications (search for “default defined applications” in the online help.)
Figure 2
Application Discovery Insight
 
When you open Application Discovery, the default applications are shown in the selector pane. However, if a port is not recognized, you can drill down to a recognizable IP address and determine what is running on that port. Alternatively you can look on the IANA site (www.iana.org/assignments/port-numbers) for a list of globalized ports. When you identify a known application, you can add it to the Defined Application Manager.
Procedure 2
Using the Application Discovery Insight
1.
From the Selector menu (top left) select from All Defined Applications or Top Applications.
Select Top Application in order to drill down.
2.
From the selector pane (middle left) select an application.
The Time Series chart, and the two left IP panes (bottom) self-populate with the selected application. The Top Connected IP pane populates after you have selected an IP address from Top Internal IP or Top External IP.
The information column (bottom middle) looks for a domain name associated with the IP address. If found, the name rather than the IP address is shown.
Double-click an IP address and launch the “IP Address” pane, which provides additional information about that IP.
3.
Once you identify a port in the selector pane (using the IP addresses information) as an application, right-click and select Create New > Defined Application.
End of Procedure 2
 
 
Business Group Audit
Business Group Audit is located on the Audit tab. The Business Group Audit can answer those questions such as “What’s running on my network?” and “Who’s talking to whom?” in one step. This Insight reveals unexpected communications from or to a Business Group, such as an unauthorized wireless router, or a new application still connecting to the DNS server in the test lab rather than the production server.
Figure 3
Business Group Audit Insight
 
The selector pane (top left) is used to choose Business Groups ranged by any metric. You can select from the Top Business Groups tab or the All Business Groups tab.
The topology pane (top right) shows the top Business Groups communicating with the group selected from the selector pane. The arrow thickness indicates relative level of ranking metric.
The top application number entered (Top) and the metric selected (By) in the toolbar drives the Insight, including the Business Groups that share up, the top Business Group links, and the links are labeled in the topology pane.
Other Application Discovery
The Other Applications Discovery Insight is located on the Audit tab. This Insight answers questions like “What applications are running on my network?” This is especially helpful during initial set-up and are pre-defined on your network. It is also helpful in finding rogue applications on your network.
Figure 4
Other Application Discovery Insight
 
Procedure 3
Using the Other Application Discovery Insight
1.
Select one or more Top Other TCP Apps or Top Other UDP Apps you would like to create for an application.
2.
Select port(s) interested in to create a time-series chart for the application selected.
The first port selected is the only one shown in the bottom windows.
3.
Once a port is identified as an application, right-click and select Define > Application.
The Defined Application Manager opens.
4.
Define a new application in the Defined Application Manager.
End of Procedure 3
 
 
Subnet Audit
The Subnet Audit is located on the Audit tab. This Insight can easily create Business Groups among servers and clients that already have commonalities. This is especially helpful during initial set-up. Already created subnets could be potential Business Groups.
Note—Subnets that create a lot of traffic are usually good candidates.
Figure 5
.Subnet Audit Insight
 
Procedure 4
Using the Subnet Audit Insight
1.
From the selector pane (upper left) select a subnet group.
2.
Using the time series graph, your knowledge of the network, and the tabs at the bottom of the window, decide whether this subnet is a viable Business Group category.
3.
To create a Business Group, right-click the selected subnet.
4.
From the context menu, select Create New > Business Group.
The Business Group Manager opens.
5.
Create a Business Group (search for “business group manager” in the online help).
End of Procedure 4
 
 
Total Traffic Dashboard
The Total Traffic Dashboard is located on the Audit tab. This Insight displays an overview of all the traffic the appliance sees and provides basic utilization and network performance data. The Severity column (top left) provides indicators about the metrics displayed. The severity indicators are based on the overall average/sum shown in the table, which is based on the current Project Time. The severity indicator threshold for each metric may be edited by right-clicking on the corresponding row in the Severity column.
The tabs in the detail window at the bottom of the Insight provide a specific view into the traffic on the network: a time series for each metric (during the Project Time), and, ranked by Total Throughput, the top IP addresses (top talkers), top IP conversations, top applications, and top /24 prefixes.
