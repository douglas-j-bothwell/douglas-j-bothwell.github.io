Capturing and Viewing Packets by VLAN
You can configure AppResponse to save VLAN information for each captured packet. This option is useful when you need to pinpoint the specific VLANs where packets were captured. For example, you can troubleshoot your VLAN routing policies by observing the flow of packets across specific VLANs and comparing the observed traffic patterns with your intended routing policies.
Enabling Preview by VLAN
The Administration > System > Capture web page has an option, Enable Preview by VLAN. When this option is selected, AppResponse saves VLAN information and stores packets with different VLAN tags separately. This enables you to
See the VLAN where each packet was captured
Observe the flow of packets that were captured in separate VLANs
Preview, filter, and download packets by VLAN in the AppResponse Capture Viewer or Transaction Analyzer
When this option is turned off, AppResponse ignores VLAN information when it captures and stores packets.
Previewing and Downloading Packets by VLAN
When the Enable Preview by VLAN option is selected, the AppResponse appliance saves VLAN information for each captured packet. This enables you to preview, filter, and download packets by VLAN in the Packet Download Manager or in Transaction Analyzer.
