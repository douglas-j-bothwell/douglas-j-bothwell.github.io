Preferred IPs
The appliance employs several algorithms to determine important IP addresses on the network. These important IP addresses are treated specially by the appliance. For instance, IP addresses that generate high volumes of traffic are not eliminated during the group and drill-down aggregation process (see Group and Drill-Down Aggregation). The user can configure preferred IP addresses that are treated specially by the appliance regardless of the volume of traffic they generate.
This section does not apply to AppResponse Standard.
For instance, as detailed in the previous section, the appliance gradually aggregates stored IP address in order to retain the most important IP addresses over time. Generally, the group and drill-down aggregation process selects IP addresses based on the Throughput (Inbound and Outbound) metric. However, Preferred IPs are selected during this process before high-volume IP addresses.
The appliance treats the preferred IPs specially in the following ways:
Preferred IPs are always included first when aggregating top IP addresses, as described in Group and Drill-Down Aggregation.
Preferred IPs are always included first in the top IP conversations, as described in Group and Drill-Down Aggregation.
The automated traceroute process always selects a number of preferred IPs. For more information, see Automated and Manual Traceroutes.
Guidelines for Specifying Preferred IPs
Note the following guidelines when specifying Preferred IPs:
The criteria that AppResponse uses to select IPs of interest is discussed in How AppResponse Selects Top Elements.
Preferred IPs are useful for monitoring client IPs of interest that might not meet these topping criteria. Adding an IP to the Preferred list ensures that AppResponse always stores metrics for that IP.
In general, you should not include more than 100 IPs in this list.
WARNING—Do not include server IPs in the Preferred IPs list, because AppResponse will also “prefer” all client IPs that talk to the server. This is important because AppResponse collects metrics for a finite number of IPs for each time window. (Specifying a heavily-used server IP can result in that server and its clients “crowding out” most or all other IPs.)
Preferred IP Manager
The Preferred IP Manager is used to create, edit and delete preferred ips. The appliance supports up to 5120 preferred IPs. Follow these instructions to create a preferred IP.
Procedure 7
Creating a Preferred IP
1.
Invoke the Preferred IP Manager from the Tools menu.
Figure 29
Preferred IP Manager
 
2.
Click the New button.
3.
Enter the desired Preferred IP addresses in the form of a single IP address (e.g., 192.168.1.1), an IP address range (e.g., 192.168.1.0-192.168.1.255), IP address prefix (e.g., 192.168.1.0/24) or hostname (e.g., www.networkphysics.com). The number of Preferred IP addresses entered cannot exceed 5120. Each IP address in a prefix or a range counts towards this limit.
4.
Repeat steps 2 and 3 as many times as necessary.
5.
Click OK to save the new preferred IPs to the database.
Preferred IP definitions can be imported and exported in CSV format. For more details, see Importing and Exporting Configuration Data.
End of Procedure 7
