NetSensor Integration
The Element Polling Insights enable you to view recent SNMP information about your network and to identify network conditions that might be contributing to problematic application performance. These Insights can help you answer questions such as:
I’m observing spikes in Round Trip Times over this time period. Is there an issue with a server in a conversation pair?
I'm observing a network-related performance issue. Is there a device in my network or path that is showing odd performance metrics?
The Insights in the Element Polling category are
AppNetwork Path Analysis Insight—Useful for investigating the end-to-end network path of an IP address pair that has problematic application performance.
Cisco IP SLA Test Query Insight—Useful for viewing data from automated tests on Cisco IOS devices
SNMP Query Insight—Useful for viewing the full range of SNMP data collected by SteelCentral NetSensor
SNMP System Monitor Insight—Useful for analyzing system and interface performance on individual systems
Note: To use these Insights, data retrieval from either NetSensor or the Basic SNMP Collector must be turned on in AppResponse (it is off by default). For details, see Setup and Configuration for Element Polling Insights.
 
Note the following about the Element Polling Insights:
Except for the SNMP System Monitor Insight, the Element Polling Insights require metric data collected and stored by Riverbed NetSensor. The metrics available from APM EI are described in the NetSensor User Guide.
In many of the data tables, you can select multiple cells (for example, to compare metrics in a chart):
–
To select several separate cells, Ctrl-click on each desired cell.
–
To select a group of contiguous cells, click on the first cell, then Shift-click on the last one.
The Polled System List in these Insights displays a list of systems that are currently being polled by NetSensor. If you are interested in data from a system that does not appear in the list, you must configure NetSensor to poll that system.
In a Polled System List, the Search field uses AppResponse search rules to let you find a host in the list by name, IP address, type, location, or description.
If you are using the NetSensor module on the SteelCentral AppResponse appliance, note that the following ports are open by default:
–
2700
–
2701
–
2710
–
2780
If you do not want all of these ports open, you can download and install the 4.8.6 update to NetSensor. This update closes ports 2700, 2701, and 2710 and leaves only port 2780 open.
