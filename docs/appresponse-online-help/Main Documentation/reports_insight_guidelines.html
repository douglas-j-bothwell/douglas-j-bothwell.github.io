Guidelines for Creating Insight Reports
Because not all of the features allowed in Insights are supported in reports, you must take care when publishing an Insight as a report. The following guidelines can help you produce successful Insight reports.
Report Templates enable you to define advanced reports quickly and easily. Riverbed recommends that you always create the report you want by customizing one of the available Report Templates.
Important! Review the latest Release Notes for any notes, warnings, additional guidelines, or new functionality related to reports.
Make sure the Insight you are going to publish contains only components that are supported by reports. These components are:
–
Tables
- Top Group
- Favorites
- Single Group
–
Charts
- Multi-Group Time Series
- Pie
- Multi-Metric Time Series
- Response Time Composition (RTCC)
- Top Multi-Group Time Series
- Top Pie
- Correlation
- Distribution
- Cumulative
- Bar
–
Text
- Static Text
- Label
–
Analyses
- Drilldown
Group Aggregation
- Group Path
- Group Search
- Metric Threshold Filter
- Report Initializer
Minimize the number of Interactions in the Insights; when possible, add a group directly (drag-and-drop) onto a visual component.
Set the data granularity for all charts (whether or not they originate in an Insight) to Automatic.
Potential Design Problems
The following design issues can cause problems when publishing an Insight as a report.
Group Interactions and Hidden Components
One common, but incorrect and unsupported, use of group interactions is to connect a hidden Favorites Table or other hidden Selector to various other Components such as Charts, Tables, and Drilldown Analyses). This design approach is not supported by Insight reports.
Riverbed recommends using the Report Initializer Analysis or Initializer Analysis to specify data in all reports.
Expanded Groups
Top or Favorites Tables can contain groups that have been expanded to one or more drilldowns. If this is published as an individual table or as part of an Insight, it can result in report generation timeouts, especially if the expansions include drilldowns to IPs or IP conversations (such as connected IPs or IP-IPs). Ideally, make sure that reports do not contain groups that have expanded drilldowns.
Suggestions for Clean-Looking Reports
Avoid possible vertical scrollbars by previewing the Insight and resizing as needed. In particular, watch out for vertical scrollbars in:
–
TextBox Components (typically used for the “banner” title) that receive a group with a name that is too long
–
Favorites Tables that have more rows than the vertical height can display
Polishing the layout
–
Remove borders from charts.
–
Set background to white for labels.
–
Use HTML to control size in labels.
–
Use HTML to format text boxes.
