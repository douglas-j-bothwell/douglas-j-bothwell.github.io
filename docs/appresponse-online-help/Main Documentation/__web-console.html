Web Console
The Web Console provides a simple, lightweight interface for navigating, discovering, and analyzing AppResponse data. The Web Console requires no in-depth knowledge of AppResponse and no special software other than a web browser.
The Web Console was designed to make it easy to discover and visualize data of interest and to share views across multiple users and groups. This makes collaborative troubleshooting easier: you can create your own data views targeted to a specified problem, save those views on the appliance, and share them with users who have permission to access those views.
You can use the Web Console to
Troubleshoot, investigate, and explore the entire range of data available on the appliance—Application Stream Analysis, Web Transaction Analysis, NetFlow, captured packets, voice, video, and so on.
Open Quick Views that show top talkers, slowest apps, slowest networks, top clients for specific web apps, and so on.
Create Favorites Views to collect, organize, and display related data of interest and to ensure your data is always accessible.
Define Custom Views with charts, tables, and layouts that organize and display related sets of data.
Define the Custom Views that specific viewers can and cannot access, using Role-Based Access Control.
Define “real-time” SLA Dashboards that can report violations over the past 15 minutes, updated every minute.
The Web Console interface appears by default when you log in to the appliance using a web browser (http:[appliance]:8080 or https:[appliance]:8443). The workspace has the following components:
Home and Side Bar
Quick Views
Custom Views
SLA Dashboards
Alert Log, Audit Log, and Appliance Information
