Group and Drill-Down Aggregation
The appliance captures information from each network session in real time. In many cases, it is not possible to store all relevant information from these network sessions for an extended period of time. For instance, an appliance might observe 100,000 unique IP addresses in a single minute on a busy network. Some of these IP addresses generate a trivial volume of traffic. Since the appliance cannot store all unique IP addresses at this rate without exhausting available storage, the less important (less active) IP addresses are not recorded in the database. This process is referred to as group and drill-down aggregation.

The appliance always selects Preferred IPs when choosing the top N IP addresses. After choosing preferred IPs, IP addresses are selected across active Business Groups. The appliance selects up to M addresses [for IP addresses in the 1-minute granularity M is (5000 - # of preferred IPs selected) divided by # of Business Groups) from each Business Group ranked by Throughput (Inbound and Outbound)]. Once this selection is complete, the appliance selects the Top N IP addresses not in Business Groups ranked by Throughput (Inbound and Outbound) where N is the number of IP addresses that were not selected from preferred IPs and Business Groups, without exceeding a total of 2000 IPs.

Metric data aggregated at the 1-day granularity is stored in the appliance database for 365 days. The appliance does not perform any aggregation beyond the 1-day granularity.
The user can also configure Preferred IPs to provide the appliance with a pre-defined list of important IP addresses that should not be eliminated during the group and drill-down aggregation process.
A similar time-based aggregation process is performed on metric data.
