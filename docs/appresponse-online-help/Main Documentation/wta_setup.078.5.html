Page Analysis: Enabling and Configuring
Enabling Web Transaction Analysis
Procedure 2
Enabling Web Transaction Analysis
1.
Configure AppResponse to mask out all sensitive information contained in key-value pairs that might appear in observed web traffic (URLs, cookies, and POST messages):
1.1.
In the Desktop Console, choose Tools > Pages > Information Privacy Manager
1.2.
Specify all parameters that might contain sensitive information in the observed web traffic.
For more information, see Masking Private Data in Observed Web Traffic.
WARNING—There is no way to mask sensitive data after it is saved on an appliance. For this reason, you must specify all sensitive parameters before you enable Web Transaction Analysis.
2.
Go to the Administration > System > Pages web page.
The Page Analysis Configuration Page appears.
3.
Select the Enable Page Analysis check box.
4.
Specify the ports and IPs (on Ports...and IP Addresses) for which you want to collect page-level metrics, or select the All checkbox on the right.
Note: Collecting web page data can be highly resource-intensive, especially on very large networks with a lot of web traffic. To minimize the performance impact on the appliance, it is good practice to limit web page tracking to ports and hosts of interest only.
5.
Change the Detault Slow Page Threshold, if desired.
6.
At the bottom of the page, click the Apply button to apply your changes.
AppResponse starts collecting web page statistics on the specified ports and IP addresses.
End of Procedure 2
 
 
Page Analysis Configuration Page
The Page Analysis Configuration web page (Administration > System > Pages) has options for collecting web page data and decoding SSL traffic. WARNING—Before you enable Page Analysis or SSL Decoding, you should mask out all sensitive parameter values in any HTTPS traffic that might get decoded, as described in Masking Private Data in Observed Web Traffic.

NOTE—Collecting web page data can be highly resource-intensive, especially on very large networks with a lot of web traffic. To minimize the performance impact on the appliance, it is good practice to limit web page tracking to ports and hosts of interest only.
Detault Slow Page Threshold
The appliance uses this threshold to determine metrics such as Slow Pages, Slow Page Rate (Servers), and % Slow Pages.
Enable SSL Decoding
When this option is selected, AppResponse decodes web-over-SSL traffic on the ports and IP addresses specified under Enable Page Analysis.
Private Key Configuration
This table describes the set of private keys that AppResponse uses to decode encrypted traffic, by displaying a user-defined label for each key.
If you want to decode traffic from a server that uses SSL certificates, you must specify the private key used by that server. For more information, see Specifying Private Keys.
SSL Servers with Missing Private Keys
This table shows all servers that have sent web-over-SSL traffic that AppResponse has observed, but could not decode because it did not have the required private key. It is good practice to update this list periodically, as described in Periodic Updating (Recommended).
