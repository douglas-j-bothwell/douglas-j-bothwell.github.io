Special Handling of Long Connections and Turns
This section describes how certain metrics for long connections are calculated and displayed:
Connection Duration, Data Transfer Time, Packet Payload, and Payload:
The values for long connections or turns are spread uniformly across all the one-minute time bins in which the connection or turn exists.
Retransmission Delay:
The value is calculated for each one-minute time bucket in which the turn exists.
These metrics are reported differently in charts and tables:
In charts, these metrics are reported as recorded: “spread out” across one-minute time bins, so that the metric value at each minute is the average for that minute. This makes the area under the chart curve equal to the value of the metric for the entire connection or turn.
In tables, these metrics are reported as the sum of the one-minute time bins within the time selection of the table in which the connections or turns exist. If the table time selection includes the end of the connection or turn, the total value for the metric is displayed.
