ScriptData: Launching Insights From an Insight Extension
By using the ScriptData:LAUNCH:[insight name] API the Insight Extension can request that the UI automatically launch an Insight.
The following Insight Extension is executed in this example:
#!/bin/sh
#
# This script demonstrates the API for launching an Insight. The
# following parameters should be passed to this script:
# parameter 1 - group
#
PATH=/bin:/usr/sbin:/usr/bin:.

echo "\"
echo "<html>"
echo " <H2>Insight Variable Parameters:</H2>"
echo " <ul>"
echo " <li>  Business Group: $1</li>"
echo " </ul>"

echo "</html>"
sleep 5

echo "ScriptData:LAUNCH: Bandwidth Hogs"
ScriptData: Interacting with the user through an Insight Extension
A common problem when running a script on the appliance is the need for user input. Insight Extensions now have the ability to query the user for input and have that input returned to the script in the command line arguments. By using the ScriptData:INPUT:[type1,label1,key1,initValue1;...;typeN,labelN,keyN,initValueN ] API the Insight Extension can request the user for input. The following types of input are supported:
String - displays a string entry field.
Password - displays an obscured string entry field.
Double - displays a string entry field with error checking to make sure the value is a double.
Integer - displays a string entry field with error checking to make sure the value is an integer.
Boolean - displays a combo box with the choices [true, false]. Available in 6.1
Options - displays a combo box with a set of custom choices. Available in 6.1
When the script requests user input the Insight kills the script and then displays a dialog that queries the user for the requested input. When the user clicks okay, the script is invoked with the previous arguments and the user input as new arguments of the form "--key1 value1 ... --keyN valueN". The following Insight Extension is executed in this Example
