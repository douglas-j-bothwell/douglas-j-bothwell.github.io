AppResponse Platform
AppResponse uses the following methods for monitoring, analyzing, and troubleshooting networks and applications.
Real-Time Traffic Monitoring
AppResponse monitors traffic flows in real time, extracts TCP/IP data from every packet, and correlates this data with route information retrieved from traceroute and BGP. For each flow, AppResponse calculates detailed metrics for measuring network and application performance: response times, throughput, utilization, congestion, packet loss, and so on. AppResponse can collect metrics for monitoring the performance of service providers, at the route level from traceroute, and the AS level (Autonomous System) from BGP. AppResponse aggregates this data so you can measure performance across different time windows (30 minutes, for example, or one or more days, weeks, or months). You can also define that aggregate traffic data for business entities such as branch offices, departments, buildings, business applications and end-to-end business services.
Packet Captures for In-Depth Analysis
AppResponse can also capture packets for in-depth analysis of individual IP conversations and user-level application transactions. The high-level workflow is as follows:
1)
Enable packet captures on the appliance.
2)
Navigate monitored traffic for problematic IP conversations or application transactions.
3)
Download packets for an individual transaction and analyze it in detail using Wireshark or Transaction Analyzer.
Problem Identification and Alerting
AppResponse automatically baselines network behavior from actual traffic, derives adaptive thresholds based on moving averages, correlates data to eliminate false positives, and alerts you to anomalous behavior. No programming is necessary, and, since AppResponse looks at system behavior rather than event data from individual network elements, it can also detect many types of security violations.
Network managers may add their own alerts to monitor critical aspects of their network connectivity. Once a problem has been identified, the appliance allows managers to organize, navigate through, and analyze it using an extensive set of tables and charts and topological displays as well as AppResponse Insights, which package the workflow of common tasks into a simple, step-by-step interface.
Groups
AppResponse automatically collects and aggregates data, in the form of metrics, into groups. These groups represent the aggregate data for all elements within the groups. The appliance provides a set of built-in groups that are supported out of the box with no configuration required.
In addition, users can defined their own groups in the form of Business Groups and Applications. Business groups provide the custom aggregation necessary to understand the relevance of data exchanges to your business. They can be defined to represent any set of relevant network users, such as customers, business units, application servers and remote offices. Applications can be defined against standard protocols and protocol/port combination and further specified again a server or set of servers and, if desired, a URL or set of URLs.
AppResponse not only lets managers examine the performance and utilization of Business Groups in isolation, but can also be used to understand and troubleshoot traffic exchanges between Business Groups. This detailed information about the connectivity and interaction between Business Groups is crucial in understanding how your valuable network resources are used and how they are performing.
Metrics
Metrics are the performance and utilization data captured directly from network traffic in real-time or from external sources, such as traceroute. Metrics provide insight into the behavior of individual IP addresses or the aggregate behavior of groups. The metrics captured by the AppResponse Plus OS are listed and described in Metrics.
AppResponse Appliance
The AppResponse Appliance attaches directly to the network via a span port or a copper/fiber tap to capture and analyze TCP/IP traffic in real-time. Data is then stored in the appliance database and can be displayed and explored using the Web Console and the Desktop Console.

The AppResponse Appliance installs in your data center and requires no changes to your content or application servers, no physical changes to your IT infrastructure, no overlay network or cooperation from downstream elements, no client-side applications, and no special protocols.
In simple networks, a single AppResponse Appliance can be installed all by itself (standalone). Larger networks are normally too complex to be covered by a single AppResponse Appliance (domain).
Domains and Directors
A large network cannot be covered by a single appliance. In these complex networks, multiple appliances deployed at distributed locations within the network can be configured to work cooperatively in a domain. A domain is controlled by a director, a special device that is responsible for coordinating all the appliances in the domain.
Domains provide the following major benefits:
Centralized Management - Configuration changes can be applied to all appliances within a domain from a single location. This allows domains to be managed centrally.
Centralized Data Access - Each appliance within a domain collects data from a different network segment. Data from any appliances in the domain can be examined within a single application without needing to know specifically which appliance originally collected the data.
