Tracking Page Users: Configuration Options
AppResponse includes a set of options that enable you to customize how Web Transaction Analysis tracks individual page users. The following options are available:
Specify the HTTP Header Field that Contains the User Name
Auto-Generate User Profiles Based on Session IDs When No Login was Observed
Use Custom String for User Label Prefix
To configure these options, log in to the appliance using an SSH-enabled utility such as putty. Note the following:
You must have Administrator privileges on the appliance to run any of these commands.
These commands affect only web traffic observed after the command is applied, and do not change any web traffic that has already been observed and stored on the appliance.
Each command includes a delete option to undo the command and revert to the default Web Transaction Analysis behavior.
After you change the Web Transaction Analysis configuration, the appliance might need to observe web traffic for several minutes before any changes become visible in the Desktop Console.
Specify the HTTP Header Field that Contains the User Name
This option is useful when the monitored web traffic stores the username in a specific, dedicated HTTP header field and Web Transaction Analysis cannot find the username by looking in the standard locations (for example, embedded as the value in a key=value string in a POST message).
Note the following:
This option supplements, but does not replace, the default behavior for finding usernames: Web Transaction Analysis searches the specified header field and also searches for key=value pairs that match the Name of User Key field (Desktop Console > Tools > Pages > User Session Tracking Manager > Parameters tab).
With this option enabled, Web Transaction Analysis searches HTTP headers for the specified field.
To enable this option, enter the following:
WTA-StitcherConfig add globals.user-tracking.username-http-field field
For example:
WTA-StitcherConfig add globals.user-tracking.username-http-field x-username
To disable this option and revert to the default (look only in standard locations for username), enter the following command:
WTA-StitcherConfig delete globals.user-tracking.username-http-field
 
Auto-Generate User Profiles Based on Session IDs When No Login was Observed
This option is useful for tracking users when the appliance did not observe the initial login sequence for a session. This can occur if a web application uses an authentication server whose traffic is not monitored by the appliance, for example, or for long-running sessions where the login occurred before Web Transaction Analysis was enabled on the appliance.
When this option is enabled, the Desktop Console displays Users with the label <prefix>+<session_tag>, where the prefix is Session and the session tag is a unique, auto-generated tag (the session ID is not included or displayed). To specify a different prefix, see Use Custom String for User Label Prefix.
To enable this option, log in to the appliance using an SSH-enabled tool such as putty and enter the following command:
WTA-StitcherConfig add globals.user-tracking.use-sessionids-for-userids true
To disable this option and revert to the default (no user tracking if the login was not observed), enter the following command:
WTA-StitcherConfig delete globals.user-tracking.use-sessionids-for-userids
 
Use Custom String for User Label Prefix
This option changes the prefix string in the User label (default = Session). To enable this option, enter the following all on one line:
WTA-StitcherConfig add globals.user-tracking.use-sessionids-for-userids-prefix [new_prefix]
Note: If you want to include a non-alphanumeric character in the new prefix, you must enclose the argument string in single quotes—for example:

use-sessionids-for-userids-prefix 'FinTrx$Session'
To disable this option and revert to the default (prefix =Session), enter the following:
WTA-StitcherConfig delete globals.user-tracking.use-sessionids-for-userids-prefix
