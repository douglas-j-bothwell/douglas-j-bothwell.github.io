Searching Multiple Directors
This release enables you to search multiple directors and all appliances in their domains. Previously, you could search one director domain only at a time.
When a federated search includes results from other directors, the search results include links to these directors. When you follow a link to another director, the console creates a new project for the external director with the current project time.
To set up a federated search, do the following:
Procedure 8
Searching Multiple Directors: Initial Setup
1.
Open a Desktop Console (release 8.5 or higher) and connect to a director (release 8.5 or higher).
2.
Cross-director search options are hidden by default. In the following steps, you will set your global preferences to make these options visible:
2.1.
Choose Tools > Preferences in the console.
2.2.
In the Search tabbed page, select the following checkboxes:
• 
Show Cross-Director Search Option– When this option is selected, the Use Director List checkbox appears as an Advanced option in the Search dialog box. Otherwise this option is hidden.
• 
Search Across Directors by Default (Director Only) – When this option is selected, the Use Director List checkbox is selected by default.
Figure 19
Federated Search Preferences
 
2.3.
Click OK to apply your settings and close the Preferences dialog box.
Note—These preference settings are stored on your local host (like all other user preferences), not the director.
3.
Choose Tools > Search.
4.
Click the Advanced link in the top-right corner of the Search window.
The Search across Directors options (checkbox and link) appear below the Advanced link.
 
5.
Click the Directors link and specify the additional directors that you want to include in the federated search (you do not need to include the director to which the console is currently connected).
 
If the Search across Directors checkbox is selected, you can now perform federated searches for the current director and all directors in the Director List
End of Procedure 8
