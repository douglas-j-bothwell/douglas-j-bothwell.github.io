Integrating with External Applications
It is often useful to examine datain an external application such as Microsoft Excel (for table data) or Transaction Analyzer (for packet and application capture files). These links to external applications are controlled through the Integration Manager.
Note: This section does not apply to AppResponse Standard.
Figure 1
Integration Manager
 
The Integration Manager is used to define a link between an AppResponse data set and an external application. These links are context sensitive and only associated with specific views. Links to external applications can be created in the following locations:
Tables
IP Topology
Alert Manager
Defined Application Manager
Business Groups Manager
Business Group Link Manager
Capture Manager
View Menu
Tools Menu
One of the following actions can be performed when the link is invoked:
Execute a script
Open a URL in the default browser
Open a file in the default browser
Creating a Link to an External Application
Note—Adding a link to an external application is a complex procedure. Review the steps listed below. For a step-by-step example that links AppResponse tables with Microsoft Word and Microsoft Excel, see Integration Example.
Procedure 1
Creating a Link to an External Application
1.
Invoke the Integration Manager from the Tools menu.
2.
Click the New button in the Integration Manager.
3.
Enter a name for the link category in the Description section. The name appears in the context menu (right-click) of the view to which it is added. Multiple links to different external applications can be added under a single link category. Each link is differentiated by a link label assigned in step 5. For an example of multiple links assigned to a single name, see Integration Example.
4.
Enter an optional list of user-defined parameters for the link in the Parameters section. Parameters are useful when defining multiple links to an external application. The value for each parameter is substituted in place of the parameter reference in the link action field (defined below). In many cases, multiple links refers to a common set of information such as a username or password. In these instances, it is desirable to define the username and password as parameters so that they can be changed in a single location.
Create a new parameter by right-clicking in the parameter display box and selecting Add. Double click Key to change the parameter name. Double click Value to change the parameter value.
5.
Define links to external applications in the Links section.
5.1.
Provide a label for this link. The label appears under the link category in the context menu.
5.2.
Specify the view to which this link is applied. The link only appears in the context menu of the view specified.
5.3.
Provide an action for this link. The action defines the external application that is launched when the link is activated. There are three supported action types:
• 
Execute a script from the local file system,
e.g., “script:cmd /c start c:\linkcommands.bat”
• 
Open a URL in the default browser,
e.g., “https://al_appliance3.myco.com:8443/reports.asp”
• 
Open a file in the default browser,
e.g., “file:/C:\file.pdf”
The action field can include references to parameters defined in the parameters section. Any parameter reference, in the form of the parameter name prefixed with a '$' (e.g., “$name”), is replaced with the parameter value. A parameter reference should always be double-quoted to avoid problems with the underlying file system command line parsing.
End of Procedure 1
 
 
Additional links can be defined by clicking on the Add button to the right of the action field. Click Remove to delete an existing link.

There are several built-in parameters which can be used to identify the active groups at link activation. The built-in parameters available depend on the view to which the link is applied. Hover over the View pull-down box to see a list of valid built-in parameters for this view. The supported built-in parameters are listed in the following table.

Integration Example
In this section, an example of an integration is defined linking AppResponse tables to the Microsoft Word and Excel programs.
1)
Create a new integration by invoking the Integration Manager and clicking the New button.
2)
Type Send to MS in the name field under Description.
3)
Do not input any parameters, they are not required for this integration.
4)
Create a link with the following values:
–
Name: MS Word
–
View: Tables
–
Action:
script:cmd c/ “C:\Program Files\Microsoft Office\Office11\Excel.exe” “$file”
5)
Click Add to define a second link with the following values:
–
Name: MS Excel
–
View: Tables
–
Action:
script:cmd c/ “C:\Program Files\Microsoft Office\Office11\Excel.exe” “$file”
NOTE—When you define a link, you must include the full command as you would enter it from a command line prompt. For this reason, it is good practice to
–
Include any commands needed to specify the drive or device (such as cmd /c in Windows).
–
Verify the full path to the executable. For example, the path shown in the previous example includes a release number (ie, Office11) which might be different on your local host.
–
Try running the full command from a default command prompt to verify that the link works as intended.
6)
Click Apply to save the new integration.
7)
Go to any table and highlight several rows. Right-click one of these rows and select Send to then MS Excel from the context menu. The system saves the entire table in a CSV file under the name supplied in the Save file dialog. Microsoft Excel is automatically invoked on the file. The rows that were originally selected in the table are identified in the Selected column.

Templates
By default, all new integrations are specific to the machine running the Desktop Console. Other users logging in to the same appliance cannot use the new integrations. Use templates to share integrations between all users of the same appliance.
To make a template, right-click in the table at the top of the integration manager and select Create New Template.
Figure 3
Integration Manager - Create New Template
 
Templates are configured in the same way that regular integrations are defined. However, templates are not active. A link must be created from a template in order for it to be used by the Desktop Console on the local machine. To create a local link from a template, select the template in the integration manager, right-click and choose Create A Link From Template.
Default Integration Links
The appliance ships with a number of default integration links.
 
Table 2
Default Integration Links 
Link Name
Description

Add links to the IP Topology and table context menus that run telnet to a selected IP address.

NBTStat
(Windows-only)
Add links to the IP Topology and table context menus that run nbtstat on a selected IP address.

Add links to the IP Topology and table context menus that run ping on the selected IP address.
On fresh installs, these default links show up automatically. However, users that have upgraded from a previous software version must enable the default links explicitly by right-clicking in the integration manager table and selecting Create Default Links.

Integration Manager - Create Default Links
 
There are separate links available for Windows and Unix systems. You may choose to create all default links at once or create any of the default links individually.
