Reports in the Web Console
This section describes how to view, delete, run (on demand), and generate URLs for reports in the Web Console. The following topics are discussed:

View a Report

Choose a Report
To choose a report, go to the Web Console > Reports > [report-category] menu. Then click on a report title in the index table (right pane). Each index provides links to all public reports in that category, as well as any private reports that you have permission to view.
Figure 1
Report Index: Web Console > Reports > Default
 
You can specify the columns shown in the index table. Most of these are based on settings for that report in the Report Manager (see Table 1). Each report has a status of
Active—The report schedule is active
Complete—A report has been generated but is not scheduled to repeat.
Deleted—The report definition has been deleted from the Report Manager, although some reports might still be available on the appliance.
Inactive—The report schedule is currently inactive, but can be re-activated in the Report Manager > Schedule page.
View a Report
To choose a report, go to the Web Console > Reports > [report-category] menu. Then click the report title in the index table (right pane). Each index provides links to all public reports in that category, as well as any private reports that you have permission to view.

On-Demand Reports
End users can generate some reports directly in the Web Console. When this functionality is enabled, the report page includes a "run on demand" link in the top-right corner of the page.
Figure 3
“Run on Demand” Link
 
Generating on-demand reports in the web UI requires the following:
The report definition must have on-demand reporting enabled (Desktop Console > Tools > Report Manager > Schedule tabbed page > "Can run on Demand" checkbox). To support backward compatibility, this option is disabled by default for each report definition.
Figure 4
“Can Run on Demand” Checkbox in Report Manager
 
The “Run on demand” link is visible only when you access the report page from the following path: Web Console > Administration > Views > Reports.
You can generate an on-demand report only if at least one instance of that report has already been generated.
You cannot generate an on-demand report unless the appliance has report data for the entire time window. For example, you cannot generate a daily on-demand report at 2:00 pm for today because the appliance does not have data for the entire time window (12:00 am - 11:59 pm).
Delete a Report
End users can delete reports directly in the Web Console. When this functionality is enabled, the report page includes a "delete" link in the top-right corner of the page.
Figure 5
“Delete” Link
 
Note the following:
The “delete” link is visible only when you access the report page from the following path: Web Console > Administration > Views > Reports.
Deleting a published report on this page does not affect the corresponding definition in the Report Manager.
Report Image URLs
The report Image URL can be used to access public reports without requiring users to log in to the appliance web interface. Private reports do not provide a direct URL. If the URL ends with latest, it always displays the most recent report image file. If an older image file was selected in the drop-down list above the image, the URL points to that specific image.
