Working with Interactions
This section briefly reviews how to add, remove, and sort Interactions. They may not be renamed, but are renamed automatically if the names of either of the Components they link are changed.
Moving Interactions from one Layer to another is described in Working with Layers.
Adding an Interaction
To add an Interaction, select the desired Layer in the Element pane, then click Add Interaction from the Insight Development Kit toolbar area.
Figure 29
Add Interaction Button
 
This brings up the Component Interactions dialog.
Figure 30
Component Interactions Dialog box
 
Type - select type of Interaction from the drop-down pick list. Refer to Creating Insight Modules for information on when to use Insight Module. Most of the time you use Insight.
Interactions - select the desired Interaction from the drop-down pick list. Only those Interactions supported by the Components in the Insight are presented.
Producers - select the Component that is to produce (send) the Interaction from the drop-down pick list. Components that do not produce the selected Interaction are not listed.
Consumers - select the Component that is to consume (receive) the Interaction from the drop-down pick list. Components that do not consume the selected Interaction are not listed.
Apply - click Apply to add the selected Interaction and continue adding more Interactions, or click OK when done.
The Interaction is automatically named using the default format Interaction_name - From: Component_name To: Component_name; e.g., group - From: Selector:BG1 To: Chart:multigrp2. This is one reason why it is so important to give every Component a meaningful name.
Removing an Interaction
To remove an Interaction, select it in the Element pane, right-click to display the Element Pane context menu, and select Remove; or from the Workflow View, select the arrow that represents the Interaction you want to remove, right-click and select Remove Interaction.
Exploring and Sorting Interactions
Quite often during the construction of an Insight you may want to review the Interactions you have already added to it. Three functions of the Insight Development Kit are designed to aid this process: Workflow view, Interaction highlighting and sorting.
For more information, see Workflow View.
Interaction Highlighting
When you select an Interaction in the Element pane, the Layout Grid outlines the Producing Visual Component in green and the consuming Visual Component in red.
Figure 31
Interaction Highlighting
 
If one of the linked Components is not visible (either because it is a Hidden Component or an Analysis Component), the visible member of the Component pair is highlighted appropriately.
Highlighting works in both the Actual and Schematic Views of the Layout Grid.
Interaction Sorting
Interactions may be sorted by Interaction name (e.g., group, metric, etc.), by Producer name, and by Consumer name. This is useful for checking to see that a given Component produces or consumes all necessary Interactions; or that a given Interaction goes to, or comes from, all necessary Components. The sorts are always in alphabetical order.
You may choose the sort order of Interactions in two ways:
Select Tools > Building Settings in the Insight Development Kit menu to open the Insight Development Kit Settings dialog. The Outline tab in this dialog allows you to specify the default sort for Interactions.
Select either an Interactions branch or an Interaction in the Element pane, then right-click to display the Element Pane context menu, and click Sort Interactions.
Select By Interaction Name, By Producer Name, or By Consumer Name.
