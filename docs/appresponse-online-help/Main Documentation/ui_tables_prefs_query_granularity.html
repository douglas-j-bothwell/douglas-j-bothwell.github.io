Table Preference: Always Query 1-Minute Granularity in Tables
When this preference is selected, all table-based queries access the one-minute traffic database to retrieve the top elements to display in the table. This results is more precise data queries and topping results, especially for smaller elements such as individual IPs and IP conversations. The trade-off is that table queries can take longer. (For this reason, the preference is turned off by default and is not invoked if the project time is longer than five hours.)
To set this preference, choose Tools > Preferences and go to the Queries tabbed page.
