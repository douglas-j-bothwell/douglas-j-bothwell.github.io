Problem Management Dashboard
The main view of the Problem Management Dashboard is divided into three sections: server, network and client. When an alert is triggered, it is displayed in the section assigned for that alert (user-specified for Static Alerts and pre-assigned for Adaptive Alerts). To open the Problem Management Dashboard, choose View > Dashboard.

The left pane of the main dashboard view shows an icon for each of the sections colored according to the number and severity of alerts for that section. Adaptive alerts are assigned one of three severities according to the frequency and deviation of outliers:
Critical (Red)
Major (Orange)
Minor (Yellow)
For Static Alerts, the severity is user-assigned.
The center pane of the main dashboard view shows the list of alerts for each section. This pane allows you to see more detail for each alert by clicking on the View Summary link of an alert. The detailed view for each user-defined Static Alert is automatically assigned by the AppResponse OS. The detailed view for adaptive alerts is pre-assigned. In either case, the detailed view displays an explanation of what triggered the alert along with a supporting table and graph(s) to enable further investigation. (For an example of a detailed view, see Detailed Adaptive Alert View.) You may also assign an Insight link to display an insight focused on the reported problem.
The right pane displays the Timeline, a small chart showing a time series of the number of adaptive alerts over the current time selection. The timeline allows the user to rapidly determine whether alerts for this section are increasing, decreasing or maintaining the same level.
The Dashboard on a Domain Director allows you to examine alerts generated by any appliance in the domain. In addition, the Dashboard allows you to filter by appliance. For more information, see Domain Director.

Filtering Alerts Displayed in the Dashboard
By default, the dashboard displays all adaptive alerts and all static alerts generated over the current time selection. In certain cases, it may be desirable to temporarily remove some of these alerts. The dashboard allows you to specify which alerts gets displayed.
Quick Filters
There are two quick filter checkboxes at the bottom of the dashboard view.
Figure 22
Dashboard - Quick Filters
 
These checkboxes are used to quickly enable or disable display of all adaptive or all static alerts within the main dashboard view.
Advanced Filters
Advanced filters provide a more flexible way to configure which alerts get displayed in the dashboard. Click the Advanced Filters button in the lower right corner of the dashboard to the configuration dialog.

Detailed Adaptive Alert View
The detailed view for an adaptive alert shows the results of the analyses that triggered the alert. The detailed view is displayed by clicking on the View Summary link of an alert in the main dashboard window.
Figure 24
Problem Management Dashboardâ€”Detailed View
 
Information presented in the detailed view is organized into four sections:
The outlier pane, located at the top of the detailed view, lists the number and duration of each outlier for this alert.
The group table pane, located directly below the outlier pane, displays a table for the group and metrics involved in the adaptive alert. This table supports all the drill-downs and controls available in a normal table. This simplifies the troubleshooting process. For instance, if the adaptive alert identifies an overloaded server, the table pane can be used to drill down to determine which connected clients are responsible for a surge of incoming connections requests.
The help pane, located at the bottom left of the detailed view, describes the adaptive alert and the metrics and correlations used in alert identification. The pane also provides helpful information for interpreting the data seen in the chart pane.
The chart pane, located at the bottom right of the detailed view, shows a time series chart for each metric involved in the alert analysis. The moving average for the metric is displayed as a black line. The adaptive threshold is displayed as a red line. Outlier periods are colored in red in the chart pane and match the list in the outlier pane. Individual outliers can be changed to the color green by clicking on Highlight Outlier in the outlier pane. This makes it easier to examine a specific event if many outliers are displayed in the same chart. Finally, time periods that do not have sufficient data (e.g., do not satisfy the minCount analysis parameter) are colored in grey.
The chart pane also provides the same controls available in other time series charts, allowing the user to change chart options. The metric, granularity and trash controls have been disabled since there is no need to perform these operations in the detailed view.
