Creating Insight Modules
An Insight Module is a set of Visual and Analysis Components that may be saved as a unit and placed in an Insight. An Insight Module is a component that is itself an Insight, but with two limitations. An Insight Module can only have one Layer and it must include a special Interaction called an Insight Module Interaction.
Insight Modules are helpful when you are continually using the same bundled sets of Elements. For an exercise in building an Insight Module see Using Insight Modules with Tabbed Containers.
Insight Module Interaction
When an Insight Module is placed in an Insight, it needs a means of communicating with the other Elements in the Insight. The Insight Module communicates with the other Elements in the Insight by using a special Interaction called an Insight Module Interaction.
An Insight Module Interaction is always between an External Component and a Component within the Insight Module. Insight Modules Interactions are added through the Component Interaction Dialog and is called “Insight Module Interaction.” Depending on whether the information flows from the Insight (Producer) or to the Insight (Consumer), either the Producer or the Consumer must be the External Component.
Procedure 1
Adding Insight Module Interactions using the Actual View
1.
Click Add Interaction.
The Components Interaction dialog opens.
2.
Change the Type to Insight Module.
3.
From the Interaction box, select Interval.
4.
From the Producer box, select External Component.
5.
From the Consumer box, select Chart 1.
6.
Click Apply.
7.
Click OK.
8.
Click Save.
End of Procedure 1

Saving an Insight Module
Insight Modules need to be saved in a directory in order to be saved into an Insight. The Insight Module directory may be defined in Tools > Builder Settings > Miscellaneous.
Figure 34
Insight Module Directory
 
When you create a new Insight Module, immediately open the Insight Properties dialog and give the module a unique name. This name is the display name for the module whenever it appears in the Insight Development Kit. To do this:
1)
Select File > New.
The Layout Grid and Outline Area are ready to create a new Insight.
2)
Open File >Properties.
The Insight Properties window opens.
3)
Enter the name of the Insight Module in the Insight Name dialog.
The default is Custom Insight.
Create the Insight Module as usual.
Figure 35
Insight Properties Window
 
Once created, Insight Modules are saved in the same way as standard Insights - from the File menu, select Save or Save As. The Insight Module is saved as an .xml file, but to distinguish from Insights, it has a .imc extension. You may need to manually type in the extension if it does not automatically show in the Save or Save As dialogs.
Inserting an Insight Module into an Insight
Once an Insight Module has been saved in the Insight Module directory, it can then be inserted into an Insight.
An Insight Module is a Visual Component if it has at least one Visual Component assigned to a grid location. An Insight Module is considered an Analysis Component if it has no Visual Components, i.e, it is made up of Hidden Visual Components and Analysis Components.
There are two ways to add an Insight Module that is a visual component. The first way is as follows:
1)
Select Grid Edit mode and right-click an active pane.
2)
Select Insert Insight Module.
3)
Select the Insight Module you wish to insert.
Figure 36
Inserting Insight Module
 
Note that no matter how many panes it took to create the Insight Module, it appears as one pane once Inserted into the Insight.
The second way to insert an Insight Module is as follows:
1)
The Module icon needs to be in the toolbar in use drag and drop. If the Module button is not in the toolbar (see Insight Module icon), choose Tools > Builder Settings > Toolbars and make sure Modules is selected.
A Module button now appears on the main window.
Figure 37
Insight Module icon
 
2)
Drag and drop the Modules button like you would with any other component.
The Select Component window opens.
Figure 38
Add Visual Insight Module Select Component Window
 
3)
Open the Modules folder and select the desired Insight Module.
4)
Click OK.
To add an Insight Module that is an Analysis:
1)
Check Tools > Builder Settings > Miscellaneous and make sure From Dialog is selected in Select Analysis.
2)
Click OK and return to the Insight Development Kit main window.
3)
Click the Add Analysis button.
The Select Component window opens.

4)
Open the Modules folder and select the desired Insight Module.
5)
Click OK.
To finish, set up using Insight Module Interaction.
Editing an Existing Insight Module
Once an Insight Module has been saved, it can be edited at any time from within the Insight Development Kit. Open the Insight Module, edit as necessary and save.
You can edit an Insight Module once it has been inserted into an Insight. These changes are only saved within the Insight - the original .imc file is not affected. The reverse is also true. If you edit the original .imc file there is no change to the Insight.
Edit the Insight Module as you would any Visual Component and Analysis Components, by selecting and right-clicking the Insight Module in the Element pane and using Edit Visual Component or Configure Analysis, respectively. Once selected, a version of the Insight Development Kit opens that has a subset of the full Insight Development Kit.
Not all menu items in the File menu are available, nor is the layering functionality. To save your changes click OK.
You can also use the Insight Module as a template for an Insight. To do this, open the Insight Module and remove all the Insight Module Interactions and save as a standard Insight .xml file. You are not allowed to save an Insight Module as an Insight unless the Insight Module Interactions are removed.
