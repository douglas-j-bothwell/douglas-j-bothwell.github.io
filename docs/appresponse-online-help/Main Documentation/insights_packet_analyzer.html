Packet Analyzer Insight
You can use the Packet Analyzer insight to view protocol decodes of captured packets from within the console. This insight is useful for troubleshooting network problems, examining security issues, debugging protocol implementations, and other workflows that require detailed knowledge of the protocol information embedded in individual packets.
Figure 1
Protocol Decodes in Packet Analyzer Insight
 
Prerequisites
To use this insight effectively, you should have a basic working knowledge of the following topics:
Analyzing protocol decodes using tools such as Wireshark.
Specifying packet filters using the Berkeley Packet Filter (BPF) filter format. For more information, search the internet using the string “bpf filter syntax”.
• Wireshark display filters. For more information, see http://wiki.wireshark.org/DisplayFilters.
Workflow Description
The following steps outline the recommended workflow.
Procedure 1
Packet Analyzer Insight: Recommended Workflow
1.
Open the Packet Analyzer insight from either of the following:
• 
Insights menu (Insights > Packet Analysis > Packet Analyzer)
• 
Packet Download Manager (Tools > Packet Download Manager)
The Packet Download Manager method is recommended because you can identify and filter packets of interest before opening the insight. The following steps outline the workflow:
1.1.
Open the Packet Download Manager.
1.2.
Edit the preview to focus on the packets that you want to analyze.
Note—You must make sure that the “Display Individual IP conversations in preview result” checkbox is selected.
1.3.
Click in the desired tab to preview the packets (Tier Pair Circle, Tier Pair Table, or Tier Table).
1.4.
Zoom in on a time window by dragging horizontally in the throughput chart, if desired.
1.5.
Select the tier or tier pair whose packets you want to analyze.
Note—Make sure that you select the time window and tier/tier pair in the correct order: zoom in on the throughput chart (first), then select the tier/tier pair (second). Otherwise the View Decode button will be unavailable.
1.6.
Click View Decode.
Figure 2
Selecting Packets for Analysis in the Packet Decode Manager
 
2.
In the Filters tabbed page, specify the set of packets to include for analysis in the insight.
This step is required if you opened the insight from the Insights menu and optional if you opened from the Packet Download Manager. You can specify the following in this page:
• 
Time window (yyyy-mm-dd hh:mm:ss) —If you change the time window, you must click Update to apply the change.
• 
IP(s) for the packets of interest. You can specify any of the following:
IPv4 or IPv6 address (example: x.x.x.x)
IP conversation (two IPs separated by a vertical bar, for example: x.x.x.x|y.y.y.y)
IP range (two IPs separated by a dash, for example: x.x.x.x-y.y.y.y)
Note: Both IPv4 and IPv6 addresses are supported.
• 
Monitoring Interface Group (optional)
• 
VLAN (optional)
• 
BPF — You can specify a packet filter in BPF (Berkeley Packet Filter) format. Note the following:
• 
If this field does not contain a correct BPF expression, no filter will be applied.
• 
BPF expressions are case sensitive. tcp or udp is a valid expression, but TCP OR UDP is not.
Figure 3
“Filters” Tabbed Page
 
3.
If you edited any of the packet filters, click View Packets.
--> The insight switches to the Packets page and displays summaries for all packets that match all the specified filters.
4.
The Packets tab to view and analyze the packets of interest. This page has the following elements:
• 
Wireshark Display Filter - Specify a Wireshark display filter to show only packets of interest in the Packet Summaries table. For information about the display filter format, see http://wiki.wireshark.org/DisplayFilters.
Note—If the display filter includes quotes, you must use single quotes (sip.To contains ‘a1762’) rather than double quotes (sip.To contains "a1762").
After you specify the filter, you must press Enter to apply it.
• 
Go to Page Containing - This filter is useful for finding packets that contain ASCII text of interest—for example, HTTP packets that contain a specific string (either in the HTML content or the HTTP header data). When you enter a string and press Enter, the insight searches the Packet Summaries table and highlights packets that include the search string. Note the following:
• 
String searches are case-sensitive.
• 
After you specify the search string, you must press Enter to run the search.
• 
Packet Summaries table - This table shows all packets that meet all filter criteria specified in both the Filters and the Packets tabbed pages. Select a packet to view the protocol decodes, header summary, and data for that packet.
This table shows the following columns by default:
• 
Frame
• 
Time
• 
Source/Destination IPs
• 
Decode — Highest decode level for the individual packet
• 
Decode Summary
You can edit the columns to show and hide in this table: Right-click in the table and choose Show Toolbar; then click the Toolbar Settings button that appears and select Edit Columns; then click the click the Edit Columns toolbar button. You can show the following additional columns:
• 
Delta — Time difference between the current frame and the first frame shown in the insight
• 
Size - Size of the packet, in bytes
• 
IP ID - IP identification field
• 
Source MAC Address/Destination MAC Address
• 
Protocol Layer table - Select the protocol layer for which you want to view decodest
• 
Headers tabbed page - Shows detailed decodes for the selected protocol layer of the packet.
• 
Data tabbed page - View the packet contents in hexadecimal and ASCII format
5.
To download packets to your computer from the Packet Analyzer insight, click on the Download Packets button at the bottom of the Packet Analyzer window (Figure 4).
Note: The packet download feature ignores the Wireshark Display Filter. If you want to view the conversations that will be downloaded, clear the contents of the Wireshark Display Filter.

Using the Packet Analyzer Insight on a Director
You can run the packet analyzer insight on a Director and view information for one or more connected appliances. When using this insight on a Director, the Packet Summaries table includes an extra column that lists the appliance.
Filtering options on the Director are the same as they are on an appliance except that on a Director you must also specify the appliance when specifying IP addresses, IP conversations, or IP ranges. Use the following syntax to specify an appliance in the packet filter:
IP address (appliancename:x.x.x.x)
IP conversation (appliancename:x.x.x.x|y.y.y.y)
IP range (appliancename:x.x.x.x-y.y.y.y)
