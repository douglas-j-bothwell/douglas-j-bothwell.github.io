Insight Development Kit Menu Reference
This section furnishes examples and further information about some of the functions explained in the first part of this chapter, and information and examples about any functions not previously discussed.
Unlike many applications, the menu bar in the Insight Development Kit UI does not duplicate functions offered by the various toolbars. Instead, with the exception of the Edit menu, it provides what may be called “housekeeping” functions associated with the overall function of the Insight Development Kit. These are organized under File Menu, Edit Menu, View Menu, and Tools Menu.
File Menu
The File menu offers the usual file functions of New, Open, Save, Save As, and Exit, as well as a pick list of recently-opened insights. In addition, it offers three other menu picks: Properties, Open Examples, Snapshot Preview, and Preview Examples.
Snapshot Preview allows you to save a .png file of the Insight Development Kit screen to your computer (if you want to save only the Insight, click Snapshot in the Test/Layout view. Open Examples lets you load example Insights into the Insight Development Kit so you can investigate how various functions, such as Layers, are accomplished. Preview Examples merely shows you these example Insights as though you were using the Test/Layout function.
Properties…
Select Properties to display the Insight Properties dialog, which lets you specify various Insight properties.
Figure 40
Insight Properties dialog
 
Insight Name - enter the name for the Insight. This is the name that the Insight is listed under when uploaded to an Update Center. See Update Center.
Insight Description - enter a description of the Insight. This is the description displayed in an Update Center. See Update Center.
Insight Author - enter the name of the author.
Insight Category - this is the Category the Insight is listed under in the Desktop Console menu. You may enter any name here and the new Category is created. The new categories can be seen in View >Insight. Enter Audit, Baseline, Troubleshoot, Secure, or Optimize, if you want the Insight to also show up in one of the five Quick Launch Insights invoked by the Insight icons in the Desktop Console toolbar.
Insight Version - enter a version number. Default is 1.0.0. Generally you only change this when a new version is created to upload to an Update Center.
Console Version - this defaults to the version number for the console the Insight was created on. This specifies the minimum version of the Console that is compatible with running this Insight. It allows others to continue to maintain earlier versions of the Insight.
Client Version - this is an internal UI version number for Riverbed use.
Appliance Version - this indicates the appliance version. You can’t run an Insight created on an appliance newer than the one the user’s Desktop Console was downloaded from.
Appliance Compatibility - choose Both, Appliance, or Director.
Hidden - choose Yes or No. A hidden Insight cannot be directly invoked by a user, only via linking from another, visible Insight.
Global Window - choose Yes or No. A Global Window is available in all projects open in the Desktop Console.
Modified - displays the last time the Insight was modified.
Background Color - you may specify the background color as RGB values or click Select Color… to display the Set Background Color dialog.

In the Edit drop-down menu you may select Cut, Copy, and Paste functions, which work only on text, not Components. You may select Layout Options… to display the Layout Options dialog box.

Layer Settings
The Layer Settings section of the Layout Options dialog lets you specify three different parameters for each Layer in the Insight, selected from the Select the Layer to Layout drop-down menu.
The following figure shows the meaning of the following three settings.
Figure 42
Component Spacing and Divider Parameters
 
Component Spacing - governs space between each Component and the edge of the Insight window or other Components in the vertical dimension.
Divider Inset - along with Divider Size, governs the horizontal space between Components.
Divider Size - in the present Insight interface the divider appears as the line between panes plus some space to either side. Setting Divider Size to zero makes it impossible to resize panes by click-dragging on the divider.
Component Settings
The Component Settings section of the Layout Options dialog lets you specify many different layout parameters for each Visual Component of the Insight, selected from the Component to Layout drop-down menu. This dialog, when the Advanced box is checked, is basically an expansion of the options available in the Layout pane of the Outline Area of the Insight Development Kit UI, which control the functioning of the Java Layout engine. See Working with the Layout Pane for a brief overview about layout parameters. The Advanced controls are best understood by experimentation and are not documented at this time.
View Menu
The View menu enables you to control the appearance and function of various parts of the Insight Development Kit UI. You may also invoke the Preview Insight Content window here, as described in Testing the Insight.
Toolbars
The Toolbars menu item calls up a nested checkbox menu that controls the appearance of the Toolbar area of the Insight Development Kit UI. It lets you specify whether or not the Analysis Components, Interactions, Layer Controls, and Layout Grid Controls sub-Toolbars are displayed, as shown in Figure 8
You may not deselect the Visual Components Toolbar all at once, but you may use the nested Visual Components checkbox menu to select or deselect any or all of the sub-toolbars for various types of Visual Components: Charts, Tables, Topologies, Selectors, Views, Text, Launching, Utilities, and Layers.
This menu is duplicated by the Insight Development Kit Settings dialog, see Builder Settings.
Outline
The Outline menu calls up a checkbox and radio button menu that controls the appearance and function of the Outline Area of the Insight Development Kit UI. Some of these functions are also available from the Tools > Builder Settings > Outline tab.
Show—Uncheck this to turn off the entire Outline Area (default=checked). This leaves more room for the Layout Grid, which may be useful for large Insights. You may not display the Outline Area without the Element pane.
Description Pane—if Show is checked, you may uncheck this to turn off the Description Pane (default is on). This leaves more room for exploring the structure of the Insight using the Element pane.
Layout Pane—if Show is checked, you may uncheck this to display the Layout pane (default is on). This leaves more room for exploring the structure of the Insight using the Element pane.
Visual Components Sorted by Name—select this radio button for the Outline pane to display the Visual Components in the Insight sorted alphabetically.
Visual Component Sorted by Grid Location—select this radio button for the Outline Pane to display the Visual Components in the Insight sorted by their grid location: row by column; e.g., R1C1, R1C2, R1C3...R2C1, R2C2, R2C3, etc.
The next three menu items are very useful for checking to see that a given Component produces or consumes all necessary Interactions, or that a given Interaction goes to or comes from all necessary Components. They can also be selected from the Element Pane context menu, as described in Interaction Sorting.

Layout Grid
The Layout Grid menu item calls up a radio button menu that controls the operational view and sizing function of the Layout Grid, as described Understanding the Layout Grid. There are three views:
Workflow View
Actual View
Schematic View
Size Changes Affect One Row - when you click-drag a divider, you are actually changing the size of the row above the divider. In this mode, all other rows maintain their selected size.
Size Changes Affect All Rows - in this mode, all rows are affected by the change, the row below being most effected.
For more information, see Size Change Mode.
Instructions
Check this to enable inspection of Component descriptions in the Description Pane by clicking their toolbar icons.
Show Insight Development Kit Tips
Select this to view best practices to follow when creating Insights. These are the same tips that are shown when opening the Insight Development Kit.
Test/Layout
Selecting Test/Layout in the View menu is the same as selecting it from the Layout Grid.
Tools Menu
The Tools menu lets you set defaults for Insight Development Kit operation, set Component sizes in terms of other Components, and work with the Insight Catalog, which is necessary for uploading Insights to an Insight Update Center.
Builder Settings
To set defaults for Insight Development Kit operations, click on Tools > Builder Settings to display the Insight Development Kit Settings dialog.
Figure 46
Insight Development Kit Settings Dialog Box
 
The Insight Development Kit Settings dialog box has the following tabs:
Toolbars - set defaults for the parameters chosen using the View > Toolbars menu. See Toolbars.
Outline - set defaults for the parameters chosen using the View > Outline menu. See Outline.
Layout Grid - set defaults for the parameters chosen using the View > Layout Grid menu. See Layout Grid.
Interaction Definitions - it is possible to change the formatting of the Interactions. Network Physics does not recommend this operation.
Miscellaneous - set the default for the Instructions checkbox in the View menu. See Instructions.
Set Component Sizes
Click Set Component Sizes to display the Component Size dialog.
Figure 47
Component Size Dialog Box
 
The Component Size dialog allows you to resize a Component based on the dimensions of other Components. It is especially useful for making Component sizes line up in different Layers.
Component to Resize - select the Component to resize. Only Components in the currently selected Layer are displayed as choices.
Layer To Get Size From - select the Layer from which you wish to select Components whose dimensions will be used for resizing.
Components to Use for Size - select the Component(s) whose dimensions will be used to resize the selected Component. The size is determined by the combined size of all Components selected in this step.
Click Apply to make the change and keep working. Click OK to accept the last change entered and close the dialog, or Cancel to close the dialog without saving.
Catalog Menu Items
These items appear on the Tools menu. They let you manage the Insight catalog and other resources related to maintaining Insights on an update server. See Update Center for details on how you can use these commands to set up your own update server to deliver custom Insights.
Figure 48
Catalog Menu Items in Tools Menu
 
Update Catalog with Current Insight
Updates the Insight Catalog with the latest saved version of the Insight currently open in the Insight Development Kit. The Insight Description, Author, and Category must be entered in the Insight Properties… dialog before it can be synced with the Catalog. If no catalog has yet been created, it is created and the Insight synced with it.
Update Catalog
Updates the Catalog with all Insights in the current directory. Select the Catalog you wish to update from. If no catalog has yet been created, it is created and updated.
Create Catalog
Creates a Catalog in the directory specified by the File Name field. All Insights in this directory will be added to the Catalog. The default name is catalog.xml. (You cannot enter a different file name in the File Name field.)
Edit Catalog
Enables you to use the Catalog Editor to edit the properties of each Insight in the Catalog and of the Catalog itself.
Figure 49
Catalog Editor
 
The following explains the fields and controls in the Catalog Editor (not all fields are mandatory):
Insight Name - change the Insight Name if you wish. This is the name that is displayed in the Name field of the Update Center window. (Not mandatory.)
Insight Author - change the Insight Author if you wish. This name is displayed in the right-hand text pane of the Update Center window, between the header and the footer. (Not mandatory.)
Insight Category - change the Insight Category if you wish. For more information on Insight categories. (Not mandatory.)
Insight Description - Enter or change the Insight Description if you wish. This text is displayed in the right-hand text pane of the Update Center window, between the header and the footer. (Not mandatory.)
Insight Version - change the Insight version if you wish. (Not mandatory.)
Console Version - the minimum console version required to run the Insight.
Location - leave the filename here unless you want it uploaded to a different directory on the web server.
Header - enter a header, which may include HTML, that is displayed at the top of the right-hand text pane of the Update Center window.
Footer - enter a footer, which may include HTML, that is displayed at the bottom of the right-hand text pane of the Update Center window.
Resources - enter the filename (and directory path if they are not in the same directory as the Insight) of all resources used by the Insight: image files (JPEG, GIF, or PNG), audio files (WAV), or HTML pages (HTM or HTML). These are automatically associated with the Insight so that anyone downloading it receives them in the proper directory for operation.
New - click here to create a new Insight; i.e., open a new file in the Insight Development Kit in the same directory as the open Catalog.
Edit - click here to open the selected Insight in the Insight Development Kit.
Delete - click here to remove the selected Insight from the Catalog. It does not delete the Insight file.
Switch Catalog - click here to work with a different Catalog via the Open Catalog File dialog.
Archive Insights to Zip
Click this to archive all the Insights in your local Insight directory along with their resources and the file structure for those resources to a zip file. Select the Catalog file to zip using the Open Catalog File dialog and click OK to create the zip file. You may use this to send Insights to another person for installation in their local Insights directory.
Upload Catalog...
Click this to upload a Catalog and the associated Insights and resource files to an update server.
Figure 50
Upload Catalog Dialog Box
 
Directory—leave this unchanged to upload the Catalog in which the currently open Insight exists. You may change it to upload any catalog on your machine.
Hostname—enter the hostname of the computer hosting the Insight files.
User Name—enter your username.
Password—enter your password.
Protocol—select FTP or SSH (default).
Port—enter the port.
Directory—enter the directory on the web server where the Update Center should look for Insights and related resources.
Update—select Replace Catalog to entirely replace the catalog with the new one. Select Merge Catalog to add new Insights, or newer versions of existing Insights (indexed by name), to the Catalog.
