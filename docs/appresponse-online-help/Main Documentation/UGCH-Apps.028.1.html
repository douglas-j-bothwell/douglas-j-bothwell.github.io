Applications
An appliance collects metrics for the applications it observes in the monitored traffic. An appliance does not necessarily collect metrics for all applications it observes, however. The following factors determine which application metrics are (and are not) collected on an appliance:
Topping limits
Because its disk space is finite, the appliance collects metrics only for the applications with the highest throughput. The appliance has a set of hard-coded application definitions and can also auto-detect other applications. These applications have topping limits imposed. (For more information, see How AppResponse Selects Top Elements.)
Defined Applications
The Desktop Console includes a Defined Application Manager that enables you to define the applications of interest in your networkâ€”those applications for which you want to collect metrics at all times, regardless of topping limits.
When you open an Applications table, you can see three different types of application:
Defined Applications
A user-configurable set of preferred applications. These applications are always monitored and no topping limits are imposed. The appliance has a set of default Defined Applications; you can edit these defaults and create new ones in the Defined Application Manager.
By default, Defined Application names are all-uppercase: AIM-TCP, FTP-CONTROL, and so on.
 
Recognized Applications
Applications that are predefined, non-configurable, and monitored automatically (with topping limits applied). You can create a Defined Application from a recognized application; this ensures that the application is always monitored, regardless of topping limits.
Recognized Application names are all-lowercase: alternate-https, ftp-ctrl, ftp-data, and so on.
 
Auto-discovered applications
The appliance also auto-discovers applications that do not correspond to any Defined or Recognized Application definition. Each unique port-number/protocol combination corresponds to a unique auto-detected application. Topping limits are applied.
Auto-discovered application names consist of the port number and protocol: 45634-TCP, 870-UDP, and so on. The appliance identifies many applications based on IANA protocols and ports. If the application matches an IANA protocol/port, the application name appears in the Information metric; otherwise, this metric reads UNCLASSIFIED.
