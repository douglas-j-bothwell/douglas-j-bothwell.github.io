Inserting Graphics into HTML
Images can be embedded in all Components that allow HTML or text display, including:
Label Component
Static Text Component
Tab Container
Link and Script Launcher
Insight Launcher
To embed images, include an HTML “img” tag in the edit field (AppResponse interprets this tag within plain text as well as within HTML). You can enter the tag manually (with or without a variable for part of the URL) or with the help of the Browse Images Dialog Box.
Image Tags and Variables
You can also, of course, put a standard URL in the base property or images can be displayed from the Internet (depending on network proxy settings) by using a fully qualified URL in the <img> tag, for example:
<img src="http://www.acme.com/images/test.gif">
Several URL variables using the same format as other Insight Development Kit variables are available to assist this, either by setting the base property in the HTML head for those Components that allow HTML display or by using the variable as part of a URL, as must be done in the case of the Tab Container.
$localUrl or ${localUrl}: the URL for the local Insight directory. This is used for adding custom images to an Insight. See Using Custom Images for instructions on how to use this variable.
$remoteUrl or ${remoteUrl}: the URL for the remote images directory on the connected Appliance or Director. This is used for adding custom images to an Insight. See Using Custom Images for instructions on how to use this variable.
$npImageUrl or ${npImageUrl}: the URL for the image directory within the jar. This directory contains graphics for UI elements, charts, tables, managers, topologies, Insight icons, Appliance images, and so on.
<html>
   <head></head>
      <body>
         <table>
            <tr valign="top"><td width="80" align="left">
               <img align="top"
                src="${npImageUrl}/products/ACELive_4100.gif">
         </td></tr>
Browse Images Dialog Box
The Browse Images dialog box simplifies the job of adding a graphic to plain or HTML text. You can open this dialog box when editing a component that allows embedded images by doing one of the following:
click the Add Image… button
select “Image” from the Tags pull-down and click Add
Use this procedure to add an image:
Procedure 5
Adding an Image with the Browse Images Dialog Box
1.
Select a directory from the Location pull-down.
2.
(Optional) If you have selected the Remote Image directory, you can click the Upload Resources button to upload an image for use.
3.
Select an image from the Resource pull-down, which lists all images available in the selected location. The selected image is shown in the right-hand pane.
4.
Click OK to insert an HTML link for the selected image.
End of Procedure 5

Using Custom Images
You can embed custom images when building Insights by using the $localUrl and $remoteUrl variables. These variables can also be used to associate audio files (WAV) with Insights using the Toolbar Audio Clip Control and HMTL pages (HTM and HTML) using the Toolbar Help Pages Control.
To use custom images, upload the images to the remote images directory on your appliance by using the Upload Images, HTML, and Audio Files menu item and then refer to them with the $remoteUrl variable.
You can also use custom images as required to do so in Insight Development Kit versions prior to v6.0:
1)
Use the $localUrl variable when you are building and testing the Insight and want access to images or other resources stored in the local Insight directory on the computer running the Desktop Console and Insight Development Kit. You may append a directory path to this variable if you have your images stored in a subdirectory of the local Insight directory.
2)
The $localUrl variable, along with any appended directory path, must be replaced by the $remoteUrl variable (without any directory path appended) before uploading the Insight to an Update Center so that this image or other resource is available to any user downloading the Insight. This can be done manually in the Insight Development Kit, or, if desired, using grep or a text editor on the Insight file itself.
