Global Search
You can quickly find the major Desktop Console windows for grouping and displaying traffic—such as Reports, Defined Applications, Business Groups, and Insights—whose properties include the specified search string. Each “hit” includes links to additional information such as charts, tables, or Manager windows.
You can search using the Search pane (top-right corner of the Desktop Console window) or the full Search dialog box (Tools> Search). The full dialog box includes search history, advanced search options and a link for additional search tips.
 
Searching for Traffic Data
The Search dialog box includes a Search for Traffic Data checkbox. This option is useful when you want to identify quickly any matching groups that include traffic data within the current time window. This option is available on both appliances and directors.
When you perform a search with this option selected, AppResponse does the following:
1)
Searches the configuration data on the appliance, or within the director domain, for all matching group definitions: Business Groups, Applications, IP Conversations, and so on.
2)
Takes the set of matching groups from step 1 and searches for groups that include traffic data for the current project time window.
3)
Shows all groups that meet the search criteria in
–
Step 1 (group definition matches the search string)—Search results include a Definition link.
–
Step 2 (contains traffic data for the current time window)—Search results include a Time Series Chart and a Favorites Table link. The Traffic Data tabbed page shows all matching groups that include traffic.
(When the “Search for Traffic Data” option is not selected, the Traffic Data tabbed page is hidden and the search results include a Time Series Chart and Favorites Table link for every group, even if the group includes no traffic.)

To set this option, click Advanced at the top of the Search window; then select Search for Traffic Data. To specify whether this option is selected by default, set the Search for Traffic Data by Default checkbox (Tools > Preferences > Search > Search for Traffic Data by Default).
Supported Search Terms
The global search in this release supports the following:
Search on multiple strings:
one two three => all entries with one | two | three
Search on strings to exclude:
one two -three => all entries with (one | two) & !three
Search on exact phrases:
“one two three” => all entries with the string “one two three”
Search on IP addresses and ranges (using subnets or ranges) as well as ports and port ranges:
192.168.14.2 => traffic for this IP (in the Related Data category) and all Business Groups with this IP
192.168.14.36-192.168.14.255 => all Business Groups with IPs in this range
192.168.14.2/16 => traffic for this IP (192.168.14.2) and all Business Groups that include IP addresses in this range (192.168.0.0-192.168.255.255)
8080 => all applications that use this port
8080-8100 => all applications that use ports within this range
Note—When you search on a range of IP addresses or port numbers, do not include any spaces in the search string (enter 1.1.1.1-2.2.2.2 and not 1.1.1.1  -2.2.2.2).
Context-sensitive searches in tables—Right-click on the item you want to search on (such as an IP address or Business Group) and choose Search.
Search using a fully-qualified host or domain name. In addition to searching for the specified search string, AppResponse performs a DNS lookup and then searches on the resolved individual IP address or multiple IP addresses.
Search for web page families
Note the following:
–
To search for one or more page families, you must enter a search that is at least four characters long and includes either a period character (‘.’) or the string http.
–
You can use asterisks as wildcards to search for web page families only.
–
The search results for web page families are additionally sorted by the times when individual pages were downloaded (most recent downloads first).
Search results include context-sensitive links for viewing results in external windows such as Time Series Charts, Favorites Tables (for viewing associated traffic), the Business Group Manager (for viewing BG definitions), and so on.
To view traffic for multiple entries in the same window, use the Ctrl-click operation in Windows or the equivalent operation on another platform. For example, you can view traffic for multiple addresses in the same Time Series Chart:
–
Search on an IP address range.
–
In the Search Results list, click on the “Time Series Chart” for one entry. A chart opens with the traffic observed for that address within the current project time.
–
Ctrl-click on the “Time Series Chart” links for other IPs to add these addresses to the same chart.
To view traffic in the same external window, you must Ctrl-click on links of the same type (such as “Time Series Chart” or “Favorites Table”).
Note: For more information about performing effective searches, choose Tools > Search and click the “Search Tips” link.
Default Color Manager
It is useful to be able to assign default colors to specific groups and applications to maintain consistency across windows and projects and over time. The Default Color Manager acts as a system default and are stored on the appliance (whereas the Group Color Manager is stored in the networkphysics.ini file and applies only to the Desktop Console of that user). Default colors can be applied to Business Groups or Defined Applications.
Setting color preferences for groups only applies to the management console. You can also set color preferences for reports.
Figure 23
Default Color Manager
 
Procedure 5
Creating a New Default Color
1.
Click New.
2.
From the Group Type drop-down list select either Defined Application or Business Group.
3.
From the Group drop-down list, select the correct Defined Application/Business Group.
4.
Click the color bar next to Color.
The Select Color dialog box opens.
 
5.
Select a color by selecting a swatch from the Swatches tab, or entering information on the HSB or RBB tabs.
6.
Click OK.
(Note that the Reset button resets the dialog to the original color.)
The Select Color dialog box closes.
7.
On the Default Color Manager window, click Apply or OK.
End of Procedure 5
 
 
Procedure 6
Deleting a Default Color
1.
Select an existing color in the information table.
2.
Click Delete.
3.
Click Cancel to close the Default Color Manager.
End of Procedure 6
