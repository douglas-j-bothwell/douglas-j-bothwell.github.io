Packet Analyzer: Viewing SteelCentral AppResponse Data
After you download, install, and run the Packet Analyzer application, you need to create a connection to the SteelCentral AppResponse appliance.
This section discusses the following:
Connecting Packet Analyzer to an AppResponse Appliance
Analyzing AppResponse Packets in Packet Analyzer (Retrospective Analysis)
Connecting Packet Analyzer to an AppResponse Appliance
Procedure 1
To connect Packet Analyzer to NetShark Module on an AppResponse appliance:
1.
Open the Packet Analyzer application.
2.
Click on the Probes tab and choose Add Probe. The Connect to Probe dialog box appears.
Figure 1
“Connect to Probe” Dialog Box
 
3.
In the Connect to Probe dialog box, enter the IP address, HTTPS port (in the “Auto” field, top right), username/password, and proxy information for the appliance.
Note: By default, an SteelCentral AppResponse appliance listens on port 8443 for HTTPS data requests. You only need to set this field if the appliance uses a non-default port to receive HTTPS data requests.
Note: You might see a Certificate Verification Warning and a prompt to accept the certificate.
4.
When Packet Analyzer verifies that it is able to connect to the appliance, a “Probe running” message appears at the bottom of the dialog box. When this message appears, click OK.
Figure 2
“Probe Running” Indicator (Packet Analyzer)
 
5.
A new NetShark Module probe appears in the Devices treeview (top left).
Figure 3
ARX Probe in “Devices” Treeview (Packet Analyzer)
 
Note the following:
• 
The Devices treeview shows multiple arx live interfaces under the ARX probe:
• 
An aggregate interface (arx (Monitoring Interfaces Aggregate)), which includes data for all packets on all connected interfaces
• 
One or more individual interfaces: mifg1_name (Monitoring Interface Group 1), mifg2_name (Monitoring Interface Group 2), etc.
The interfaces that are active depend on whether the appliance has Monitoring Interface Groups defined (Administration > System > Interface Groups > Collect data for Monitoring Interface Groups).
If no Monitoring Interface Groups are defined, the Aggregate interface is active and the individual live interfaces are inactive.
If one or more groups are defined, each corresponding Monitoring Interface Group job is active (if the MIfG port is connected to at least one physical port) and the aggregate job is inactive.
End of Procedure 1
 
 
 
Analyzing AppResponse Packets in Packet Analyzer (Retrospective Analysis)
After you set up RPM Integration on an appliance (as described in Procedure 1), you can view and analyze the packet data in Packet Analyzer. This section discusses the following:
Important Notes about Retrospective Analysis
Selecting Packets in Packet Analyzer for Retrospective Analysis
Selecting Packets in AppResponse for Retrospective Analysis in SteelCentral Packet Analyzer
Exporting AppResponse Packets from the SteelCentral NetProfiler Interface to Packet Analyzer
Important Notes about Retrospective Analysis
Note the following:
This workflow requires that packet storage be enabled on the appliance during the time window of interest. Otherwise the Java console shows a message saying that the agent does not have the requested preview data.
Packet storage is configured in the Administration > System > Capture web page.
Packet Analyzer supports export of packet data to SteelCentral Transaction Analyzer (release 16.5.T PL1 and higher). If you have a license for Transaction Analyzer, you can send packet data from Packet Analyzer directly to Transaction Analyzer. You can quickly open large packet files for offline, multi-tier transaction analysis and performance prediction. This supports a streamlined troubleshooting workflow across three products (SteelCentral AppResponse > Packet Analyzer > Transaction Analyzer).
For more information, see the Packet Analyzer Reference Manual > Overview > Packet Analyzer - Feature Summary > Transaction Analyzer Integration.
Selecting Packets in Packet Analyzer for Retrospective Analysis
After you set up a probe to the NetShark Module on the AppResponse appliance, you can analyze packets using the same tools and workflows as you would when analyzing packets on a NetShark appliance. All the standard Packet_Analyzer functionality is supported: Trace Clips, Views, Filters, Drill Downs, and so on.
Note: This workflow requires AppTransaction Xpert Decode Module (or higher) of Advanced ACE Capabilities.
You can download software and documentation for Packet Analyzer from Riverbed Support (https://support.riverbed.com).
Selecting Packets in AppResponse for Retrospective Analysis in SteelCentral Packet Analyzer
This workflow involves the following general steps:
1)
Select the packets you want to analyze in an AppResponse alert, insight, table, and chart.
2)
Save the packets to a trace clip or PCAP file on the appliance, or a trace file on your local host.
3)
Analyze the trace clip, PCAP file, or trace file in Packet Analyzer.
This workflow is supported in all versions of RPM Integration.
Procedure 2
Selecting packets in AppResponse for retrospective analysis in Packet Analyzer:
1.
Open a Java console and connect to the AppResponse appliance.
2.
Java console: Drill down to the traffic you want to analyze in Packet Analyzer. You can analyze traffic from an insight, a table, or a chart.
Note: It is good practice to drill down to the traffic of interest, and to exclude as much irrelevant traffic as possible, from your Packet Analyzer analysis. The more traffic you try to analyze, the longer it takes for the NetShark Module and Packet Analyzer to process and display the data. It is especially important to
• 
Define as short a time window as possible.
• 
Include only IP conversations of interest in the traffic selection.
Figure 4
Drilling Down to Traffic for Analysis in Packet Analyzer
 
3.
Right-click on the traffic of interest and choose Analyze In > Packet Analyzer. The Packet Analyzer Preview window appears. This window has the same interface and workflow as the Packet Download Manager.
4.
Packet Analyzer Preview window: Edit the preview data further if desired.
The goal is to include all traffic of interest to your Packet Analyzer analysis, while at the same time excluding as much irrelevant traffic as possible. You can
• 
Select individual tiers or tier pairs (Tier Pair Circle/Tier Table, Tier Table)
• 
Zoom in on a time window (drag horizontally in time graph)
• 
Edit the time window and IPs displayed (click Edit Preview)
Figure 5
Packet Analyzer Preview
 
5.
When you finish editing the preview, click Analyze Selected.
The NetShark Module creates a new trace clip for the selected packets. The trace clip appears below the capture job in the Packet Analyzer “Files” pane (Figure 6).
(If RPM Integration v1 is installed, the NetShark Module creates a PCAP file rather than a trace clip, as described in “Note: Retrospective Analysis in RPM Integration v2 vs. v1” on page 79.)
You can now use the full range of visualization, analysis, and reporting functionality in Packet Analyzer on your AppResponse trace clip.
Figure 6
AppResponse Packet-Capture Files in Packet Analyzer
 
End of Procedure 2
 
 
Exporting AppResponse Packets from the SteelCentral NetProfiler Interface to Packet Analyzer
If NetProfiler uses AppResponse as a data source, you can now export AppResponse packets to Packet Analyzer directly from the NetProfiler interface. This feature requires the following:
SteelCentral NetProfiler version 10.6 or higher
Packet Analyzer version 10.6 or higher
A NetShark Module license on the AppResponse appliance
The AppResponse packet buffer must include packets for the selected traffic and time window.
Figure 7
NetProfiler: Sending Packets to Packet Analyzer for Analysis
 
The workflow is the same as exporting NetShark packets to Packet Analyzer:
1)
In NetProfiler, select the AppResponse traffic whose packets you want to analyze in Packet Analyzer. You can select a host, host pair, protocol, or flow.
2)
Right-click on the selected item and choose one of the following:
–
Packets for this [item] > Analyze in Packet Analyzer
Select the AppResponse appliance and capture job when prompted. A new Trace Clip is created on the appliance and appears in Packet Analyzer.
–
Packets for this [item] > Export to PCAP file > From NetShark
Select the AppResponse appliance, capture job, and PCAP options when prompted. SteelCentral NetProfiler then shows a link to download the file. (Depending on your browser and settings, the file might download automatically.)
