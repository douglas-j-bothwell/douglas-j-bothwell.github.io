Warning—Do Not Capture Sensitive Data over Unsecured Connections
WARNING: AppResponse does not change any packet capture data before storing it to disk. For this reason, Riverbed does not recommend capturing sensitive packet data over an unencrypted connection. If you cannot capture sensitive packet data over an encrypted connection, Riverbed recommends that you configure the appliance using one of the following methods:

     * Capture in "headers-only" mode, as described in Headers-Only Mode.

     * Set the Packet Size Limit to no higher than 128, as described in Configuration Options for High-Speed Captures.
Note—Avoid Multiple Simultaneous Packet Operations on the Appliance
When you want to do multiple packet operations in AppResponse, do the operations in as sequential, “one-at-a-time” manner as possible. In this context, a packet operation is any operation that searches, retrieves, or otherwise accesses “raw” packets (rather than Microflow Index data) stored on the AppResponse appliance.
Known Issue
AppResponse has a known issue in which performing more than two of the following actions concurrently can result in various errors when the appliance tries to retrieve packets:
–
Analyze In Cascade Pilot preview operations that cannot use Microflow Index data
–
Packet Download Manager preview/download
In Cascade Pilot (performing packet operations on appliances with RPM Integration):
–
Applying views to trace clips that lack Microflow Index data
–
Saving trace clips as files
–
Sending packets to AppTransaction or Wireshark
Symptoms
Error messages such as the following indicate that the appliance is having trouble processing concurrent packet tasks:
Unable to retrieve preview data. The server is busy, please try again! (y)
Error processing the view (A generic problem occurred while applying the view '[view name]'. (Cascade Pilot)
No packets captured! As no data was captured, closing the temporary capture file! (Wireshark)
Workaround
Try to execute packet operations as sequentially (one operation at a time) as possible. If you want to download packets using the Packet Download Manager and analyze packets in Cascade Pilot, for example, perform one operation and wait for all packets to download before you perform the next operation.
If the appliance gets into a state where no actions are being performed and no new actions can be performed, reboot the appliance.
