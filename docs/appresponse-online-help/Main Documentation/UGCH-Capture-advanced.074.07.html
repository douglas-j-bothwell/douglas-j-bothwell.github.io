Monitoring Interface Groups (MIFGs) and Packet Captures
A monitoring interface group is a logical grouping of one or more physical interfaces. AppResponse treats all physical interfaces in the same group as one interface. When multiple interfaces in the same group capture the same packet, AppResponse retains one copy and discards any duplicates. Monitoring interface groups are useful in the following situations:
You need to capture packets at the same location using multiple interfaces—for example, by capturing packets on two unidirectional ports on the same device.
You want to define a logical network location based on the areas that are monitored by one or more interfaces.
The MIFG/packet capture functionality enables you to
Define logical MIFGs for multiple interfaces that monitor the same network location.
Save the interface group—and, by extension, the network location—where each packet was captured.
Observe the flow of packets that were captured in different interface groups.
Preview, filter, and download captured packets by interface group in the Packet Download Manager.

Viewing Packet Capture Status on Monitoring Interfaces
This release includes a mifstat command that displays throughput levels, errors, and packet drops on one or more monitoring interfaces. This command is available on all appliance models that support High Speed Captures, except the 4100 (1G) appliance.
This command has the following command-line options:
-w     The reporting interval, in seconds. The default setting is 1 second.
-I     The monitoring interface (1, 2, 3, 4) on which to report.
If you do not include this option, mifstat reports on the aggregate traffic for all monitoring interfaces.
To exit mifstat, enter Ctrl-C.
In the following example, mifstat reports the packet capture status on monitoring interface #1 at five-second intervals:
my-arx-appliance.riverbed.com> mifstat -w 5 -I 1
input (Total) [Wed 12 Oct 14:25:36 2011]
packets errs bytes drops
148685 0 93210675 0
197297 0 134998332 0
177549 0 117112716 0
^C
De-Duplication of Captured Packets
Some AppResponse appliance models can de-duplicate packets that are captured on multiple interfaces.1 An appliance de-duplicates packets as follows:
No MIFGs defined on appliance:
AppResponse de-duplicates all captured packets and stores them in the rolling buffer.
One or more MIFGs defined on appliance:
AppResponse de-duplicates all packets captured within each MIFG and stores them in the rolling buffer.
Note that AppResponse does not de-duplicate packets captured in different MIFGs. This means that, if you download packets that were captured in different MIFGs during the same time window, your capture data might include duplicate packets.
If you have an Riverbed solution that includes Transaction Analyzer functionality, you can de-duplicate captured packets by importing the AppResponse capture data into Transaction Analyzer.
Previewing and Downloading Captured Packets by MIFG
When the Collect data for Monitoring Interface Groups option is turned on, the AppResponse appliance saves the MIFG where each packet was captured. This enables you to preview, filter, and download packets by interface group in the Packet Download Manager.
