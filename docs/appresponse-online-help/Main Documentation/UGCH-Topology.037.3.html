IP Topology
The IP topology is used to examine layer-3 connectivity to an IP address. This information is based on data collected through traceroute. The IP topology tab can be used to manually initiate and visualize live traceroutes. It can also be used to examine historic automated traceroutes that are stored in the appliance database. For information on traceroute collection and configuration, see Automated and Manual Traceroutes.
To invoke the IP Topology view, choose View > Topology > IP Topology.
Figure 7
IP Topology view
 
IP Topology Window
The IP topology window is blank by default. Its operation depends on the settings of the checkboxes at the top of the selector pane, which are therefore reviewed next.
Live and Historic Traces
Checking the Live checkbox at the top of the IP Topology selector pane causes a traceroute to be executed against the IP address(es) selected when the Go button is clicked. See Automated and Manual Traceroutes.
The results of all automated traceroutes are stored in the appliance database. Checking the Historic checkbox at the top of the IP Topology selector pane causes traceroutes previously executed against selected IP addresses to be displayed when the Go button is clicked. If none were executed, the display remains blank unless Live is also checked.
View an IP Topology
You can view an IP topology using any of the following methods:
Enter an arbitrary IP address or Hostname in the text box labeled Trace and click Go.
Separate multiple addresses with commas.
Select an IP address from the lists in the selector pane:
–
Top Traffic IPs
Drill down on Top Traffic IPs to see a list of IP addresses with the most amount of traffic.
–
Preferred IPs
Drill down on Preferred IPs to see a list of IP addresses configured in the Preferred IP Manager. Choose an IP address from the list and click Go.
–
Historic Traces
–
Drill-Down on Historic Traces to see stored traceroutes organized by IP Address, Business Group Container, or Business Group. Only Business Group Containers or Business Groups containing IP addresses that have been traced are displayed. The Historic checkbox must be checked to display these traceroutes (see Live and Historic Traces).
To select multiple addresses, use ctrl+click or shift+click.
Drag and drop an IP address from another Desktop Console view (e.g., table) into the display pane of the IP topology window.
When a new destination IP or hostname is added to the IP topology the resulting topology is charted in the display pane. IP addresses are continually added to the existing display until the display pane is cleared.
To clear the display pane, click the Clear button in the toolbar.

IP addresses displayed in the topology are listed in the index, which appears at the bottom of the selector pane on the left side of the IP topology window.
Figure 9
IP Topology - Index
 
The IP Topology window can display multiple traceroutes to a single destination IP address. This allows you to compare the current traceroute to a destination to historic traceroutes stored in the database. Each traceroute is identified in the index by the timestamp at which the traceroute was executed. Drill-down on an IP address within the index to display the timestamp of the traceroute charted. The end-to-end round trip time is displayed in the trace route trip time (TRTT) rightmost column. If the TRTT is blank, the trace was not completed.
When a traceroute is selected in the index, the corresponding topology is highlighted in the chart pane. The selected topology appears in bold. Likewise, when a topology is selected by clicking on one of its elements in the display pane, the corresponding index entry is highlighted.
IP Topology Index Context Menu
The index context menu provides a number of controls. To invoke the menu, drill down on a specific IP address in the index and right-click.

IP Topology - Index Context Menu
 
Expand All
The Expand All control displays the timestamps of all traceroutes currently displayed in the IP topology window. This has the effect of expanding all nodes in the index.
Show/Hide Columns
The Show/Hide Columns control allows you to add or remove the Group and TRTT columns.
Show Hostnames
The Show Hostnames control shows hostnames instead of IP addresses.
View Trace Details
View Trace Details opens a separate table and shows information on each individual trace.
View Trace Summary
View Trace Summary opens a separate table and shows average values for the entire path. You can drill down to specifics.
In addition to the information displayed by the standard traceroute command line program, the detailed view includes the AS number of each hop in the traceroute. Similar information can also be displayed in the topology itself by hovering the mouse pointer over any node.
IP addresses are resolved to hostnames using DNS. AS numbers are mapped to hostnames using the WHOIS Database (see WHOIS Database).
Removing an IP Address
An IP address can also be removed by left-clicking on a node and clicking on the trash can icon or pressing the delete key. The traceroutes that include the selected nodes are highlighted. Clicking on the trash-can icon, or pressing the delete key, causes all highlighted traceroutes to be removed. This means the route to the IP address is no longer displayed in the topology window. The traceroute is still stored in the appliance database.
Node Types
There are several types of nodes displayed in the topology:
Home node
The home node represents the appliance. The appliance is the source of the traceroutes used to determine the topology to each destination. The home node is represented by an image of the appliance in the topology map.
Figure 11
IP Topology - Home Node
 
Intermediate node
The intermediate nodes represent routers, or switches, that route traffic IP traffic between the home node and the end nodes (traceroute destinations). Intermediate nodes are represented as cylinders.
Figure 12
IP Topology - intermediate Node
 
Cloud node
Cloud nodes represent an IP, or set of IP addresses, that do not respond to traceroute queries. Traceroute determines the route to a destination by issuing queries with ever-increasing IP Time to Live (TTL) values until the destination is reached. Each hop along the route must decrement the TTL. When the TTL reaches zero, the packet must be discarded and an ICMP error message is returned to the originating host. The ICMP error message allows traceroute to identify routers along the path to the destination. Some routers (or networks) are configured not to return ICMP error messages. Traceroute is unable to identify these hops in the absence of the error message. A cloud node is used to represent a single IP address, or set of IP addresses, that do not return an ICMP error.
Figure 13
IP Topology - Cloud Node
 
End node
The end node represents the destination IP address of a traceroute.
Figure 14
IP Topology - End Node
 
Node Coloring
By default, nodes displayed in the IP topology window are colored by the Traceroute Round Trip Time. Darker shades of red indicate higher traceroute round trip times. Nodes can be colored by any metric collected by the appliance. To do so, select the desired metric in the Metric control.
Figure 15
IP Topology - Metric Control
 
Metrics other than Traceroute Round Trip Time (TRTT) are collected only for end points, not for intermediate nodes. This means that coloring by a metric other than TRTT results in only the end nodes being shaded red.
NOTE—It is important to distinguish between Round Trip Time and Traceroute Round Trip Time since their names are similar but the values are often different:
Traceroute Round Trip Time is collected through traceroute and is available for each intermediate node in the topology.
Round Trip Time [msec] is collected by parsing TCP headers and is available for end nodes only. As a result, when coloring by Round Trip Time, the intermediate nodes are left blank.
Finally, the values of Traceroute Round Trip Time and Round Trip Time are often different due to the manner in which they are collected. Round Trip Time is measured on live TCP sessions. These sessions include data transfer that causes natural congestion, in the form of increased queue depth, on the path to the destination. This congestion causes a slight increase in measured Round Trip Time. Traceroute Round Trip Time is measured through an active traceroute. This polling method utilizes a very small query packet that often does not coincide with large data transfers to the destination (unlike the TCP Round Trip Time measurement). Traceroute Round Trip Time measurement is often made when queues are empty resulting in lower values.
IP Topology Toolbar
The following controls are available on the IP Topology toolbar.
Figure 16
IP Topology toolbar
 
The list below describes the functions of the controls on the top from left to right:
Clear Graph — Clears the current graph from the display pane.
Selection Mode — Select a node in the graph. The selected node is subject to the next operation.
Zoom mode — Click and sweep on areas to zoom in on.
Fit Content — Adjusts the zoom level so that the graph fits inside the display pane.
Zoom To 100% — Shows the graph to scale.
Zoom In x2 — Doubles the zoom level.
Zoom Out x2 — Halves the zoom level.
Show Labels — Shows the hostnames in the graph.
Group by AS — Shows all IP addresses that belong to the same Autonomous System (AS) are grouped in a single box in the topology window.
Select Background Image — Sets background image in the display pane. The image can be removed by right-clicking in an empty part of the display pane and selecting Clear Background from the menu.
Delete — Deletes selected groups from the graph.
Export Graph — Exports the current graph to a .gif, .jpg or .png file.
Help Pages — Shows the Operations Manual help pages.
Layout — Select the layout view of the graph in the display pane. There are five to choose from: Tree (Horizontal), Tree (Vertical), Organic, Circular and Orthogonal.
The following list describes the functions of the controls on the bottom from left to right:
Metrics — Selects the metrics shown in the graph.
Toolbar Options — Choose which controls you’d like in the toolbar.
Note: You must press Enter after changing the number in the Top field for your choice to take effect.
