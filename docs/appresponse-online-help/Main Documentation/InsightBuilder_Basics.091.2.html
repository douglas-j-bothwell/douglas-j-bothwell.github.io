Summary of Terms
All of the terms used to describe the elements and functions of an Insight and the Insight Development Kit are English words whose standard meaning, while similar to their use in this manual, may connote slightly different things to different readers. This summary describes the conventions used in this manual to describe the elements and functions of Insights and the Insight Development Kit. As a convenience, it is repeated at the beginning of each chapter.
Element—When capitalized, this comprises Components, Interactions, Layers, and Controls, and Five Building Blocks. Lower-case instances refer to the generic meaning of the term and most often refer to parts of the Insight Development Kit UI.
Component—When capitalized, this refers to the objects (Visual and Analysis Components) that make up an Insight.
Interaction—When capitalized, this refers to the messages sent between Insight Components.
Control—When capitalized, this term refers to sub-components within the Toolbar Component that control the Interactions that the Toolbar sends out or control internal interactions between Controls. Lower-case instances refer either to controls in the toolbars of Visual Components such as Charts or to the controls in the Insight Development Kit UI itself.
Layer—When capitalized, this refers to the instances of the inheritance scheme used in Insights, which act much like pages. Layers are described in Working with Layers.
The following terms are used to describe the elements of the Insight Development Kit UI.
Window - used only to describe the entire Insight Development Kit window.
Area - a group of panes within a Window.
Pane - a sub-component of a Window set off from other panes by dividers that can be moved to resize the pane.
View - a display mode, as in Actual View or Schematic View.
Mode - a functional mode, as in Grid Edit Mode or Component Edit Mode.
