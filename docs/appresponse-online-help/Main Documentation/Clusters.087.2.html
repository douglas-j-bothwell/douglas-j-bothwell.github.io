Domain Architecture
A domain consists of a Director, a special AppResponse Appliance responsible for managing the domain, and a set of regular AppResponse Appliances. Users can log into the Domain Director using the AppResponse Console to access data consolidated from all the appliances in the domain, or they can use the AppResponse Console to connect to the individual appliances directly to view only the data collected by that appliance.

Illustration of Domain Architecture
 
Domain Director
The Director is responsible for managing the appliances in the domain. The director provides the following services to the domain:
Push global configuration changes to the AppResponse Appliances in the domain and log the results of configuration operations.
Consolidate data collected by the AppResponse Appliances in the domain and provide single sign-on access to all domain data.
Unlike the appliances, the director does not collect data directly from the network.
