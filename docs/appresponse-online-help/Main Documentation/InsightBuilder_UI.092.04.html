Working with Visual Components
This section briefly reviews how to add, position, resize, and edit Visual Components. Visual Components may also be sorted in the Element pane by name or grid position.
Adding Controls to a Toolbar is covered in Edit Toolbar.
With a few exceptions, Layer-related Visual Component operations are discussed in the section Working with Layers.
Adding, Removing, and Replacing Visual Components
The Insight Development Kit enables you to add, remove, and replace Visual Components.
Add Visual Components
You may add a Visual Component in two ways. First make sure that you have selected the correct Layer in the Element pane.
Click on the Insight Development Kit toolbar icon for the desired Visual Component, then click on the active Layout Grid pane where you want it.
Click and drag the Insight Development Kit toolbar icon, for the desired Visual Component, to the active Layout Grid pane where you want it.
Clicking on any Insight Development Kit toolbar control turns off the Grid Edit Mode if it is on.
The Insight Development Kit automatically assigns a unique name to the Visual Component, which consists of its display name as listed in Components, plus an incremental number; e.g., Chart:multigrp1. It is suggested you rename the Visual Component immediately to a name that has more meaning to your Insight (this is helpful when you start to add Interactions). To do this, right-click the Component in the Element pane and select Rename.
Remove or Replace a Visual Component
You may remove or replace a Visual Component in a number of ways, using the Layout Grid layout context menu or the Element pane context menu:
In the Grid Edit mode, right-click on the Visual Component to display the Layout Grid layout context menu and select Remove.
Select Remove from All Layers if you wish to remove the Component from all Layers in which it exists.
Select Replace with New Component to display the Select Component dialog (Figure 27), and select the desired Component to replace it in any Layer in which it exists. Interaction links that existed between the old Component and other Components remain if the new Component is capable of consuming/producing them. Interactions that the new Component cannot consume/produce will be removed.
In the Element pane, select the Visual Component, right-click to display the Element pane context menu, and select Remove.
Select Remove from All Layers if you wish to remove the Component from all Layers in which it exists.
Select Replace with New Component to display the Select Component dialog (Figure 27) and select the desired Component to replace it in all Layers in which it exists. Interaction links that existed between the old Component and other Components remain if the new Component is capable of consuming/producing them. Interactions that the new Component cannot consume/produce are removed.
Sizing, Positioning, and Spanning Visual Components
The Insight Development Kit allows you to resize, reposition, and span (occupy more than one pane) Components.
Resize Visual Components
You may resize Visual Components in two ways. In all cases, resizing affects an entire row or column.
In the Layout Grid, in Component Edit mode (see Selector Control (Edit Mode)) of the Actual View (see Layout Grid - Actual View), click-drag a divider. Resizing Visual Components vertically in this manner is governed by the size change mode, as described in Size Change Mode.
You can use the Test/Layout to resize the window or any Visual Components within it. Those changes are recorded when you save the Insight. This is a handy way to put the finishing touches on the appearance of an Insight. Click Apply to apply any size changes and continue working in the preview window.
Reposition Visual Components
You may reposition Visual Components in two ways. Repositioning operations affect only the Layer currently selected. The size of the Component and all its Interaction links are maintained. You may only reposition into an Active pane.
In the Layout Grid, in the Grid Edit mode (see Selector Control (Edit Mode)) of the Actual View, click and drag a Component to a different Active pane. The Component does not move until you release the mouse button.
In the Element pane (with the Layout Pane in either Actual or Schematic View), click on the Component name, right-click to display the Element Pane context menu, and select Move To to display the Move Visual Component dialog.
Figure 23
Move Visual Component dialog box
 
The drop-down pick list allows you to choose the new grid location of the Visual Component in “row, column” format. Panes already filled by a Visual Component are not displayed. The Move Visual Component dialog only shows the available cells in the grid.
Span Visual Components
For visual purposes, you may also span a Visual Component so that it takes up two or more panes in a row or column. You may only span into an active pane. The span is not visible in the Schematic or Workflow Views.
In the Layout Grid, in the Grid Edit mode (see Selector Control (Edit Mode)) of the Actual View, click on a Visual Component to select in and right-click to display the Layout Grid layout context menu. Select Fill to Right to fill both its current pane and the pane to the right.
Select Fill Below to make the Component fill both its current pane and the pane below it.
The new span does not appear until you either click Test/Layout and return to the Actual View, click the Refresh button on the Selector toolbar, or select the Schematic or Workflow View and return to the Actual View.
In the Element pane (with the Layout Pane in either Actual or Schematic View), click on the Component name, right-click to display the Element Pane context menu, and select Fill to Right to make the Component fill both its current pane and the pane to the right.
Select Fill Below to make the Component fill both its current pane and the pane below it.
The new span does not appear until you either click Test/Layout and return to the Actual View, click the Refresh button on the Selector toolbar, or select the Schematic or Workflow View and return to the Actual View.
In the Layout Grid, in either the Component Edit mode of the Actual View or in the Schematic View, right-click on the Active pane into which you wish the Component to move. This brings up the Active Pane context menu.
Figure 24
Active Pane Context Menu
 
Select Replace With Component To The Left to allow the Component to the Left to span into this Active pane.
Select Replace With Component From Above to allow the Component Above to span into this Active pane.
The new span does not appear until you either click Test/Layout and return to the Actual View, click the Refresh button on the Selector toolbar, or select the Schematic or Workflow View and return to the Actual View.
To undo a span, the Layout Grid must be in Actual View, Grid Edit mode (see Selector Control (Edit Mode)). Click on the Component to select it, right-click to display the Layout Grid context menu, and select Remove Component Assignments (Fill Right, Fill Below). The change appears immediately.
Editing, Renaming, and Sorting Visual Components
The Insight Development Kit enables you to edit and rename Visual Components.
Edit Visual Components
There are five ways to edit a Visual Component:
In the Schematic View of the Layout Grid, edit the Component using its edit dialog.
In the Actual View, Grid Edit mode of the Layout Grid, either double-click on the Component, or right-click it to display the Layout Grid layout context menu and select Edit Visual Component to display its edit dialog.
In the Actual View, Component Edit mode of the Layout Grid, select any control in the Component or use any relevant method to call up a dialog, e.g., in a Chart, double-click to display the Chart Setting dialog.
In the Element pane, select the Visual Component, right-click to display the Element Pane context menu, and select Edit Visual Component to display its edit dialog.
In the Workflow View, either double-click on the Component node, or right-click on it to display the Layout Grid layout context menu and select Edit Component to display its edit dialog.
Rename Visual Components
The are three ways to rename a Visual Component:
In any view, select it in the Element pane, right-click to display the Element Pane context menu, and select Rename to display the Rename dialog.
In any view, double-click it in the Element pane to display a text box.
In the Workflow View, select the Component node, right-click to display the Element Pane context menu, and select Rename to display the Rename dialog.

Rename dialog box
 
Any Interactions that this Component produces or consumes are also renamed.
Sort Visual Components
You may sort Visual Components in the Element pane by name or grid position, by using the Insight Development Kit Settings dialog, Figure 46.
Working with Hidden Visual Components
Hidden Visual Components are used to construct multi-Layer Insights (see Working with Layers) and Tabbed Containers (see Tab Container). To add a Hidden Visual Component, in the Element pane select the Visual Components branch of the Layer you want the Hidden Visual Component to be in.
Figure 26
Select Visual Components Branch
 
Right-click on the selected branch to display the Element Pane context menu and select Add Hidden Visual Component. This brings up the Select Component dialog from which you may select the desired Visual Component.
Figure 27
Select Component Dialog
 
Select the Component you want and click OK to add the selected Hidden Visual Component.
Other Operations on Hidden Visual Components
Hidden Visual Components can only be edited, renamed, replaced, or removed using the Element Pane context menu. The operations are the same as with visible Visual Components.
Hidden Visual Components can become visible by selecting them in the Element pane and dragging to the Layout Grid. Remember that you must drag to an Active pane.
