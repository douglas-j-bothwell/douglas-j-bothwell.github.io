Distributing Configuration Information
An appliance that belongs to a domain receives configuration information from the Domain Director. This allows an administrator to make changes in a single location (on the Director) and apply those changes to all appliances in the domain.
Some configuration items are automatically distributed to all appliances within a domain. Other configuration items can be selectively distributed to specific appliances using a distribution list. For these items, the distribution list can be specified in the Distribution tab in the relevant manager. The Distribution tab is only available on the Domain Director.
For instance, the following figure shows the Distribution tab of the Alert Manager running on the director of a domain that contains three machines: cuda1, cuda2 and cuda3. The Distribution tab shows an alert, initially configured on the director, that is distributed to appliances cuda1 and cuda2, but not cuda3.
Figure 6
Alert Manager - Distribution Tab
 
To add an appliance to the distribution list for a configuration item, select the desired appliance from the Available Appliances pane and click the right arrow button.
 
When the configuration change is applied (by clicking on the OK or Apply button), the director pushes the configuration item to each appliance in the distribution list. If the configuration push is successful, the Status field in the Distribution List pane shows Distributed. Otherwise, an error appears in the Status field. The error is also recorded in the Director Update Log.
Configuration items distributed from the director to an appliance show up in the relevant manager on an appliance. These items can be differentiated from locally defined items by looking at the Origin column in the manager.
Figure 7
Origin of a Distributed Configuration Item
 
Configuration items distributed from the director have a global origin. Global items cannot be modified on an appliance, they must be changed on the director.
Configuration items defined on a specific appliance have a local origin. These items are modified using the relevant manager on the appliance itself.
The following types of configuration items can be distributed from the director to the appliances in a domain:
Business Groups
Alerts
Applications
AS Information
User Accounts

To simplify this process, the Business Group Manager allows you to set the distribution list for several Business Groups simultaneously. To do so, select the desired Business Groups in the table at the top of the Business Group Manager, right-click and select Configure Distribution to invoke the distribution dialog.

Conflict Resolution
A conflict may occur when pushing a global configuration item to an appliance. For instance, a local configuration item may exist with the same name as a new global configuration item. These conflicts are resolved during distribution and recorded in the Director Update Log. The rules for dealing with a conflict during distribution differ depending on the type of configuration item. These rules are documented in the following sections.
Distributed Business Groups
Use the Distribution tab in the Business Group Manager to distribute Business Groups from the director to specific appliances. Data for Business Groups distributed to appliances are collected locally on each appliance. Users connected to each appliance can examine data for these Business Groups using regular tables or charts.
Business Group Conflict Resolution
If a local Business Group exists with the same Name, Definition and Status as a new global Business Group, then the existing local group is replaced with the global Business Group. It does not matter if the description or status of the Business Groups differ. The group definition must match exactly. This means that the IP/AS definition for regular Business Groups must be the same or the Member definitions for Business Group containers must be identical.
If a local Business Group exists with the same Name as a new global Business Group but the Definition or Status is different, then the local Business Group is renamed to include the suffix “(local)”. For instance, a local Business Group named “New York” that conflicts with a new global Business Group of the same name is renamed to “New York (local)”.

Conflict Resolution
 
Distributed Alerts
Alerts defined in the Alert Manager on the director can be distributed to specific appliances using the Distribution tab. The alert executes locally on each appliance. This means that each appliance checks the alert condition against the data seen by this appliance to determine whether to generate a notification. In addition, if the alert actions specify that an SNMP trap is to be generated, the trap is sent to the SNMP Trap destinations configured on that appliance. (See Configuring SNMP.)
If an alert is defined against a Business Group, then that alert can only be distributed to those appliances that have also received the Business Group configuration.
Alert Conflict Resolution
When a new global alert is distributed from the Domain Director, the appliance checks if a local alert exists with the same Name, Group, Condition, Schedule, Status and Actions. If such an alert exists, it is replaced with the new global alert. Notice that the description, email and snmp settings for the alert are not considered during the conflict resolution process.
If a local alert exists with the same Name, but with different settings for Group, Condition, Schedule, Status or Actions, the local alert is renamed to include the suffix “(local)”.
Distributed Applications
Applications defined in the Defined Application Manager on the director are always distributed to all appliances in the domain. As a result, the Distribution tab is only used to display the status of the distribution operation. Recognized Applications are not distributed to appliances until they have been upgraded to v6.0.
Scaling is limited to 300 local applications and 300 global applications (600 total).
Application Conflict Resolution
When a new global application is distributed from the Domain Director, the appliance checks if a local application exists with exactly the same specifications. If such an application exists, it is replaced with the new global application.
If a local application exists with the same Name, but with different specifications, the local application is renamed to include the suffix “(local)”. Metrics are collected for both definitions. In the case of otherwise identical applications with different turn tracking, the global application takes precedence, and metrics are not collected for the local application. (For more information, see Turn Tracking on page AL-3-56.)
Distributed AS Information
ASes modified in the AS Information Manager on the director are always distributed to all appliances in the domain. As a result, the Distribution tab is only used to display the status of the distribution operation. Since each appliance ships with a default configuration for AS Information, the distribution tab for the AS Information is blank until an AS is modified and pushed to the appliances.
AS Information Conflict Resolution
New global AS Information distributed from the domain director always overwrites the local AS Information stored on the appliance.
When a global AS Information item is deleted from the director, the appliances revert to the default value for that AS. This default value is defined in the static copy of the WHOIS directory shipped with the appliance software release. See AS Information Manager.
Distributed User Accounts
For an explanation of distributing user account information within a domain, see Managing User Accounts.
Deleting Global Configuration Items
When a global configuration item is deleted on the director, it is also deleted from each of the appliances to which it was distributed. However, if the global configuration item is being used by a local configuration item (e.g., a local static alert can be defined against a global Business Group), then the original global configuration item is simply converted to a local configuration item on appliances that reference it.
Director Update Log
The Director Update Log maintains a list of errors that may have occurred while the director distributes configuration information to the domain appliances. As a result, the Director Update Log is only accessible on the director.
To view the Director Update Log on the Director, choose View > Logs and then select “Director Update Log”.
Figure 12
Director Update Log
 
The director update log records the following conflicts and warnings.
 
Table 2
Directory Update Log: Conflicts and Warnings
Message
Description
Local Item With Duplicate Name Renamed
A global configuration item has been distributed with the same name as a local configuration item and caused the local item to be renamed according to the conflict resolution rules for that item type.
Deleted Item Demoted Due To Dependencies
A local configuration item has a dependency on a global configuration item that was just deleted on the director. The dependency does not allow the item to be delete. Instead, it is changed to a local configuration item on the appliance(s) with the dependency.
Could Not Find Resource On Appliance
A configuration item (resource) that should have been distributed to an appliance is missing. For instance, a global alert that refers to a global Business Group may trigger this log entry if the global alert is distributed correctly, but the global Business Group was not distributed properly.
Could Not Push Item to Appliance
A global configuration item has not been completely distributed to an appliance. It is not unusual for distribution to require a few minutes to complete. During this time, this entry appears in the log for each configuration item being distributed. The log entry is automatically cleared after distribution is completed successfully. If the entry persists for more than 15 minutes, it is likely that an error occurred (e.g., network connectivity broken) while trying to distribute a global configuration item to an appliance.
Could Not Delete Item from Appliance
This log entry is the same as the above log entry, except that it is generated when there are problems deleting global configuration items.
Could Not Merge Business Group Tree
An error occurred while distributing a global Business Group to an appliance. The AppResponse Appliance performs a sanity check on the Business Group hierarchy each time a new group is added. This log entry is generated if a problem is detected during this check. To date, this error condition has never been observed. Please contact Riverbed support if this entry appears in your Director Update Log.
Global Item Not Added Due To Local Item With Duplicate Name
A global user account has been distributed with the same name as a local user account. As a result, the global account is not distributed and the local account remains unchanged.
This log shows the last error or warning for every group or appliance. If a problem occurs during a global configuration item distribution, an entry is created for that specific configuration item and the affected appliance in the log. If the operation is reapplied and the configuration item is distributed successfully, the warning remains in the log and must be manually deleted by clicking on the Delete button. If the same configuration item is distributed and there is a new warning on the same appliance, a new log entry overwrites the previous one.
