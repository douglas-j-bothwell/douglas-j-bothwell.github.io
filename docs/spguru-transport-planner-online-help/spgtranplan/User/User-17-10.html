Dimensioning: Batch Options

Table 16-6 lists the options you can specify for a dimensioning command. For more information, see Dimensioning .

Table 16-6 Dimensioning: Batch Options 
Option
Description
Comment
-cp
Client protection:
  • 0 = no
  • 1 = yes
For dimensioning with 1+1 protection only
-ctyp
Candidate line system (as defined in WDM Link Equipment Properties) / bit rate types:
0 = <first_LS_type> / STM-1 / OC-3
1 = <second_LS_type> / STM-4 / OC-12
2 = <third_LS_type> / STM-16 / OC-48
3 = <fourth_LS_type> / STM-64 / OC-192
4 = <fifth_LS_type> / STM-256 / OC-768
5 = <sixth_LS_type> / STM-512 / OC-1536
6 = <seventh_LS_type> / STM-1024 / OC-3072
Relevant only if optimized set is selected. To specify multiple types, separate the numbers by a `+' character
(for example, "-ctyp 1+3+4")
-dr
Diverse Routing:
  • 0 = no
  • 1 = yes
For dimensioning without protection only
-drm
Diverse routing method
  • 0 = with the same end node
  • 1 = within single traffic matrix
  • 2 = between multiple traffic matrices
For diverse routing only
-dro
Diverse routing optimization
  • 0 = no
  • 1 = yes
For diverse routing only
-dth
Threshold in %, resources used below threshold or no candidate for removal
This option is relevant only if link or node optimization is chosen. You can express a value between 0 and 100
-eab
Equip all bands on added fiber pairs:
  • 0 = no
  • 1 = yes
For OCH dimensioning only
-imname
Name and path of the file containing the routes to import
 
-k
  • Value for K shortest paths
  • Value for the number of alternative paths
  • For dimensioning with protection
  • For optimized diverse dimensioning
-layer
Layer to dimension:
  • DCL = digital client layer
  • OCH = optical channel layer
The DCL option is applicable for all-ECC networks only (that is, networks with no EOCC or OCC nodes)
-ldlay
Link disjointness down to:
  • DCL = digital client layer
  • OCH = optical channel layer
  • OMS = optical multiplex section
  • OTS = optical transmission section
For dimensioning with 1+1 protection only
-lex
Link expansion:
  • 0 = default line system
  • 1 = optimized set of line systems
Not relevant for transparent routing
-load
Traffic matrix to accommodate in the network
 
-lut
Apply link usage thresholds
  • 0 = no
  • 1 = yes
 
-nd
Disjointness of working and protecting path:
  • 0 = link disjoint
  • 1 = link and node disjoint
For dimensioning with 1+1 or shared path protection only
-noi
Number of optimization iterations
Relevant only if optimized set is selected
-oadmdim
OADM dimensioning:
  • 0 = do not add OADMs
  • 1 = expand existing OADM relations
  • 2 = add unique OADM relations
  • 3 = add multiple OADM relations
For transparent routing only
-otl
Optimize link expansion:
  • 0 = no
  • 1 = yes
For OCH dimensioning with no protection or 1 + 1 protection only
-otn
Optimize node expansion:
  • 0 = no
  • 1 = yes
For OCH dimensioning with no protection and 1+1 protection only
-pt
Protection type:
  • 0 = no protection
  • 1 = 1+1 protection
  • 2 = shared path protection
  • 3 = link restoration
  • 4 = path restoration
Shared path protection and link or path restoration are applicable for transparent networks only if link dimensioning is included
-rc
Routing cost:
  • 0 = hop+fiber
  • 1 = hop+reg+fiber
  • 2 = fiber length
  • 3 = real cost
  • 4 = user specified
  • 5 = max availability
  • 6 = least used
Only hop+fiber and real cost are applicable when using transparent routing
-rrf
Name and path of restoration routes file
For dimensioning with mesh restoration only
-shm
Value for m
For dimensioning with shared path protection only
-shn
Value for n
For dimensioning with shared path protection only
End of Table 16-6
