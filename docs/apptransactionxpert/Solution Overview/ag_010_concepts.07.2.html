Terminology
architecture directory
An architecture-specific system directory within a particular AppTransaction Xpert release. Executables and libraries are typically stored in this directory. Architecture directories are subdirectories of the sys directory of a particular release. In this documentation, the symbol <arch> represents an architecture directory. binary directory
A subdirectory of the installation directory that contains the AppTransaction Xpert binaries. The binaries for a specific release and computer architecture are located in <release>/sys/<arch>/bin. Under Linux, a special binary directory contains scripts that allow AppTransaction Xpert to locate the binaries for the platform being used.
In this documentation, the symbol <bindir> represents the path to the directory where binary files for AppTransaction Xpert are stored. For example, if AppTransaction Xpert 17.0.T is installed in the directory /usr/opnet, then <bindir> represents these path names:
•
(Windows) \opnet\17.0.T\sys\pc_intel_win32\bin
•
(Linux) /usr/opnet/17.0.T/sys/pc_intel_linux/bin
CD-ROM directory; CD-ROM drive
The mount point for the CD-ROM drive of the install workstation. The symbol <cd_dir> represents the CDCD-ROM directory in this manual. (Under Windows, the CD-ROM drive will be mounted at a particular drive letter, rather than a directory. The symbol <cd_drive> represents the drive letter of the CD-ROM drive.)
client workstation
A workstation used to execute AppTransaction Xpert programs. Each client workstation must have access to the AppTransaction Xpert directory, either on its local disk, on a server disk, or via the Network Filesystem (NFS).
install workstation
A workstation with a compatible CD-ROM drive. This workstation must be able to mount the server workstation disk partition containing the AppTransaction Xpert directory via the Network Filesystem. At many sites, the server workstation has a CD-ROM drive and is therefore also the install workstation. At completely stand-alone sites, a single workstation fills all three roles: client, server, and install workstation.
installation directory
The directory where AppTransaction Xpert is or will be loaded. You can choose where you want to install the software; almost any location will work. However, because the software occupies a significant amount of disk space, the disk partition holding the installation directory must have enough free space before installation begins (to determine the amount of partition free space required, refer to technical support).
Typical locations for the AppTransaction Xpert directory are:
Linux: /usr/opnet or /usr/local/opnet
Windows: C:\Program Files\OPNET
The symbol <install_dir> represents the installation directory. Typically, there is only one path name to this directory. Different subdirectories within the installation directory might contain different releases of AppTransaction Xpert.
library directory
A directory containing the AppTransaction Xpert libraries. The libraries for a specific release and computer architecture are located in <release>/sys/<arch>/lib.
In this manual, the symbol <libdir> represents a library directory. For example, if AppTransaction Xpert is installed in the directory /usr/opnet under Linux, then <libdir> represents the directory path name /usr/opnet/17.0.T/sys/pc_intel_linux/lib.
licensing system
The licensing system allows you to run AppTransaction Xpert or another licensed application on any workstation on the local network. See Licensing System Overview for a description of the important components of this system.
release CD-ROM
The distribution media supplied as part of the AppTransaction Xpert release materials.
release directory
A subdirectory of the installation directory that contains the software for a particular AppTransaction Xpert release. Release directories are subdirectories of the installation directory. In this documentation, the symbol <release> represents the release directory. You can derive the name of the release directory from information printed on the CD-ROM label. For example, if the label reads “AppTransaction Xpert 17.0.T”, the corresponding release directory is <install_dir>/17.0.T.
server workstation
The workstation that stores the AppTransaction Xpert on its disk. A server workstation may also be a client workstation. More commonly, however, a server workstation is a file server that is shared by a workgroup.
