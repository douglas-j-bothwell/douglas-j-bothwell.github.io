Routing: Batch Options

Table 16-5 lists the options you can specify for a routing command. For more information, see Routing .

Note—To perform a design action on several traffic matrices at once, specify the -load option multiple times. For example, the option -load OCH_0 -load OCH_1 performs a single design action on both traffic matrices, "OCH-0" and "OCH_1."

Table 16-5 Routing: Batch Options 
Option
Description
Comment
-aedb
Allow equipping of dark bands:
  • 0 = no
  • 1 = yes
For OCH routing only
-cp
Client protection:
  • 0 = no client protection
  • 1 = client protection
For 1+1 protection only
-drm
Diverse routing method
  • 0 = with the same end node
  • 1 = within single traffic matrix
  • 2 = between multiple traffic matrices
For diverse routing only
-dro
Diverse routing optimization
  • 0 = no
  • 1 = yes
 
-imname
Name and path of the file containing the routes to import
 
-layer
Layer in which to route:
  • DCL = digital client layer
  • OCH = optical channel layer
 
-ldlay
Link disjointness down to:
  • DCL = digital client layer
  • OCH = optical channel layer
  • OMS = optical multiplex section
  • OTS = optical transmission section
For 1+1 protection only
-load
Traffic matrix to route
 
-lut
Apply link usage thresholds
  • 0 = no
  • 1 = yes
 
-mli
Maximum number of local iterations without improvement
For optimized routing only
-ncos
Initial changes per major iteration
For optimized routing only
-rnl
Node limitations:
  • 0 = no node limitations
  • 1 = impose node limitations
Only if discrete nodes are applied
-rnld
Constrain OCH routing by DCL node size:
  • 0 = no constrain by DCL node size
  • 1 = constrain by DCL node size
For routing in the OCH layer and discrete DCL node types only
-ncli
Number of changes per local iteration
For optimized routing only
-nd
Disjointness of working and protecting path:
  • 0 = link disjoint
  • 1 = link and node disjoint
For 1+1 protection only
-nos
Number of optimization steps
For optimized routing only
-pt
Protection type:
  • 0 = no protection
  • 1 = 1+1 protection
 
-pur
Penalty unrouted
For optimized routing only
-rc
Routing cost:
  • 0 = hop count
  • 1 = hop count with in-line regenerators considered as an extra hop
  • 2 = fiber length
  • 3 = real cost
  • 4 = user specified
  • 5 = max availability
  • 6 = least used
  • 7 = combined metric
Hop count and real cost are applicable for transparent routing only
-rs
Routing strategy:
  • 0 = sequential
  • 1 = distributed
  • 2 = combined
  • 3 = optimized
  • 4 = diverse routing
 
-sits
Apply service identifiers
  • 0 = no
  • 1 = yes
For DCL routing only
-sort
Sorting order:
  • 0 = unsorted
  • 1 = longest first
  • 2 = shortest first
  • 3 = biggest first
  • 4 = smallest first
 
End of Table 16-5
