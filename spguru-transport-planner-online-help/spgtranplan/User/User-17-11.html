Grooming: Batch Options

If a batch file runs a grooming operation with 1+1 protection and link disjointness at the OCH, OMS, or OTS layer, SP Guru Transport Planner automatically accommodates the supporting OCH traffic matrix in the network.

In all other cases, the batch file must also include a command to accommodate the supporting OCH traffic matrix in the network. To do this, run a routing or dimensioning action. For more information, see Grooming DCL to OCH Traffic .

Table 16-7 lists the options you can specify for a grooming command.

Table 16-7 Grooming: Batch Options 
Option
Description
Comment
-cp
Client protection:
  • 0 = no client protection
  • 1 = client protection
For grooming with 1+1 protection only
-gra
Grooming algorithm:
  • 0 = end to end grooming
  • 1 = link by link grooming
For grooming with optimized routes only
-grbr
Target bit rate optical channels (if DCL layer):
  • 0 = STM-1 / OC-3
  • 1 = STM-4 / OC-12
  • 2 = STM-16 / OC-48
  • 3 = STM-64 / OC-192
  • 4 = STM-256 / OC-768
  • 5 = STM-512 / OC-1536
  • 6 = STM-1024 / OC-3072
Target bit rate digital trunks (if LOP layer):
  • 0 = STM-1 / STS-1
  • 1 = STM-4 / STS-3
  • 2 = STM-16 / STS-12
  • 3 = STM-64 / STS-48
  • 4 = 1 GbE / STS-192
  • 5 = 1 GbE
 
-grfixed
Grooming approach:
  • 0 = optimized routes
  • 1 = fixed routes
For DCL grooming only
-gro
Grooming optimization:
  • 0 = no optimization
  • 1 = optimization
For DCL grooming only
-gruec
Use existing capacity:
  • 0 = no
  • 1 = yes
For LOP grooming only
-granc
Add new capacity:
  • 0 = no
  • 1 = yes
For LOP grooming only
-imname
Name and path of the file containing the OTS routes to import
For grooming with fixed routes only
-k
Value for K shortest paths
For DCL grooming with 1+1 protection only
-layer
Layer in which to groom:
  • LOP = lower order path layer
  • DCL = digital client layer
 
-ldlay
Link disjointness down to:
  • DCL = digital client layer
  • OCH = optical channel layer
  • OMS = optical multiplex section
  • OTS = optical transmission section
For grooming with optimized routes and with 1+1 protection only
-load
Traffic matrix subject to the grooming action
 
-lut
Apply link usage thresholds
  • 0 = no
  • 1 = yes
 
-nd
Disjointness of working and protecting path:
  • 0 = link disjoint
  • 1 = link and node disjoint
For grooming with 1+1 protection only
-pt
1+1 protection in DCL:
  • 0 = no protection
  • 1 = 1+1 protection
For DCL grooming only
-rc
Routing cost to calculate OTS routes:
  • 0 = hop+fiber
  • 1 = hop+reg+fiber
  • 2 = fiber length
  • 3 = real cost
  • 4 = user specified
  • 5 = max availability
  • 6 = least used
For grooming with fixed routes only. These options are applicable only if the option "Calculate OTS Routes" is chosen (that is, if the routes are not imported from a data file)
-rnp
Name and path of the file containing the candidate node pairs to import
For DCL grooming only
-sits
Apply service identifiers
  • 0 = no
  • 1 = yes
For DCL grooming only
-split
Split off ungroomable
  • 0 = no
  • 1 = yes
For DCL grooming only
End of Table 16-7
