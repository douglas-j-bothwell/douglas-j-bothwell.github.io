Transparent Network Routing: Batch Options

Table 16-10 lists the options you can specify for transparent network routing operations. These options are relevant in transparent networks when you apply a routing, dimensioning, or grooming command. For more information, see Transparent Networks .

Table 16-10 Transparent Network Routing: Batch Options 
Option
Description
Comment
-art
Add/remove transponders:
  • 0 = no
  • 1 = yes
For WP-OXCs with integrated TMs only
-atp
Add trib ports:
  • 0 = no
  • 1 = yes
For WP-OXCs with integrated TMs only
-awc
Allow wavelength conversion:
  • 0 = no
  • 1 = yes
 
-cf
Cost factor
 
-cop
Algorithmic routing approach under link dimensioning condition:
  • 0 = apply transparent routing algorithm
  • 1 = convert opaque routes to transparent routes
Only relevant if link dimensioning is included
-cr
Regeneration cost
 
-cwc
Wavelength conversion cost
 
-ild
Include link dimensioning:
  • 0 = no
  • 1 = yes
For the dimensioning algorithm only
-lsm
Transparency length specification:
  • 0 = length units (km or miles)
  • 1 = OA Spans
 
-owo
Options for WP-OXCs with integrated TMs:
  • 0 = BOXC – trib
  • 1 = BOXC – tap
  • 2 = BOXC – mix
For WP-OXCs with integrated TMs only
-podw
Protection on different wavelength:
  • 0 = no
  • 1 = yes
For 1+1 protection only
-ponc
Penalty per optical node crossed
 
End of Table 16-10
