Restricting License Users

You can restrict the users of a particular license server by creating a user authorization file (user_auth). In this file you list machine-user pairs that are allowed to receive licenses from that server. A license server with a user_auth file will grant licenses only to those machine-user pairs listed in the file.

WARNING—If you do not create a user authorization file, any user from any machine can obtain licenses and run applications.

A user authorization file is a text file with the same format as an administration authorization file (see Restricting License Server Administration for details). Name the user authorization file as user_auth and place it on the license server for which you want to specify users, in the same directory as the license file:

Windows: <primary_hard_drive>:\OPNET_License\

Solaris: /opt/OPNET_license/ or
                /var/adm/OPNET_license/

Linux: /opt/OPNET_license/

After adding or editing a user authorization file, you must do one of the following things to make it take effect:

  • In the License Manager, choose Tools > Refresh Server Authorization Files.
  • Stop and restart the license server.
