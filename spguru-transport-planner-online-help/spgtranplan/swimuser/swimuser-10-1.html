File Organization and Formats

An integrated SWIM project is composed of three sets of files:

  • SP Guru Network Planner files—The project file (.prj), its associated network models (.m) and any other associated files (such as files generated by Flow Analysis)
  • WDM Guru files—Another project file, with a suffix _WDM added to the project name of the SP Guru Network Planner project (thus, ending in _WDM.prj) and its associated model files.
  • SWIM files—Every scenario in a SWIM project includes an XML file that defines the mappings between the two scenarios (SPG node/WDMG node mappings and SPG link/WDMG connection mappings).

    • The XML file can also contain default mapping settings. This is a list of link models in SP Guru Network Planner and specifies the SP Guru Transport Planner traffic settings (layer and bit rate) that should be used for the connections such links are mapped to. These default settings are used for automatic mapping of links; for more information, see Setting Link Mapping Defaults and Automatic Link-to-Connection Mapping (New SP Guru Transport Planner Traffic Matrix).

The distinction between SP Guru Network Planner project files, SP Guru Transport Planner standalone project files, and SWIM project files is achieved through so-called companion files. If a .prj file is accompanied by another file with the same name with an added suffix of .wdmg, then it is considered to be a standalone SP Guru Transport Planner project. In contrast, if the same name with an extra .SWIM suffix is found, it is treated as the SP Guru Transport Planner part of a SWIM project.
