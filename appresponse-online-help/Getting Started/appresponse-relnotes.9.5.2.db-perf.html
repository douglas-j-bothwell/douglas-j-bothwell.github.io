Database Performance Module Enhancements
This release includes the following enhancement to the Database Performance Module:
Management Console changes
–
The Management Console of the Database Performance module now uses https encryption. To access the management console, go to https://<appresponse-appliance-ip>:2780 and log in as usual.
–
You can now access the Management Console of the Database Performance module directly from the AppResponse web console (Administration > System > Launch > Database Performance Module Configuration).
Authentication
–
The Database Performance module now honors AppResponse authentication. This means that you can use existing AppResponse credentials (username and password) to log in to the Database Performance module and you no longer need to set up separate credentials for this module. Module usernames still work and you can use these in addition to or instead of the AppResponse credentials.
–
The module now supports role-based authorization. There are three types of roles: superuser, config, access. Roles can be assigned to individual accounts or to groups of users. The root and admin users should use the superuser role.
Automatic discovery of databases to monitor
–
The Database Performance Module now supports auto-discovery, which will automatically find the databases to monitor. With this feature, one general monitoring rule that enables auto-discovery is sufficient for most situations, so that IP-specific monitoring rules are no longer required.
–
The module still supports IP-specific rules for their various benefits, such as specifying logical DB nodes and port bands or optionally limiting which IP addresses to monitor.
–
To construct a single general rule for autodiscovery, enter the screen for configuring monitoring rules, create a rule having IP address 0.0.0.0 with netmask 0.0.0.0 and database type "Any", for ports 1025-65535 and select the "autoconfigure" option.
For more information, see Using the Database Performance Module.
