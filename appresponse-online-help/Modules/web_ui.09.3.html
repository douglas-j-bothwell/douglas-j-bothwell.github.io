VoIP Alerts
Module for VoIP Performance enhances the existing AppResponse alert functionality with the following features:
Support for defining VoIP-specific alerts in the Desktop Console
Support for viewing detailed alert information in the Module for VoIP Performance web UI
All VoIP alerts issue notifications only when the alert begins and ends and not for periods in between. (In the Alert Manager > Actions page, this type of alert is specified as an on crossing rather than a "continuous" alert.) Because SNMP Trap processors require start and stop conditions to track state, this enables the trap processors to manage these alerts more accurately.
Defining a VoIP Alert
Defining a VoIP alert is similar to defining a static alert, but a VoIP alert definition has some unique information fields. The following steps outline the basic workflow.
Procedure 3 Creating a VoIP Alert

In the Desktop Console, choose Tools > Alert Manager.
2.

4.
Specify the following VoIP-specific information in the following tabbed pages. (For information about other fields, click the Help (?) button in bottom right of the Alert Manager window.)

Alert Lookup Page
The Alert Lookup page (VoIP Data > Alert Lookup) enables you to search for VoIP alerts on an appliance, view high-level information about specific alerts, and drill down into an alert (by viewing the included calls in the Call Lookup page). You must define VoIP alerts in the Desktop Console before you can use Alert Lookup.
To view alert information, follow this workflow:
1)
Go to the Alert Lookup page (choose VoIP Data > Alert Lookup in the Module for VoIP Performance menu).
2)
Display an Alert Table:
–
Click Alert ID to display alerts by ID, or
–
Click Search to display alerts for a specific time period, optionally filtered by various metrics
3)
View details about all alerts in the displayed group or about a specific alert:
–
Alert Info—information about the call that triggered a specific alert
–
Task Comparison—alert frequency and severity for the displayed alerts
Alert ID
The Alert ID link on the Alert Lookup page lets you look up alerts by alert ID. To display an alert table, enter a comma-separated list of one or more alert IDs, then click Go.
Search
The Search link on the Alert Lookup page lets you look up alerts by searching on a time period and some combination of filters.

Looking Up VoIP Alerts
Use Procedure 4 to look up alerts in the Module for VoIP Performance web UI.
Procedure 4 Looking Up VoIP Alerts
1.
Go to the Alert Lookup page:
Choose VoIP Data > Alert Lookup in the Module for VoIP Performance menu.
2.
Specify the time window for the search using the To and From fields (under the Time heading).
3.
(Optional) Specify additional search filters using the fields under the Filters heading.
For more information, see Table 2 Alert Lookup Page: Search Filters.
4.
Click Go.
The Alert Table appears at the bottom of the page, showing the alerts that met the search criteria. For more information, see Table 3 Alert Table Fields.
5.
View information about the alerts using the methods described in Alert Info and Task Comparison.
 
End of Procedure 4

Violation Distribution Bar Chart
(far right)
Each Violation Bar Chart is colored according to the relative number of calls that fall into the four violation categories (non-violating, Minor Violation, Major Violation, Critical Violation).
Each chart has a black vertical bar that indicates the calls that triggered the alert; the calls to the right of this line represent the number of calls in the Violating column. These are the calls that triggered the alert and determined the alert severity.
Alert Info
Clicking on a row in the alert table automatically looks up all calls in the alert interval. The calls are listed on the Call Lookup page, from which you can view details about all calls in the displayed group or about a specific call or call channel:
–
Call Info—detailed information on a single call
–
Signaling—call ladder for a single call
–
Statistic Charts—summary information about all displayed calls
–
Topology—connection diagrams for all calls
Task Comparison
The Task Comparison pane provides a high-level view of the frequency and severity of alerts over the entire time window of the alert lookup.
Figure 9 Task Comparison Pane
 
The comparison graphs uses the following conventions:
Each graph represents the total number of individual alerts for a specific task in the lookup results.
The header to the left of the graph shows the task name, business group, and codecs specified in the alert definition.
The entire time window of the alert lookup is divided into sub-windows. Use the Previous and Next keys to navigate across these sub-windows.
Colored vertical bars represent the violation status for a specific alert: green (non-violating), yellow (minor), orange (major), and red (critical). Only alerts included in the alert lookup results are shown.
To view a distribution graph of all violations of the specified task over the entire alert lookup time window, click the Details link.
