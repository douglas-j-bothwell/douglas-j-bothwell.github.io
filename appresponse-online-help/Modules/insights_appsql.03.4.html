Identify databases and database applications that are contributing significantly to high response times.
Break down the performance for a specific database, database application, or query command by components such as SQL statements, database clients, or tags.

The following procedure describes the suggested workflow for this Insight.
Procedure 3 Database Troubleshooting: Suggested Workflow
1.
Select a DB Server or DB Client in the Top DB Servers or Top Servers Connected to DBs table (top left).
A DB Server is a host that receives SQL data requests; a DB Client is a host that sends SQL data requests directly to a server. A DB client might initiate data requests on its own, or forward requests from other hosts (for example, an application server forwarding requests from its clients).
The DB Instances/DB Applications tables (bottom left) show results for the selected DB Client or DB Server.
The DB Performance charts (top right) show SQL data response times and transaction rates for the selected server or client.
2.
Optional: Select (by dragging horizontally) a time window in one of these charts where database response times are high.
The DB Instances/DB Applications tables (bottom left) and the DB Performance Analysis) are updated with results for the selected time window.
3.
Optional: Select a database, application, or query command in the DB Instance, DB Applications, or Query Commands tab.
The DB Performance Analysis is updated with results for the selected item.
4.
