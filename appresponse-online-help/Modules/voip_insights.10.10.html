VoIP Call Record Lookup
This insight enables you to look up and examine individual voice and video calls as well as the channels for those calls. This interface provides an alternative (and some additional advantages) to the Call Lookup page in the VoIP web UI.
Figure 9 VoIP Call Record Lookup
 
This insight has the following advantages over the VoIP web UI > Call Lookup page:
You can identify and select calls of interest in other insights, then right-click and examine the calls in this insight.
This insight identifies metrics that indicate possible problems (such as high delays, packet loss, or jitter) and shows troubleshooting tips when you select a metric in the bottom table.
You can right-click on selected calls and download the corresponding packets for in-depth analysis in Transaction Analyzer or Wireshark.
You can examine the end-to-end network path for a call of interest: right-click on the call and choose Start AppNetwork Path Analysis. This represents a “bottom-to-top” workflow, in which you identify a problematic call and then examine the network path for potential issues that might be affecting voice or video performance between the callers. (Note—This functionality requires SteelCentral NetSensor.)
Note the following:
You can search on caller names as well as phone numbers in the “Search Phone #” field.
Each table has an Edit Columns toolbar button that enables you to specify the metrics that are shown. If you don’t see this toolbar button, right-click in the table background and choose Show Toolbar.
The following procedure describes the suggested workflow for this Insight.
Procedure 1 VoIP Call Record Lookup: Suggested Workflow
1.
(Optional) Set the Project Time to the desired time range.
The call list (top) updates to show calls during the specified time range.
Note—If this Insight was opened from the Conference Dashboard, the Project Time has no effect. The call list always shows calls in the selected conference.
2.
Expand a call in the call list to see the channels that compose it.
3.
Select a call or channel in the call list, then click a tab (bottom) to see detailed information about that call or channel.
4.
(Optional) Right-click on a call or channel to examine packets or perform further analysis in other Riverbed products.
End of Procedure 1
