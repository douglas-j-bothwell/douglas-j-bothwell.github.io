Lync Deployments
Note the following configuration details for Lync voice and video calling:
AppResponse supports the following Lync features: voice and video calling only. Other Lync features (such as instant messaging and desktop sharing) are not monitored.
Once the SSL key has been installed and the appliance is configured for SIP/TLS, the Lync client must log out and log back in for calls to be decoded by the appliance. The SSL handshake happens at client login and is reused indefinitely. The ARX appliance needs to see that handshake in order to decode the call.
Disable SIP compression: Lync can be configured to use compressed SIP, which Wireshark cannot decode even if it has the SSL key.
As with WTA, the server must not be using Diffie-Hellman encryption ciphers.
