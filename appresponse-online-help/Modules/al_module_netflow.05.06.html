Filtering Directional Flows
You can configure the NetFlow collector to filter out unwanted flows. With direction filtering, any flow matching a specified device, interface, and direction will be excluded from merging to BGs, total traffic, and so on. (Filtering does not affect Interface tables, which will continue to show all flows.)
Direction filtering can help reduce redundant data by excluding duplicate flows. This can be helpful in various situations, such as when
de-duplication is not possible because sampling is being used.
router interfaces must be configured for both ingress and egress.

With no de-duplication, the traffic between IP1 and IP2 will be added twice (representing all four flows). To include only flows 1 and 3 in the calculation of traffic between IP1 and IP2, you can define a filter as follows:
IP_of_router_2 3 0
IP_of_router_1 2 0
This filter will exclude the ingress to interface 3 on router 2 (flow 2) and the ingress to interface 2 on router 1 (flow 4), leaving only one flow per conversation.
The workflow for using direction filtering is
1)
Create a configuration file identifying the flows to be filtered.
2)
Enable direction filtering.
3)
Upload the configuration file to the Appliance.
Direction Filtering Configuration File
The configuration file for direction filtering is a text file in the following format:
# DeviceIP ifIndex Direction (0: ingress, 1: egress)
10.20.30.40 15 0
10.60.89.244 2 1
  .
  .
This file can have any name and contain any number of lines. Lines beginning with # are treated as comments and ignored.
Using Direction Filtering
Use this procedure to enable NetFlow direction filtering.
Procedure 2 Using Direction Filtering
1.
If necessary, use a text editor to create a configuration file specifying the flows to be filtered.
2.
Enable direction filtering and specify the configuration file to use:
2.1.
Log in to the Appliance CLI as an administrator.
2.2.
Enter the following command:
nfconfig TOTALTRAF_EXCL_CONFF <file_name>
where <file_name> is the Direction Filtering Configuration File.
3.
Upload the Device Filtering Configuration file to your Appliance. From a UNIX system, use the command:
cat <file_name> | ssh <account>@<appliance> set_nfdev_filter
where <file_name> is the configuration file, <account> is the name of an account with administrator privileges, and <appliance> is the name or IP address of your ARX Appliance.
