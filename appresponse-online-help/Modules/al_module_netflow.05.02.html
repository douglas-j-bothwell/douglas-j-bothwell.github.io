Setting Up NetFlow Data Collection
The following procedure describes how to set up NetFlow data collection.
Procedure 1 Setting Up NetFlow Data Collection for AppResponse
1.
Add a NetFlow Collection Module license.
For information about obtaining this license, contact your Riverbed sales representative.
After obtaining a license key, open the Desktop Console and enter the license key in the License Manager window (Tools > License Manager).
2.
Set up NetFlow data collection on the AppResponse Appliance.
For more information, see Configuring NetFlow Collection on an AppResponse Appliance.
3.
Configure the devices of interest in your network to export NetFlow statistics.
For each device, do the following:
3.1.
Configure NetFlow as described in Configuring NetFlow on Network Devices.
If you anticipate using NetFlow sampling correction, you should also consider the requirements described in Correcting Sampled NetFlow Data when deciding on a collection strategy.
3.2.
If you are collecting data from a Cisco 65xx or 76xx device that supports Multi-Layer Switching, you must also enter the following commands on the device:
• 
mls flow ip full
• 
mls nde sender version 5
3.3.
Configure the device to send NetFlow data to the IP address/UDP port of the AppResponse Appliance.
By default, the appliance is configured to listen on port 9996. To specify other ports, change the NetFlow Port(s) preference (described in NetFlow Data Collection Options in Administration > System Web UI).
For detailed instructions, see the vendor documentation for your network devices.
4.
(Optional) Configure additional collection options for specific devices or interfaces.
5.
(Optional) If you want to use sampling correction, perform Procedure 6 Setting Up NetFlow Devices for Sampling Correction.
6.
Wait about 30 minutes after you turn on NetFlow data collection, then check the collector to ensure that NetFlow data sent to an AppResponse Appliance is not getting dropped.
End of Procedure 1
 
 
After NetFlow data collection is enabled, the AppResponse Appliance auto-detects all incoming NetFlow data and organizes the data by device, interface, and other grouping criteria.
