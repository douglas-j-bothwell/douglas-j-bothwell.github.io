Configuring NetFlow Collection on Specific Devices and Interfaces
The Device/Interface Manager (accessed by choosing Tools > External Data > Device/Interface Manager) enables you to monitor and configure devices and interfaces for NetFlow data received by the appliance.

Device/Interface Information Table
The Device/Interface Table, shown in Figure 1, shows the devices and interfaces monitored by the NetFlow traffic sent to the appliance. The appliance parses and organizes the NetFlow data by individual device and interface. The table has the following fields:
Name—Name of the device or interface.
When the appliance receives NetFlow data for a new device, it creates a new Device entry and sets the Name attribute to the IP address or (if SNMP polling is turned on) the sysName of the device.
When the appliance receives NetFlow data for a new interface, it creates a new Interface entry and sets the Name attribute to the ifIndex or (if SNMP polling is turned on) the ifName of the interface.
Description—The IP address of the device or Description of the interface.
AppResponse can fill in these fields automatically, based on description fields on the devices themselves. (This feature is turned on by default on new appliances running AppResponse 8.0 or higher only. If you want to enable this functionality on an older appliance, please contact Riverbed Technical Support.)
Interfaces Seen / Available (devices only)—The number of interfaces seen with NetFlow data and the total number of interfaces for the device as SNMP polled from the device. For example, 5 / 15 indicates five interfaces seen with NetFlow-monitored data out of a total of 15 polled by SNMP. If SNMP is disabled, this field is blank.
When the sorting icon is selected in the column header, the rows are sorted by the number of seen devices and then by the total number of interfaces.
Collection Status—Collection status of the device or interface.
For devices, this field shows one of the following:
–
OK—All interfaces for this device have a status of OK.
–
Disabled—Data collection is disabled for this device.
–
Timeout—All interfaces on this device have a status of Timeout.
–
Timeout (n)—The subset of interfaces that have a status of Timeout.
For interfaces, this field shows one of the following:
–
OK—Data collection is enabled for the device, and the Appliance has collected data for this interface within the last timeout interval.
–
Timeout—Data collection is enabled for the device, but the Appliance has not collected data for this interface within the last timeout interval.
Note—The default timeout interval is 24 hours. You can change this on the Collection Status tabbed page of the Device Settings window, as described in Display/Interface Manager Settings.
SNMP Status (devices only)—The current SNMP configuration status on the device. These fields show one of the following:
–
OK—SNMP is enabled and the last SNMP poll of the device was successful.
–
Failed—The last SNMP poll of this device failed due to an authentication error.
–
Timeout—The last SNMP poll of this device timed out.
–
Disabled—SNMP polling is disabled for this device.
Modified by—The last user to modify any configuration settings of the device.
When a device is initially created or subsequently modified, this column shows the user that made those changes.
If SNMP is enabled for this device and the applicable configuration settings have been subsequently modified, this field shows “Auto”.
Last Modified—The date and time stamp of the last modification to the settings of the device.
Last Collection—The date and timestamp of the last data collection for the device or interface
Edit—This field is checked if the device settings have been edited by the user in the current Manager session.
SNMP Initialization (devices only)—Shows the current state of SNMP initialization for the device. This fields shows one of the following:
–
Disabled—No SNMP credentials have been defined.
–
Querying—One or more SNMP credentials have been defined, the device has been discovered, and the appliance is currently trying to connect to the device.
–
Complete—One or more SNMP credentials have been defined, the device has been discovered, the appliance has connected to the device, and the SNMP Status attribute has been set.
Link Speed (In/Out) (interfaces only)—Shows the detected ingress/egress speed on the interface.
You can specify many of these fields using the tabbed work area.
Identify Tabbed Page (Devices)
This page shows the following fields and options for the selected device:
Enable Data Collection—This checkbox is enabled by default when a new device is observed. If you disable this option, the NetFlow Collector will discard any new NetFlow data received from this device and any monitored traffic through any of its interfaces.
Name—The name of the device. By default, this is the sysName (if SNMP is enabled) or the IP address (if SNMP is disabled).
You can specify custom names for NetFlow devices. This can be useful in cases where you want more user-friendly device names to appear in NetFlow tables, reports, and insights. To specify a custom device name, select the device in the table, uncheck the Use Device Name checkbox, and enter the custom name.
You can also rename multiple devices at once using the data import/export feature, as described in Renaming Multiple Devices Using Data Import/Export.
IP Address—The IP address of the NetFlow-enabled device.
Note—If traffic was seen on only one interface of that device, the appliance might set this field to specify that interface only. To specify the entire device, change this field from the interface address to the device address.
SNMP Tabbed Page (Devices)
This page shows the following fields and options for the selected device:
Enable SNMP Polling—Auto-discover and update the various device and interface attributes, including sysName (device name), ifName (interface name), and ifSpeed (physical link speed of the interface).
Polling Interval—The frequency that the SNMP client will query the device for updates to its attributes.
Update with SNMP Defaults—You can create and load predefined templates of SNMP default settings, as described in Saving and Loading SNMP Default Templates.
SNMP Version—The SNMP version used on the device.
The SNMP version and security level determine the additional parameters that are available: security level/name, authentication protocol/password, and so on.
Test—When clicked, the Desktop Console determines whether the appliance can successfully authenticate with the device using the current SNMP settings.
Interface Tabbed Page (Interfaces)
This page shows the following fields and options for the selected interface:
ifIndex—ifIndex value of the interface.
Name—The interface name.
This field is editable only if SNMP is disabled on the device. If you specify a name manually, note the following:
–
To make it easier to search for specific interfaces, it is good practice to include strings that indicate properties such as the link type or speed
–
You cannot have duplicate interface names for the same device.
Description—User-specified description for the selected interface.
Link Layer—The data link layer type. Can be one of the following:
–
WAN
–
ATM
–
Ethernet
–
FDDI
–
Frame Relay
–
Token Ring
–
Unknown (default)
Link Speed
–
Physical—Physical speed of the interface. If SNMP polling is enabled, this field is set automatically to ifSpeed.
–
Inbound—Ingress speed of the interface. If SNMP polling is enabled, this is set to the Link Speed automatically; otherwise it must be set manually. This value is used in the Utilization (Inbound) metrics to normalize the link Throughput (Inbound).
–
Outbound—Egress speed of the interface. If SNMP polling is enabled, this is set to the Link Speed automatically; otherwise it must be set manually. This value is used in the Utilization (Outbound) metrics to normalize the link Throughput (Outbound).
Saving and Loading SNMP Default Templates
You can save templates of SNMP defaults. This makes it easier to enter commonly used settings in the SNMP tabbed page (described in SNMP Tabbed Page (Devices)).
To view or edit an SNMP template, choose Tools > SNMP Defaults. After defining a template, you can apply it in the SNMP tabbed page of the Device/Interface Manager.

SNMP Defaults Table
Identity Tabbed Page

SNMP Defaults Table
The SNMP Defaults table, shown in Figure 2, has the following fields:
Name—Name of the template.
SNMP Version—The SNMP version used on the device.
The SNMP version and security level determine the additional parameters that are available: security level/name, authentication protocol/password, and so on.
Polling Interval—The frequency that the SNMP client will query the device for updates to its device and interface attributes.
Modified by—The last user to modify any default template.
When a template is initially created or subsequently modified, this field shows the user that made those changes.
Last Modified—The date and timestamp of the last modification to the default settings.
Edited—This field is checked if the template has been edited by the user in the current Manager session.
Identity Tabbed Page
This table has the following fields:
Name—Name of the template.
Search Priority—The order of the set of authentication settings to try when initially discovering a new device. A priority of 1 means that the identity will be tried first, 2 that it will be tried second, and so on.
SNMP Tabbed Page
This page has the following options:
Polling Interval—The frequency that the SNMP appliance will query the device for SNMP updates to its device and interface attributes.
SNMP Version—The SNMP version used on the device.
The SNMP version and security level determine the additional parameters that are available: security level/name, authentication protocol/password, and so on.
Display/Interface Manager Settings
You can configure the options and information shown in the Device/Interface Manager, as shown in the following figure.
Figure 3 Configuring Display Settings in the Device/Interface Manager
 
Figure 4 Device Settings Window
 
The Device Settings window has the following options:
Toolbar tabbed page—Show/hide the toolbar; if the toolbar is shown, show/hide the search fields, the “Show/hide columns” button, or the “Restore default columns” button.
Filter tabbed page—Filter only devices with the specified IP addresses or interfaces. You can enter addresses in the field manually; you can also click Add to specify addresses based on address, prefix, or Business Group.
Collection Status tabbed page—Specify the SNMP collection timeout, in hours, for all devices. The default setting is “24” (one day). The minimum timeout is “1”.
