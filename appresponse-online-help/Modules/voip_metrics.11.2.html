Call Metrics
The following table describes all available call metrics.
 
Table 1
VoIP / Video Call Metrics (Updated)
Metric
(Supported Call Types)
New/Updated in Release
Description
Call Id
(Voice and Video)
7.2.3
A unique number assigned by the Monitoring Module to this call
Callee Name
(Voice and Video)
7.2.3
The name of the called party, as transmitted by the signaling session. In the case of SIP calls, the name is extracted from the SIP URI if recognized; otherwise the full URI is used.
Callee Number
(Voice and Video)
7.2.3 / 8.5.5
The phone number of the called party as transmitted by the signaling session. In the case of H323, the Alias is used as the name.
The 8.5.5 patch introduced Alias support in H323.
Caller Name
(Voice and Video)
7.2.3
The name of the calling party, as transmitted by the signaling session. In the case of SIP calls, the name is extracted from the SIP URI if recognized; otherwise the full URI is used.
Caller Number
(Voice and Video)
7.2.3
The phone number of the calling party as transmitted by the signaling session. In the case of H323, the Alias is used as the name.
Channels
(Voice and Video)
7.2.3
The number of RTP media channels associated with a specific call
Codecs
(Voice and Video)
7.2.3
A CSV list of all codecs observed on the RTP streams
Conf ID
(Voice and Video)
8.5.5 Patch
The Conference ID into which this calling party was joined. This value is taken from the signaling used to set up the call. If the caller did not join a conference call, then a N/A is used.
Duration
(Voice and Video)
7.2.3
The period from the first observed RTP packet on any channel related to this call up to the time of the last observed RTP packet on any channel related to this call. If a call is in progress, this value reflects the duration as of the last sample and will contain a suffix of “(Active)”. If a call has no associated RTP channels, this value will represent the period of time the signaling indicated that the call was active, which may include a time out value.
DSCP
(Voice and Video)
7.2.3
A CSV list of the QOS values of all of the RTP streams, formatted in DSCP format.
Jitter
(Voice and Video)
7.2.3
A CSV list of the Jitter calculations for all of the channels associated to the call
Media Type
(Voice and Video)
7.2.3
The Media type as observed in the signaling.
Min/Max Jitter (PPDV)
(Voice and Video)
7.2.3
The minimum and maximum of the average Packet to Packet Delay Variation (PPDV), or jitter, in milliseconds, for the channel reporting a PPDV jitter measurement. This is calculated per RFC 3550.
Min/Max MOS CQ
(Voice)
7.2.3
The minimum and maximum of the Conversational MOS score for all Voice RTP channels attached to this call.
Min/Max MOS Video
(Video)
V8.5.5 Patch
The minimum and maximum of the Video MOS score for all video RTP channels attached to this call.
Min/Max Pkt. Discard Ratio
(Voice and Video)
7.2.3
The minimum and maximum of all packets discarded, as a percentage of all observed packets on all channels associated with the call.
Min/Max Pkt. Loss Ratio
(Voice and Video)
7.2.3 / 8.0.11
The minimum and maximum of lost packets, as a percentage of all observed packets or all channels associated with the call.
In release 7.2.3, this metric was available under the name "Degradation." in release 8.0.11, this metric was renamed to its current name.
Min/Max RT Delay
(Voice and Video)
7.2.3 / 8.0.11
The minimum and maximum delay based on all channels associated to the call. The Delay calculation is compliant with RFC 3550.
In release 7.2.3, this metric was available under the name "Min/Max RTT." in release 8.0.11, this metric was renamed to its current name.
MOS CQ
(Voice)
7.2.3
A CSV list the Conversational MOS for all voice RTP channels associated with the call.
MOS Video
(Voice)
8.5.5 Patch
A CSV list of the video MOS values observed for every video stream associated with the call.
Originator
(Voice and Video)
7.2.3
The IP address of the host that started the phone call. For cases where a call has no RTP streams, this field is blank.
Quality Level
(Voice and Video)
8.0.11
The quality level calculated for the whole call. This is the lowest Quality Level of all voice and video RTP channels attached to the call. The V8.5.5 patch changed calculation to include the MOS Video.
Ringing time (s)
(Voice and Video)
7.2.3
The number of seconds from the point in the signaling protocol where the end phone number is ringing until RTP packets begin to flow. In cases where the signaling is insufficient to generate this number, an N/A is used.
RT Delay
(Voice and Video)
7.2.3 / 8.0.11
A CSV list of all delay values for all of the channels associated to the call. This metric was available under the name "RTT" in 7.2.3 and renamed to its current name in 8.0.11.
Setup Time (ms)
(Voice and Video)
7.2.3
The milliseconds from the from the first observed signaling packet until the RTP packets begin flowing. This value is shown as N/A if the signaling was insufficient to generate the number.
Signaling Type
(Voice and Video)
7.2.3
An acronym for the signaling protocol used to setup the RTP conversation. These values are used: SIP, MGCP-SSCP (means that the MCGP protocol and SCCP were both used to set up the call), SCCP, H323 (includes H245)"
Start Time
(Voice and Video)
7.2.3
The time of the first observed signaling packet. In cases where no signaling is observed, it is the time of the first observed RTP packet.
Status
(Voice and Video)
7.2.3
This field contains a character version of the completion status used by the signaling protocol. If the signaling protocol does not support completion status, this field is blank.
Terminator
(Voice and Video)
7.2.3
The IP address of the host that was targeted by the Originator of the phone call. For cases where a call has no RTP streams this field is blank.
ToS IP Precedence
(Voice and Video)
7.2.3
A CSV list of the QOS values of all of the RTP streams, formatted in TOS format.
