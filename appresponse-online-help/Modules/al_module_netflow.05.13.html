Interface Table
Interface tables let you view the top interfaces based on interface-specific metrics for throughput and utilization. To open an Interface table, choose View > Table > Interface.
A table cell shows the value of one metric (column) for one group (row) calculated over the current Time Selection.
Understanding Interface IP Drill-Downs
The NetFlow Insights menu includes a diagram (Figure 9) meant as a reminder of the terminology used by Riverbed in NetFlow data. The following sections describe this terminology in more detail.
Figure 9 Interface IP Drill-Downs
 
Observed vs. Affiliated IPs
AppResponse uses the terms observed IP and affiliated IP to describe the relationship of a given IP address to a selected interface.
An observed IP is one whose traffic is seen directly by the selected interface.
An affiliated IP is one whose traffic is not seen directly by the selected interface. Instead, its traffic is forwarded to the selected interface from another interface on the same router.
Observed IPs and affiliated IPs appear in the Interface table as parts of various drill-down paths.
Data Directions
For the purpose of interface metrics, AppResponse defines “inbound” and “outbound” with respect to the selected interface. Thus:
An inbound (ingress) packet on the selected interface is one that left the observed IP as an outbound packet and will arrive at the affiliated IP as an inbound packet.
Similarly,
An outbound (egress) packet on the selected interface was an outbound packet on the affiliated IP and will be an inbound packet on the observed IP.
Interface Table Metrics
The interface metrics represent NetFlow data for the selected interface. Inbound and outbound are always with respect to the selected interface, as described in Data Directions.
