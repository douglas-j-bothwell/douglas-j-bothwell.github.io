Managing ToS Groups
The ToS Group Manager is used to create, edit, or delete ToS groups. To access the ToS Group Manager, choose Tools > External Data > ToS Group Manager.
Figure 6 ToS Group Manager
 
The ToS Group Manager displays a list of existing ToS Groups. There are no ToS Groups defined on a freshly-installed appliance.
The ToS Group Manager lets you make multiple changes to the group list and save them all at once. These changes can include creating, deleting, and editing groups. These changes are not saved until you click Apply. Pressing Cancel discards all changes entered since the last Apply and closes the ToS Group Manager.
ToS Group definitions can be imported and exported in CSV format. For more details, see Importing and Exporting Configuration Data.
Creating a ToS Group
Data for new ToS Groups can only be viewed within the AppResponse Desktop Console in time selections after the group was defined. You can view the definitions of any new ToS Groups immediately in the ToS Group Manager, but it might take several minutes before data for a newly-created ToS Group can be viewed.
Procedure 3 Defining a New ToS Group
1.
Starting on the Identity tab (the default tab), click New in the tool bar on the lower left of the ToS Group Manager window. The new ToS Group is displayed and selected in the table at the top of the window. The group attributes can be edited in the lower pane of the window.
2.
Click in the Name field and enter a unique name to identify the ToS Group. The name can include any character except for the following:
• 
comma (,)
• 
forward-slash (/)
• 
close-triangle-bracket (>)
3.
(Optional) Click in the Description field and enter a description of the ToS Group.
4.
Leave the Status of the group as Active. For information on Business Group status, see Group Status.
5.
Click the Definition tab.
Figure 7
ToS Group Manager - Definition Tab
 
6.
In the DSCP Values field, enter a comma-delimited list of individual DSCP values and DSCP ranges.
Errors in the list of DSCP values are indicated by a red font. To view the error list, click “Check ToS Group Definitions and Overlaps”.
Valid formats for an entry are as follows:
• 
DSCP value: #
• 
DSCP range: # - # (where the first value is smaller or equal to the second)
7.
In the Device IPs field, enter a comma-delimited list of the device IPs and IP ranges from which the NetFlow statistics are being exported.
Errors in the list of device IPs are indicated by a red font. To view the error list, click “Check ToS Group Definitions and Overlaps”.
Valid formats for an entry are as follows:
• 
IP Address: #.#.#.#
• 
IP Address Prefix Form: #.#.#.0/24
• 
IP Address Range: #.#.#.# - #.#.#.# (where the first IP is smaller or equal to the second)
8.
(Optional) Click the Add Row button to extend the ToS Group definition with additional DSCP values and devices. For example, you would do this if there are other devices in different network segments whose traffic should also be associated with this ToS Group even though they have different DSCP values assigned.
9.
Click “Check ToS Group Definitions and Overlaps” to make sure that the definition of this ToS Group does not conflict with existing groups. If so, edit the definition to remove the conflict.
10.
(Optional) Repeat steps 1.–9. to define additional ToS Groups.
Note—None of the new groups will be saved until you click Apply or OK. Clicking Cancel erases all changes.
11.
Click OK or Apply to save changes, or Cancel to abort.
End of Procedure 3
 
 
Editing a ToS Group
Follow these instructions to edit an existing ToS Group within the ToS Group Manager.
Procedure 4 Editing a ToS Group
1.
To edit an existing group, select the group name from the list of groups that appears at the top of the window.
2.
Edit the group attributes (Name and Description) in the lower part of the window.
Note—You can edit as many groups as you wish. However, none of the changes will be saved until you click Apply or OK. Clicking Cancel erases all changes.
3.
Click OK or Apply to save changes, or Cancel to abort.
End of Procedure 4
 
 
Deleting a ToS Group
When you delete a ToS Group, the group (and any data associated with it) is no longer available in the AppResponse Desktop Console. You will no longer be able to access any of the historical data for that group from the AppResponse Appliance. As an alternative to deleting the group, you can set the Group Status to “Inactive”, which stops new data collection, but still lets you access historical data.
Procedure 5 Deleting a ToS Group
1.
In the ToS Group Manager, select a ToS Group in the table.
2.
Click the Delete button in the tool bar on the lower left of the ToS Group Manager window.
3.
Click OK or Apply to save changes, or Cancel to abort.
End of Procedure 5
 
 
Group Status
There are certain cases where it is desirable to retain the collected data for a ToS Group that is no longer valid. For instance, suppose you have defined DSCP=12 as “Gold,” but a new policy change redefines DSCP=12 as “Silver.” You might still want to look at the historical data for “Gold” and see what the ToS Group definition was when the data was collected. In this case, you can change the status of the Gold ToS Group to “Inactive” and create a new active group for “Silver.” The AppResponse Appliance will stop collected data for Gold, the existing data will be retained in the database.
Figure 8 Group Status - Identity Tab
 
In addition, IP addresses that belong only to inactive ToS Groups are not considered when selecting the Top IP addresses during the aggregation process.
