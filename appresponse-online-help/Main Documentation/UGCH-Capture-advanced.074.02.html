High-Speed Captures
High-speed captures are especially useful for capturing traffic in high-speed environments such as data centers.

Packet Size Limit
Specify the number of bytes per packet that are stored in the rolling buffer.
The default setting is “unlimited”—that is, to capture all data in each packet. In most cases, you will want to use this setting unless you are measuring very high volumes of traffic or you want to extend the time period for which you retain packets.
You can specify one of the following settings:
•
Unlimited—Retain the entire contents of every packet. Recommended in most cases.
•
768—Retain all protocol header data and some (but not necessarily all) application payload data.
•
512—Retain all protocol header data and some (but not necessarily all) application payload data.
•
256—Retain all protocol header data and some (but not necessarily all) application payload data.
In many cases AppResponse retains enough data to classify the application characteristics of traffic, but you might be unable to perform detailed application-layer analysis in Transaction Analyzer.
•
128—Retain all protocol header data up to and including TCP. There is no guarantee that any application payload data will be retained. This setting is useful only if you want to measure large volumes of traffic and do not want to perform any application-layer analysis.
WARNING—The 128 setting is not recommended if your network uses any form of tunnelling that encapsulates IP packets.
Note—If you are using Database Monitoring Module or VoIP Monitoring Module with packet slicing enabled, you must enable “fullframemode” in the CLI. For more information, see VoIP / Database Monitoring Modules: Enable Full Frame Mode when Packet Slicing is Enabled.
Snapshot Storage
The percentage of total disk space for captured traffic (continuous capture rolling buffer + snapshot archive) that is reserved for Alert Snapshots. Thus, a setting of 10 means that
•
10% is reserved for the snapshot archive.
•
90% is reserved for storing “live” packets as they are seen on the monitoring interfaces.
This field cannot be edited.
Minimum Retention Time
The time, in hours, that the traffic for a snapshot is guaranteed to be available after an individual alert ends. If you specify a retention time of 10 hours, for example, then every snapshot will be retained for at least 10 hours after the alert ends.
The retention time counter starts immediately after the last packet for a snapshot is written to disk.
Maximum Snapshot Rate
The maximum average capture rate for alert snapshots.
AppResponse calculates this rate automatically to ensure that all snapshots are saved. When calculating this rate, AppResponse considers the total available disk space, the Snapshot Storage setting, and the Minimum Retention Time setting.
VoIP / Database Monitoring Modules: Enable Full Frame Mode when Packet Slicing is Enabled
AppResponse has a fullframemode CLI option that is disabled by default. You should enable this option if both of the following conditions are true:
You want to collect data for either the VoIP Monitoring Module and the Database Performance Module, AND
Packet slicing is enabled on the appliance (that is, the Administration > System > Capture page > Packet Size Limit option is set to anything other than "unlimited")
To enable full frame mode, log in to the appliance using the CLI and enter the following command:
cmd> fullframemode enable
Note: Enabling this mode is recommended only if you want to collect data for either of these modules and enable packet slicing at the same time. If this mode is enabled and these conditions no longer apply, go to the CLI and enter fullframemode disable.
