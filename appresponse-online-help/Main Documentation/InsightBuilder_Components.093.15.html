Some Analysis Components can be configured with default settings, and these configuration parameters are described below in the Set Parameter section for each Analysis Component. By using the Configure - Combo Box or Configure - Button Controls in the Toolbar Components, some of those parameters may be exposed to the Insight user as user-configurable parameters, and these are listed at the end of the Set Parameters section for each configurable Component below.
Using the “When Action Received Checkbox”
Almost all configurable Analysis Components have a Start parameter that controls whether the Component sends out Interactions immediately upon startup or waits to receive an Interaction from another Component before doing so. This can be used to determine whether Components that are consumers of Interactions from the Analysis are immediately populated by information or wait until there is a user interaction.
To make the Analysis Component immediately send out Interactions, check When Action Received. This is the default setting.
To make the Analysis Component wait until it receives an Interaction from another Component before sending out Interactions, uncheck When Action Received.
Analysis Aggregation Component
The Analysis Aggregation Component takes analysis results from several Components as an input and outputs the union of all the analysis results. This analysis is used to aggregate the results of multiple Threshold Analyses and pass them to the Severity Indicators in a Single Group Table, as in the Application Dashboard Insight.
The Analysis Aggregation Component has no configurable parameters.

Report Initializer Analysis
The Report Initializer Analysis Component is useful for creating Insight reports whose content is easily configurable. The Report Initializer Analysis specifies the default content in an Insight report: Business Groups, metrics, labels, descriptions, and so on. Report Initializers allow end users to edit the content of an Insight report using a simple interface:
When an Insight report is run as an Insight, a dialog box appears that lets you edit the default content defined in all Report Initializers in the Insight.
You can also edit default content from the Report Manager: the Content tabbed page includes a Configure button for editing the settings of all Report Initializers in the Insight. This makes it easy to generate new reports, with customized content, from existing reports.
This analysis supports the following workflows:
Create Insight reports whose content is easily configurable. You can specify the initial report content—Business Groups, metrics, labels, descriptions, and so on—within the Report Initializer.
Run the same report with different (configurable) content. You can edit the Report Initializer content easily in the Report Manager and generate the same report with different Business Groups, Applications, metrics, and so on.
Report Templates (introduced in release 8.5.3) include Report Initializers with configurable content settings. These templates support a wide variety of reporting workflows.
Without a Report Initializer, the only way to edit Insight report content is to edit the Insight directly in the Insight Development Kit.

The Threshold Analysis Component determines the severity (none, minor, major, critical) of a group path based on a condition that is applied to the group path's average metric values.
