NetShark Command Reference
The following table describes the CLI commands to configure Packet Analyzer capture jobs and to export data to SteelCentral NetProfiler. Enter shark at a CLI prompt to run these commands.
 
Table 1
“NetShark” CLI Command Reference
Command
Description
capture_job index enable
Enables Microflow Indexing on all capture jobs.
capture_job index disable
Disables Microflow Indexing on all capture jobs
capture job list
Returns the current state of the NetShark Module capture jobs.
certificate profiler_export generate
Generate a new self-signed certificate and private key for export to NetProfiler. The certificate can then be viewed using the command certificate profiler_export list and copied/pasted to the NetProfiler web UI.
certificate profiler_export list
List local certificates for NetProfiler export. You can copy/paste a certificate from this list to trusted certificates of SteelCentral NetProfiler (NetProfiler web UI > Configuration > Appliance Security > Encryption Key Management).
certificate profiler_export update
Update a new certificate on the SteelCentral AppResponse appliance. You must then enter the certificate PEM, which must contain both public and private keys. A PEM with multiple lines is acceptable.
certificate profiler_trusted add
Upload a trusted NetProfiler certificate to the SteelCentral AppResponse appliance (create a new certificate if needed). You will be prompted for a certificate ID and the certificate PEM. The certificate ID is any string chosen by the user to name the certificate. You can copy/paste the PEM from the NetProfiler web UI > Configuration > Appliance Security > Encryption Key Management. A PEM with multiple lines is acceptable.
certificate profiler_trusted delete
Delete a trusted NetProfiler certificate. To obtain a list of acceptable certificate IDs, enter certificate profiler_trusted list.
certificate profiler_trusted list
List certificates of trusted Profilers that have been configured on the SteelCentral AppResponse appliance.
exit
Exit from the NetShark CLI console. Typing Ctrl+C has the same effect.
help
Display help for the top commands. For help on a specific command, enter the following: help <command>
history
Display a list of commands that have been executed in the current NetShark CLI session.
log view <num-lines>
View the NetShark server log to troubleshoot potential export problems with export to Profilers. The default number of lines to show is 100.
profiler_export ports configure
Configure adapter port parameters: enable/disable NetProfiler Export, specify BPF filter, etc. To accept the default, press Enter.
profiler_export ports list
List the adapter ports and their configuration.
profiler_export profilers configure
Configure up to two Profilers to receive flow records from the local RPM Integration process. Enter the NetProfiler IP address/hostname for a new NetProfiler, press Enter accept the default NetProfiler, or press Space to remove the default NetProfiler.
profiler_export profilers list
List profilers configured to receive flows from local NetShark.
system start
Start the NetShark process. This must be done to enable export to NetProfiler.
system status
Show NetShark status: started or stopped, running or not, licensed or not. A NetShark in started and not-running state indicates a problem.
system stop
Stop the NetShark process. This must be done to save resources when NetShark is not licensed and no export to NetProfiler has been configured.
system wipe
Stop the NetShark process and wipe all NetShark data from the disk: views, logs, configurations, and certificates. Run this command to clean up disk space.
