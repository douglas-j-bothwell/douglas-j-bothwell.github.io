Installing the Desktop Console
Procedure 1
Installing the Desktop Console
1.
Start a web browser and go to the appliance web interface by opening one of the following URLs:
http://<appliance_hostname>:8080
http://<appliance_ip_address>:8080
This automatically redirects the browser to a secure (SSL) connection on TCP port 8443.
The browser may display the certificate validation popup window. Accept the certificate to proceed to the Login page.
For full support of all features, use Internet Explorer version 5.5 (or later) or Netscape 6.2 (or later) and have Java enabled.
2.
At the Login page, enter the Username given to you by your administrator.
3.
Type the Password for the user account given to you by your administrator.
4.
Click Login.
If a valid username and password are supplied, the main page of the Web Console appears.
5.
In the Web Console, choose Administration > System.
This Administration > System page provides a brief overview of the functionality available within this interface. It also displays the name and access rights of the current user.
Figure 2
Administration > System Web Interface - Main Page
 
6.
On the main page, click the Installation Instructions link near the top of the page and follow the on-screen instructions.
7.
If this is the first time you have launched the Desktop Console, the License Manager dialog comes up and ask for your License Key, as described in Licensing a New Appliance.
End of Procedure 1
 
 
The Desktop Console is now installed on your desktop machine. The application is also automatically launched. You may now log in (see Installing the Desktop Console) and access data collected by the appliance.
If a request to save a .jnlp file is encountered while installing the Desktop Console from the Administration > System page, cancel the download and log off the appliance. This issue can be resolved by restoring the default settings for the browser. To accomplish this task in Internet Explorer, select Internet Options from the Tools menu. Click the Advanced tab and select Restore Defaults.
Note: To successfully run the Desktop Console, you need access to the appliance from your desktop machine through TCP port 22.
Launching the Desktop Console
The Desktop Console is automatically started when it is installed from the Administration > System web interface.
Procedure 2
Launching the Desktop Console
1.
Create and use a desktop shortcut. Java Web Start offers to automatically create a desktop icons and a shortcut in the Start Menu (Microsoft Windows only).
2.
Start the application through the Java Web Start Application Manager (available through the Windows Start menu).
3.
Log in to the web interface and click Launch at the top of the screen, then click AppResponse Console.
The Desktop Console appears.
The Connect dialog box appears.
4.
In the Connect dialog box, enter the following:
• 
Hostname name or IP address of the appliance—such as live.np.com
• 
Username
• 
Password
Instead of Step 1, from within the Desktop Console, you can click the New toolbar icon.
Tip for advanced users: in the Connect dialog box, you can view or modify the SSH port by selecting the “more options” check box. This should only be done if the appliance has also been configured to use a non-default SSH port, as described in Configuring Network Ports.
5.
Click OK. If you supplied a valid username and password, you are logged onto the system. If this is the first time you have used the Desktop Console on this computer, a new project are created (see Projects). Otherwise, the action specified in the Start Action field of the General tab of the Preferences dialog are performed (see General Preferences). The first-time action is the same as the New setting in the Start Action field.
For advanced users, in the Connect dialog box, you can view or modify the SSH port by selecting the “more options” check box. This should only be done if the appliance has also been configured to use a non-default SSH port, as described in Configuring Network Ports.
End of Procedure 2
 
 
Note: Java Web Start ensures that you are always using the latest available version of the Desktop Console. Every time the console is launched, Java Web Start checks the appliance to see if a new version of the console is available. New versions are automatically downloaded and installed.
Before describing the dashboard and the other features available within the console, the remaining sections in this chapter cover essential information that should be understood before using the application.
In general, all the elements of the console have a tool-tip associated with them, which are displayed by placing the mouse pointer over the element in question. Specific tool-tips are described where appropriate.
Projects
All Desktop Console windows are organized into projects in the project display area. A project is a tabbed set of windows that share the same data source and time selection (see Data Source and Time Selection). Multiple projects may be open simultaneously in the Desktop Console.
Figure 3
Projects
 
A project has four visual components (top to bottom):
1)
The project tab, which displays the project name and enables the user to select the project;
2)
The project toolbar, which displays the Select Time toolbar, the Update button, the logged-in user, and the project data source.
To configure the project toolbar, right-click in the toolbar and choose Project Toolbar Options. You can add, remove, and arrange controls such as the Project Time button and the Recent Hour/Day/Week/Month buttons (which provide one-click project time updates).
3)
The project work area, which displays all the non-minimized windows created in the project and allows both vertical and horizontal scrolling.
4)
The window tab area, which displays a named tab for all windows in the work area.
A project can be saved to disk and resumed at a later date. All state from the time the project was saved are restored. Project files can also be shared between machines. See Project Operations for information on working with projects and project files.

The data source for a project may be either an appliance or a director (described in AppResponse Appliance and Domains and Directors).The data source for a project is indicated in two places:
The right side of the project toolbar, along with the ID for the logged-in user.

Appliance Data Source on Project Toolbar
 
As a tool tip visible by placing the mouse pointer over the project tab. The latter makes it easier to work with multiple projects connected to different data sources (see Maintaining Projects).
Figure 5
Appliance Data Source as a Tool Tip
 
In general, a project created for an appliance can only be connected to an appliance, and a project created for a director only connected to a director. Attempting to connect to the wrong kind of data source results in an error message.
You may choose a different data source when opening a project by typing the appliance or director name or IP address into the Connect dialog (see Procedure 2). To change the data source of an open project, select that project’s tab, click Disconnect in the File menu of the Desktop Console, then click Connect to open the Connect dialog.
Time Selection
Time selection is the time interval for data analysis for all windows created in a project: that is, all windows created in a project display data from that time interval. If the time selection is changed, all windows created in the same project are updated to reflect the new time interval.
There are three ways to change the time selection.

The project toolbar displays the time selection for the project. The time selection can be changed directly on the toolbar by entering a new time interval and clicking Update Now.

During the process, the Update Now button is changed to a Cancel button. Press the Cancel button to terminate data retrieval if you would like to change the time selection or if the operation takes too long.

Time Selection Tool
The time selection tool provides the most flexible way to change the time interval. Click the Select Time button on the toolbar to invoke the time selection tool.
Figure 10
Time Selection Dialog Box
 
The time selection tool displays the following information:
Appliance Time—Displays the time on the appliance.
Client Time—Displays the time on the desktop machine running the Desktop Console. It also indicates whether the desktop and NP times are reasonably in sync, or shows the number of minutes that they are out of sync.
Calendar—Use the calendar to select a time interval. The selection within the calendar depends on the Interval, which is set to the right of the calendar.
Interval—Determines whether calendar selections are made by month, week, day or hour.
Time Range—The Time Range field allows you to input a specific time interval.
Shortcuts—Click the appropriate button to set the time interval to a recent time period. The current time is used for the end time.
Automatic Update—Check this option to automatically update the windows within the project at the time intervals (measured in minutes) you specify in the Real Time Every field. The minimum interval is 1 minute. An hourglass appears in the toolbar showing how much time is left before all view are automatically updated.
The data displayed in all open views within a project are updated every time the project time selection is changed. Keep this in mind when using automatic update. If you have open views containing data that should not change, consider exporting the data or taking a snapshot of the view before enabling automatic update. These topics are described in Exporting Data from Tables, Charts and Snapshots.
Using the Calendar
The calendar shows a list of dates that may be selected by a mouse click. The Interval determines what kind of time range gets selected. If the interval is set to Hour or Day, then a single date may be selected. If the interval is set to Week or Month, the calendar selection changes so that the whole week (Sun-Sat) or month is selected when a particular date in that week or month is clicked.
All dates in the calendar are selectable unless there is no data available. If no data was recorded on a given day, that day is greyed out in the calendar. Thus, dates in the future are not selectable. The Recent Hour selection sets the time interval to include the last 60 minutes. If the data is available up until the current time, then the Recent Hour is the last hour before the current time.
Project Operations
A project can be saved to disk and resumed at a later date. All state from the time the project was saved are restored. The following sections cover the procedures involving with saving, opening and managing project files.
Creating a New Project
The first time the application runs, a new project is automatically created. Alternatively, a new project can be created if the Desktop Console is already running. Multiple different projects can exist within the Desktop Console simultaneously. This allows you to examine data from multiple data sources (multiple appliances) or data from different time intervals on a single appliance. In general, a single project is used for each task (e.g. troubleshooting a server problem). Simultaneous projects allow you to diagnose multiple problems and events at once.

Project Properties
Selecting Project Properties opens the Project Properties dialog.
Figure 15
Project Properties Dialog
 
The Project Properties dialog displays the name of the project, its file system location, and its time interval. Checking the Use Recent Interval check box changes the behavior of the project the next time it is opened: it maintains the same time range (in this case, one hour) but sets the end time to the most recent time when next opened. This is useful for creating a project that you can use to focus on a specific type of problem or set of resources whenever you open it without having to reset the time.
Maintaining Projects
After you start the Desktop Console, on the File menu, click Disconnect command to disconnect the project from the appliance. Likewise, click the Connect command to re-establish a connection to a appliance. This can be useful if you are using a project to examine data on an appliance and wish to see the same information from a second appliance. To do so, disconnect the project from the first appliance and connect it to the second appliance. Alternatively, you can save the project file under two different file names, open the files and connect each to a different appliance. This allows you to compare the data displayed in the project against both appliances.
Customizing the Project Toolbar
You can configure the toolbar in each project window. The Customize Toolbar dialog box enables you to add, remove, and arrange controls such as the Project Time button and the Interval Selector—that is, the project start and end time—fields. You can also include Recent Hour/Day/Week/Month buttons in the project toolbar, which provide one-click project time updates. (The Recent Hour button is included in the toolbar by default.)
To configure the project toolbar, right-click in the toolbar and choose Project Toolbar Options.

Window Right-Click Menu
Right-clicking on the top of a window, or on the window icon in the control bar, opens a menu that allows you to restore, minimize, or maximize the window, as well as to select the Global Window option and rename the window.
Figure 19
Window Right-Click Menu
 
“Global Window” Option
Checking the Global Window option makes that window appear in all open projects. Its time interval is derived from the parent project: the one in which the window was created. If the time interval of the parent project is changed, the time interval of the global window changes across all projects. If the Global Window option is unchecked in any project, the window can again be seen only in the parent project.
“Always on Top” Option
All Desktop Console windows include an Always on Top option that ensures that a window never gets hidden, regardless of the window that is currently selected. This option is especially useful for drag-and-drop operations to destination windows such as charts and Favorites tables: you can navigate to any number of source windows and the destination window is always available.
To enable this option, right-click in the title bar of the window and select the Always on Top checkbox.
“Rename Window” Operation
Clicking on the Rename menu item opens the Rename Window dialog box. Type a name in the entry field and click OK to rename the window, which makes for easier identification of multiple windows.

DNS Lookup Preferences
The Desktop Console can use the Domain Name System (DNS) to look up the hostname of each IP address displayed. The hostname is displayed within the Information column in tables. Click the DNS Lookup tab of the preferences window to enable or disable DNS and configure new DNS servers. All DNS lookups are issued from the desktop machine running the console.
This tab has the following preferences:
Lookup Method—This preference determines how DNS lookups are performed by the Desktop Console. It can be set to one of four possible values:
–
None—Disable all DNS hostname lookups.
–
Sequential—Issue DNS lookups for each IP address serially.
–
Parallel—Issue DNS lookups for all IP addresses at the same time. This option is faster than Sequential, but results in a sudden burst of requests to the DNS server.
–
Custom—Issue DNS lookups to the DNS servers listed in the Custom Servers field in addition to the operating-system DNS servers. In some environments, an additional DNS server is required to perform external name resolution. In this mode, DNS lookup requests are issued in parallel.
Try All Servers—If this option is selected, the Desktop Console queries all defined DNS servers in an attempt to resolve the IP address of a hostname even if a server has already returned an authoritative hostname/domain name not found message. The Desktop Console stops issuing DNS queries when an authoritative domain name is returned. Again, this may be useful in environment with split internal and external DNS servers.
Custom Servers—A list of DNS servers used by the Custom lookup method. These DNS servers should be entered as a list of IP addresses separated by comma or whitespace.
Metrics Preferences
In the AppResponse OS, many metric definitions have two components: the data stored in the database, and calculations made in the Desktop Console using that data.
To support the display of deprecated metrics from an upgraded appliance, check the Show Deprecated Metrics checkbox in the Metrics Preference tab of the Preferences dialog. The default is unchecked: deprecated metrics are not displayed.
Queries Preference
This tab has the following preference:
Always Query 1-min Granularity Data in Tables—When this preference is selected, all table-based queries access the one-minute traffic database to retrieve the top elements to display in the table. This results is more precise data queries and topping results, especially for smaller elements such as individual IPs and IP conversations. The trade-off is that table queries can take longer. (For this reason, the preference is turned off by default and is not invoked if the project time is longer than five hours.)
Insights Preferences
This tab has the following preferences:
Local Insight Directory—If you connect to the AppResponse Update Center directly from the appliance, you can download new or updated insights from the Riverbed support site and place them in this local directory. Insights stored in this directory are available through the Tools menu. Click Change to enter a new directory path.
Automatic Update Server—This is set to the AppResponse Update Center by default. Enter a new URL in this field to change the update server. (This option does not apply to AppResponse Standard).
Note—If your organization uses a proxy server, you must specify the proxy server settings used by Java. The user interface for specifying the proxy server used by Java is platform-specific. For more information, see the documentation for your specific platform.
Check for updates automatically—If this option is selected, the Desktop Console checks for updates from the automatic update server specified here whenever you connect to the appliance, and notifies you if updates are available. For more information, see Automatic Update Checking.
Include supported modules in the Insight Index—If this option is selected, the Insights Index window shows insights associated with any AppResponse modules (VoIP Monitoring Module or NetFlow Data Collection) that have licenses installed on the appliance.
Transaction Analyzer Preferences
The following preferences enable you to configure how AppResponse invokes Transaction Analyzer (for example, when you right-click on an IP conversation and choose Send To > Transaction Analyzer). You can invoke Transaction Analyzer automatically using one of three modes:
Use Registry (run Transaction Analyzer in "default" mode)
Run Transaction Analyzer in “default” mode by running the executable associated with Transaction Analyzer capture files in the OS registry on the local host.
Select Application
Invoke a specific Transaction Analyzer solution and executable. This option is useful when you have multiple Riverbed solutions and/or releases installed and you want to invoke a specific solution/release which might not be the default as defined in the OS registry.
Custom
Invoke the Transaction Analyzer solution using a text command as entered at a command prompt. This is an advanced option that is useful for users who want to invoke a specific Transaction Analyzer executable with a custom set of command-line arguments.
Note the following:
–
The command line string should not include any newline or break characters.
–
Before you enter a command in this field, it is good practice to verify that the command works as intended by entering and running the full command (with all arguments) from a command prompt.
Wireshark Preferences
You can specify one of two modes for invoking Wireshark automatically:
Use Registry (run Wireshark in "default" mode)
Run Wireshark in “default” mode by running the Wireshark executable associated with capture files in the OS registry on the local host.
Select Application
Invoke a specific Wireshark solution and executable. This option is useful when you have Wireshark releases installed and you want to invoke a specific release which might not be the default as defined in the OS registry.
