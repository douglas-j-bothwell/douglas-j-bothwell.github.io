Packet Download Manager
The Packet Download Manager (Tools > Packet Download Manager) enables you to
Preview captured packets from an individual appliance, or from multiple appliances in a domain (when connected to a director)
The Packet Download Manager supports both IPv4 and IPv6 packets.
Select packets based on time intervals and IP conversation pairs
Send packets to Transaction Analyzer or Wireshark for in-depth analysis
Download and save the selected packets to a capture file for further analysis

Workflow Description
The following procedure describes the recommended workflow.
Procedure 5
Previewing and Downloading Packets Using the Packet Download Manager
1.
(web UI) Navigate to the Administration > System > Capture page and verify that the packet capture process is running.
2.
(Desktop Console) If you know the specific time window for which you want to download packets, set the Project Time to match this window.
3.
(Desktop Console) Select an object that includes the packets that you want to preview and analyze.
You can preview and download captured packets for the following objects in the Desktop Console:
• 
IPs
• 
IP Conversations
• 
Business Groups
• 
Alerts
• 
All packets in the rolling buffer
Note: It is good practice to identify and select the packets of interest in the Desktop Console before you preview packets. This reduces the amount of irrelevant traffic in the Packet Download Manager.
If you want to analyze packets in Transaction Analyzer, you will get best results if you download all packets for one user-level transaction (such as a single file or web page download), from the first request packet to the last response packet.
4.
(Desktop Console) Preview the traffic of interest in the Packet Download Manager.
The method for specifying the initial preview data traffic depends on the selected object(s):
• 
IPs, IP Conversations, and Business Groups:
   • Select the objects of interest in a Table, Insight, or Topology window.
   • Right-click and choose Send To > Packet Download Manager.
• 
An Alert episode:
   • Open the Dashboard (View > Dashboard)
   • Click View Summary for an episode of interest
   • Click the “View Packet Download Manager” link
• 
All packets and/or set of IP addresses:
   • Choose Tools > Packet Download Manager.
The Packet Download Manager opens with a preview of the traffic you selected. For information about using this window, see Packet Download Manager.
5.
(Packet Download Manager) If the “Specify Traffic to Preview” Dialog Box is not visible, click Edit Preview.
6.
(Specify Traffic to Preview dialog box) Edit the time window, tiers, and tier pairs that you want to preview in the Packet Download Manager. Your goal is to filter out as much irrelevant traffic as possible.
For more information, see Edit Preview.
7.
(Packet Download Manager) To filter out additional traffic by time, drag horizontally on the time window of interest in the throughput graph.
Figure 13
 Selecting a Time Range in the Throughput Graph
 
8.
(Packet Download Manager) Select the tiers/tier pairs for the packets of interest in the Tier Pair Circle, Tier Pair Table, or Tier Table.
9.
(Packet Download Manager) Check Download buttons for the amount of capture data currently selected. Filter additional traffic if necessary.
10.
(Packet Download Manager) Download the packets of interest by doing one of the following:
• 
To download all packets for the current time window, click Download (<traffic_size>).
• 
To download only the selected packets, click Download Selected (<traffic_size>).
11.
(Specify Download Options dialog box) Specify the options for downloading the captured packets:
• 
Capture file name and directory
• 
Action (save capture file only, open in a specific Transaction Analyzer window, or in Wireshark)
• 
Filter captured packets
• 
Packet slicing (save all bytes, first X bytes, or headers only)
For more information, see Specifying Download Options.
End of Procedure 5
 
 
Packet Download Manager
The Packet Download Manager provides a preview of all captured packets on the appliance. You use this window to navigate the preview data, select the packets you want to analyze, and download these packets to your local machine.
Figure 14
Packet Download Manager
 
The following steps outline the general workflow for previewing and downloading packets:
1)
Specify the traffic to preview (the time window and the specific IP conversations of interest in that time window).
You can query up to 1000 conversations; the preview shows traffic for the top X conversations (by throughput) for the specified time window.
All other captured traffic is grouped as “OTHER.” OTHER specifies traffic for all IP conversations that do not meet the topping threshold for the specified time window. You cannot download OTHER traffic.
All non-IP traffic that was dropped is grouped as “Non-IP Traffic.”
2)
Select the packets you want to download.
You can use Ctrl-click to select IP conversations of interest; you can also drag horizontally in the time graph to narrow the time window of interest.
This step is necessary only if the current preview includes packets that you do not want to download.
3)
Download the packets of interest.
–
To download all packets in the preview, click Download.
–
To download only selected packets, click Download Selected.
You cannot select and download packets specified as OTHER. If OTHER traffic is selected, the Download Selected button is unavailable. The only way to download OTHER traffic is to click Download.

Specifying Download Options
This dialog box appears after you click Download Selected or Download in the Packet Download Manager, and prompts you for the following information:
The capture filename and directory
Packet filter for excluding irrelevant packets
Packet slicing options
The action to take:
–
Download Packets (that is, simply save the capture file)
–
Analyze in Transaction Analyzer Trace Explorer
When you save in Trace Explorer, Transaction Analyzer saves the downloaded packets in a trace file (<filename>@<timestamp>@<appliance>.appcapture).
–
Analyze in Transaction Analyzer (Treeview or AppDoctor)
When you open packets in Treeview or AppDoctor, Transaction Analyzer saves both the trace file (<filename>@<timestamp>@<appliance>.appcapture) and the Transaction Analyzer file (<filename>.atc.m) in the same directory.
NOTE—Transaction Analyzer does not retain downloaded packets on the local host unless you save the resulting file in Trace Explorer or Transaction Analyzer. If you close either window without saving, the downloaded packets will be discarded on the local host. (If you want to view these packets again, you will need to download them again before they are overwritten in the appliance buffer.)
NOTE—If is good practice to open packets in the Treeview or AppDoctor window only if the capture file contains packets for one user-level transaction, with as few irrelevant packets as possible. If the file contains multiple transactions or is very large, you should open it in the Trace Explorer window instead.
See also Preferences for Launching Transaction Analyzer
–
Send packets to WireShark (if WireShark but not Transaction Analyzer is installed locally). For more information, see Wireshark Integration.
