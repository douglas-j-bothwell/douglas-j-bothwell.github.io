NetProfiler: SteelCentral AppResponse Flow Export to NetProfiler
RPM Integration includes a netshark CLI program for configuring the appliance to export flow data to NetProfiler. The following sections describe how to use this program.

Configuring NetProfiler Export: Workflow Description
The following procedure describes the standard workflow for configuring NetProfiler export on an appliance.
Procedure 3
To configure NetProfiler on an SteelCentral AppResponse appliance:
1.
It is good practice to synchronize the NetProfiler and the SteelCentral AppResponse appliances to an accurate NTP time server. To synchronize the appliances, do the following:
Specify an external NTP server for SteelCentral NetProfiler:
1.1.
Log in to the NetProfiler web UI.
1.2.
Navigate to the Configuration > General Settings page and check the “Synchronize to an external NTP server” option (under Time Configuration).
1.3.
If this option is not selected, select it and specify an NTP server IP.
1.4.
Click Configure Now (at the bottom of the page) and wait for the appliance to reboot.
Specify an external NTP server for SteelCentral AppResponse
1.5.
Log in to the SteelCentral AppResponse web UI.
1.6.
Navigate to the System > Setup page and check the NTP Servers field.
1.7.
If this field does not include the same NTP server as the NetProfiler appliance, add the NTP Server IP.
1.8.
Click Apply (at the bottom of the page) and wait for the appliances to reboot.
2.
Log in the appliance as a user with Administrator privileges, using an SSH-enabled program such as putty.
3.
Enter the following command at the CLI prompt: shark
4.
Run the following command: system status
5.
If the system status is not running, enter the following command: system start
6.
(Optional) Set up security certificates on the AppResponse appliance and the NetProfiler.
Note: NetProfiler is pre-installed with a set of default certificates. You need to do this step only if you do not want to use the default certificates. The CLI provides the following operations for setting up certificates:
• 
certificate profiler_export generate
• 
certificate profiler_export list
• 
certificate profiler_export update
• 
certificate profiler_trusted add
• 
certificate profiler_trusted delete
• 
certificate profiler_trusted list
For detailed descriptions of these commands, see NetShark Command Reference.
6.1.
Generate a new certificate on the appliance. In the shark CLI, enter the following command: certificate profiler_export generate
6.2.
Enter the certificate information requested in the CLI: email, organization, unit, and so on.
6.3.
Enter the following command at the CLI: certificate profiler_export list
6.4.
Go to the NetProfiler web UI > Configuration > Appliance Security > Encryption Key Management page.
6.5.
In the Trusted Certificates tabbed page, click New Certificate.
6.6.
Copy the certificate string (including the -----BEGIN CERTIFICATE----- and -----END CERTIFICATE-----delineators) from the shark CLI to the Add New Certificate page, enter a description for the certificate, and click OK.
6.7.
In the Local Credentials tabbed page, navigate to the certificate you want to install on the appliance. In the pull-down appliance, click Download Certificate and save the PEM file on your local host.
6.8.
In the shark CLI, enter the following command: certificate profiler_trusted add
6.9.
When prompted, enter the certificate ID and the PEM.
7.
Specify the target NetProfiler(s) for the AppResponse data:
7.1.
At the shark CLI, enter the following command:
profiler_export profilers configure
7.2.
This command prompts you for two Profilers. You can enter an IP or a DNS name. If you are specifying one NetProfiler only, press the Space key at the second prompt.
8.
Configure the adapter ports used to export the NetProfiler data.
8.1.
Enter the following command:
profiler_export ports configure
8.2.
For each interface under the ARX probe, the CLI prompts you to specify the following options. To accept the default setting, press Enter:
• 
Enable/disable flow export
• 
Enable/disable VoIP data
• 
Specify BPF filter
9.
Verify the connection between the SteelCentral AppResponse appliance and the NetProfiler:
9.1.
Wait 20 minutes and then log in to the NetProfiler web UI.
9.2.
Navigate to the System > Devices/Interfaces page and examine the Devices & Interfaces treeview.
If the SteelCentral AppResponse appliance and interfaces you specified appear in the treeview, the configuration was successful and the SteelCentral NetProfiler appliance is able to receive traffic flow records from SteelCentral AppResponse.
Note—This release of RPM Integration has a known issue in which SteelCentral AppResponse devices and interfaces appear with yellow lights (Device clock is out of sync) even if their status is OK. This issue will be fixed in a future release of RPM Integration.
