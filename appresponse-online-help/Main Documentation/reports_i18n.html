Internationalization (I18n) Functionality
Various internationalization enhancements have been added to AppResponse over the past few releases. Since Release 7.1.6, I18n support has been extended to Japanese and Korean in addition to Simplified Chinese. This section describes the officially supported I18n functionality in Release 9.5.
Note: This functionality requires that you run the console on a Chinese, Japanese, or Korean version of Windows.

Generating Internationalized Reports
To generate an internationalized report, go to the System page of the web UI and set the Locale pull-down menu to the desired language. With this setting, generated reports can use the selected language to display the UI elements specified in the previous table.

Viewing Reports in Simplified Chinese
It is recommended that you use Windows Notepad to view any exported text files (generated using File > Export in the Desktop Console) that contain Japanese, Korean, or simplified Chinese characters. As of publication of these Release Notes, Microsoft Excel and Wordpad are not compatible with the UTF-8 encoding format used to display these languages.
