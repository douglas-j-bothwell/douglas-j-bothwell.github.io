Chart Types
The following chart types are available:
Multi-Group Time Series Chart
This chart is useful for viewing and comparing one metric (for one or more groups) as it varies over time.
Multi-Metric Time Series Chart
This chart is useful for viewing and comparing multiple metrics for one group as they vary over time. You can include metrics with different units in the same chart, but you can show metrics for one group only in the same chart.
You can also specify a horizontal threshold marker on the Y axis: right-click in the chart (at the vertical level where you want to specify the threshold) and then choose Set Y-Axis Marker.
Multi-GroupMetric Time Series Chart
The Multi-GroupMetric Time Series Chart supports multiple groups and multiple metrics in the same chart. This chart type extends the functionality of the Multi Group Time Series Chart (multiple groups, one metric only) and the Multi Metric Time Series Charts (multiple metrics, one group only).
These charts are useful for
–
Viewing different metrics for multiple Defined Applications that work together as a multi-tier, end-to-end application—for example, a chart that shows page times for a front-end Web Application and server response times for a back-end Server Application.
–
Viewing the same metric from two different Business Groups with opposite perspectives, such as client vs. server or inbound vs. outbound.
For example, suppose you have two Business Groups, Remote Office and Data Center, that exchange traffic and you want to compare the server response times from both a specific (Remote Office) and an overall (Data Center) perspective. You can create a chart with two GroupMetric time series:
Remote Office / Server Response Time (Clients))
Data Center / Server Response Time (Servers))
Correlation Chart
This chart is useful when you want to illustrate a possible correlation between two metrics. Each axis represents a metric, so you can determine if one metric varies in direct correlation with another.
For example, User Response Time is often correlated to the rate of incoming Connection Requests. You can use a Correlation Chart to determine how many Connection Requests a server can handle before the User Response Time becomes unacceptable.
Distribution Chart
A Distribution Chart shows the frequency with which a metric takes a specific value. The bars displayed in the distribution chart show the percentage of time over the time selection at which the metric charted has the value listed on the X Axis.
Cumulative Chart
The Cumulative Chart shows the cumulative distribution of a metric value for one or more groups. The chart shows the percentage of time a metric takes a value lower than indicated on the X Axis.
Pie Chart
A pie chart displays the relative frequency or value of a metric between different groups. To export a group to a Multi-Group Time Series Chart, double-click on a pie slice.
Bar Chart
A bar chart displays the relative frequency or value of a metric between different groups.
Response Time Composition Chart
The Response Time Composition Chart shows a stacked region chart with key metrics that contribute to end-to-end response time. This chart makes it easy to identify at a glance which metrics are responsible for response time issues.
Top Charts
Top Charts show the top groups for a specific metric. These differ from other charts in that the groups shown in these charts are selected automatically by AppResponse based on topping criteria specified in the chart itself.
