Riverbed NetFlow Insights
Riverbed NetFlow Insights enable you to
Monitor WAN throughput by application on individual Riverbed devices.
View detailed throughput and compression metrics for individual applications
Analyze passthrough, optimized, and total WAN throughputs and WAN/LAN compression metrics for individual applications.
Analyze applications and IPs that are affected the most (and the least) from Riverbed optimization.

System Requirements
These Insights require the following:
AppResponse release 8.0.8 or higher
A NetFlow Data Collection license
A Riverbed Steelhead appliance with an RIOS release that can export flow data in NetFlow v9 format
Configuration and Setup
These Insights require that each Steelhead be configured as follows:
The Steelhead has a Flow Collector that is configured as follows:
–
Exports data to the AppResponse appliance in Netflow v9 format.
–
Exports data for at least one LAN and one WAN interface.
–
For each interface, the Flow Collector exports data for both Optimized and Passthrough traffic.
–
The Active Flow Timeout is set to 60 seconds.
–
The Inactive Flow Timeout is set to 60 seconds or less.
The Steelhead has SNMP traps enabled and a Read-Only Community string specified.
The following procedure describes how to configure a Steelhead appliance using the RIOS v6.0 web interface.
Procedure 1
Configuring a Steelhead for AppResponse Riverbed NetFlow Insights
Note: This procedure is based on the web interface for RIOS version 6.0.2. Some menus, options and UI elements might be different if you are using a different version of RIOS.
1.
Log in to the web interface of the Steelhead.
2.
Choose Configure > Networking > Flow Export.
3.
Under Flow Export and Top Talker Settings, configure the options as follows:
• 
Enable Flow Export = selected
• 
Disable Top Talkers = selected
• 
Active Flow Timeout = 60 seconds
• 
Inactive Flow Timeout = 15 seconds
Figure 1
Flow Export/Top Talker Settings
 
4.
Under Flow Collectors, create a collector for the AppResponse appliance:
4.1.
Specify the IP address of the appliance, and the port on which the appliance listens for Netflow data.
Note—The collector port number must be included in the Netflow Port(s) field in the Administration > System > Advanced page of the AppResponse web UI.
4.2.
Set the Version to Netflow v9.
4.3.
For each LAN and WAN interface of interest, set the pull-down menu to All.
4.4.
For each RIOS interface, set the pull-down menu to None.
Figure 2
Flow Collector Settings
 
5.
Choose Configure > System Settings > SNMP Basic.
6.
Configure the following settings:
• 
Enable SNMP Traps = selected
• 
Read-Only Community String = a string that enables AppResponse to read data from the Steelhead.
Figure 3
SNMP Server Settings
 
End of Procedure 1
 
 
WAN Usage Insight
This Insight is useful for viewing the overall WAN throughput and utilization in your network, and for identifying the interfaces and applications used the WAN most heavily during the current project time window. Note the following:
The Riverbed WAN interfaces table supports many right-click menu operations available in other tables.
You can double-click and drag-and-drop slices in the Top Applications pie chart to export data to external charts and tables.
You can zoom in on a specific time window in the graph (right) to view the top applications for this window in the pie chart.

WAN Performance Insight
This Insight is useful for analyzing the WAN acceleration performance on a specific interface. This Insight shows interface throughput levels and traffic reduction over time, identifies top applications based WAN throughputs (overall, optimized, and pass-through), and shows the time-based effects of WAN acceleration on top applications.
Figure 5
WAN Performance Insight
 
In this Insight, you can
View time-based throughputs and LAN-to-WAN throughput reduction per Steelhead interface.
The Data Reduction graph shows two measurements of data reduction:
–
Optimized Traffic Data Reduction—The percentage by which LAN throughput (optimizable) is reduced to WAN throughput (optimized). If the Data Reduction graph shows 10%, for example, the optimized WAN throughput is 90% of the non-passthrough LAN throughput.
–
Overall Traffic Data Reduction—The percentage by which total LAN throughput (optimizable and pass-through) is reduced to total WAN throughput (optimized and pass-through).
Identify top applications by WAN throughput (total, optimized, and pass-through) on a selected interface. You can also view top applications and application-specific throughputs for a subset of time by dragging horizontally in the top right graph.
View time-based throughputs and compression ratios per application.

Customize, edit, export, and analyze data for selected interfaces, throughputs, and applications via right-click menus.

This Insight is useful for troubleshooting WAN acceleration performance on a specific interface.
