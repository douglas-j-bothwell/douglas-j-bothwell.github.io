Report Manager
The Report Manager (Tools > Reporting > Report Manager) provides a central interface for defining, scheduling, and editing report definitions. The following topics are discussed:
Report Manager Window
Report Parameters
Define Content for a New Report
Report Manager Window
To manage reports, do one of the following:
From the Desktop Console, choose Tools > Reporting > Report Manager
From the Business Report Manager, choose Tools > Reports > Report Manager
The Report Manager window appears, as shown in the following figure. To create a new report, click New and define the Report Parameters.
Figure 1
Report Manager Window
 
Report Parameters
The following table describes the parameters in the Report Manager.
 
Table 1
Report Parameters
Tab
Parameter
Description
Identity
Name
Enter a name that clearly describes the report content. This makes it easier to find the report in the web UI > Reports page.
Description
Enter a description that highlights the unique attributes of the report. This makes it easier to find the report later in the Report Manager.
Type
The type of report: Standard or Business. This determines the scheduling options for the report, as described in Schedule (Standard Reports) and Schedule (Business Reports).
Category
The category under which the report entry is listed in the Report Manager. Enter a new category or select one in the list.
Access
•
Public—Users must be logged in to the device to view published reports
•
Private—Users can view published reports via URLs without logging in to the device
Status
•
Active—The device publishes new reports based on the report schedule.
•
Inactive—New report publication is suspended.
You can have up to 200 active reports on an appliance and up to 400 on a director.
Content
 
See Define Content for a New Report.
Action
Generate image file
Publish a GIF, JPEG, or PNG report.
Generate data file
Publish a CSV or TSV report.
Send reports by email
Whenever a report is published, send email notifications and reports to the specified addresses. Click the ‘i’ button to view invalid-email-syntax errors.
Schedule (Standard Reports)
Initial Report Date
Initial Report Time
Start publishing reports at this date/time.
Report Time Selection
Publish a report for the time window ending when the report is published.
Repeat Interval
Repeat every specified time interval (if enabled).
On-Demand Reports
When selected, each web page for this report includes a "run on demand" link in the top-right corner of the page.
Note—You cannot generate an on-demand report unless the appliance has report data for the entire time window. For example, you cannot generate an on-demand report at 2:00 pm for today because the appliance does not have data for the entire time window (12:00 am - 11:59 pm).
Schedule (Business Reports)
Frequency
Daily, Weekly, or Monthly
Business Hours
Select a defined profile, or click the button to open the Business Hours Manager.
Time Zone
Time zone used to schedule business reports
Start Date
The date when the appliance starts publishing Business Reports
On-Demand Reports
Select the checkbox if you want to enable users to generate on-demand reports using this Business Report definition.
Note—You cannot generate an on-demand report unless the appliance has report data for the entire time window. For example, you cannot generate an on-demand report at 2:00 pm for today because the appliance does not have data for the entire time window (12:00 am - 11:59 pm).
Define Content for a New Report
Procedure 1
Defining Content for a New Report
1.
In the Desktop Console, open the Report Manager (Tools > Reporting > Report Manager).
2.
Click New. The Edit Report Content dialog box prompts you to select the report type.
Figure 2
Specifying the Report Type
 
3.
Select the report type. A blank window of that type appears in the console.
Figure 3
Report Manager - Edit Report Content Window
 
4.
Identify the groups and metrics to be displayed in the report.
The blank window includes all the same controls available in a regular view. Use the window controls in the same manner that you would use the controls of a regular view. For instance, you can drag-and-drop traffic groups from a table to add them to a report.
Note: It is good practice to include the Information Panel in all reports. This includes useful data, such as the time selection. It can also be used to add an arbitrary note to the report.
5.
Specify the image height (up to 1200 pixels) and width (up to 1600 pixels).
Figure 4
Report Manager - Content Pane Size Control
 
6.
Click OK to save the report content and return to the Report Manager.
Figure 5
Report Manager - Content Tab
 
End of Procedure 1
