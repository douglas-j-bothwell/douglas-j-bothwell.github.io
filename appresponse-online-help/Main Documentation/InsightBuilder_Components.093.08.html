Topology Components
All three topology views offered by the Desktop Console are available for building Insights:
AS Topology
BG Topology
IP Topology
Editing Topology Components
All Topology Components are edited using the Topology Options Dialog or by directly setting controls.
AS Topology
The AS Topology allows you to examine autonomous system connectivity to a destination. It shows the service providers that data packets must transit in order to reach a remote autonomous system. The data displayed in AS Topology is collected from traceroute. The AS path from the local network to the destination IP address is determined by mapping each hop in the traceroute to an AS number using the WHOIS database. A Destination AS cannot be displayed in the AS Topology view if a traceroute to that destination is not stored in the database. To minimize this likelihood, the automated traceroute selection algorithm will choose traceroutes across active Dest AS groups.

BG Topology
The BG (Business Group) topology allows you to visualize data exchanges between business groups. The topology identifies business groups that communicate with each other and illustrates the nature of that communication in the form of performance and utilization metrics.

IP Topology
The IP topology is used to examine Layer-3 connectivity to an IP address. This information is based on data collected through traceroute. The IP topology can be used to manually initiate and visualize live traceroutes. It can also be used to examine historic manual or automated traceroutes that are stored in the appliance database.
