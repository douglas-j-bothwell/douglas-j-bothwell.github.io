Text Components
The Insight Development Kit offers three Text Components:
Banner Component
Label Component, which displays dynamic text, including HTML
Static Text Component, which displays static text, including HTML
Images may be embedded within the HTML in both types of Text Component. For more information, see Inserting Graphics into HTML.
Label Component
The Label Component holds one line of text and is most often used as a label for other Visual Components. You may embed HTML tags in the text to change the font or style of the text.
Edit the Label Component
The illustration below shows the edit dialog for the Label Component.
Figure 70
Edit Label Component
 
This dialog box has the following components:
Background Color—enter RGB values or click Select Color... to use the Set Background Color dialog, which operates in the same fashion as the Set Color Dialog.
Group Index—See Using the Group Index.
Text Alignment - Left, Right, or Center
Using Insight Variables with the Label Component—You may use Insight Variables in a Label Component to display the values passed by Interactions, as described in Insight Variables.
Embedding HTML in Text Labels—HTML may be embedded in the text fields of the Edit the Component window by bracketing the text string with HTML tags, e.g.:
<html><font color=”red”>Label for Group ${group}</font></html>
The <br> tag may be used to create multiple lines.
You can insert graphics into the HTML. See Inserting Graphics into HTML.
Dynamic Highlighting—A Label Component can display two different text strings: a normal one and a highlighted one. The edit dialog box for the Component allows you to enter these separately. Enter the normal text in the Enter the text that is to appear on the line field. Enter the highlight text in the Enter the text that is to appear when highlighted field.
When an action is received the Component will react in accordance with its state (Normal or Highlight) and, if it is an index Interaction, in accordance with the group index value passed in the index Interaction, as detailed in the Interaction Consumed table below.
The Highlight Index setting defines both the response to an index Interaction and the Clear Interaction, as detailed in the Interaction Consumed table below.
Checking Highlight Blinking will cause the highlighted text to blink when this Component is highlighted.
Interactions Produced
The Text Label Component produces the following Interaction:
 
Table 60
Interactions Produced by Text Label Component
Interaction
How Produced
index
When the Label Component is highlighted it transmits its highlight index, most often to a Highlight Analysis.
The Label Component does not send out any Interactions when it is created.
Interactions Consumed
The Label Component consumes the following Interactions:
 
Table 61
Interactions Consumed by Label Component 
Interaction
Component Action
box
The component retrieves the box list and replaces all occurrences of the box variable ($box or ${box}) in the text with the box list. If highlight text is defined a non empty box interaction will cause the component to highlight, while an empty box interaction will cause the highlight to be removed.
clear
The component highlights if it has a highlight index of 0, otherwise it does nothing.
group
The component retrieves the group name from the Interaction and replaces all occurrences of the group variable ($group or ${group}) in the text with the group name. If highlight text is defined a non-empty group interaction will cause the component to highlight, while an empty group interaction will cause the highlight to be removed.
index
The Component action depends on its state:
•
Highlighted:
Component is displaying highlight text AND Interaction index = Component group index: display highlight text (i.e., maintain state).
Component is displaying highlight text AND Interaction index /= Component group index: display normal text (i.e., change state).
•
Not Highlighted:
Component is displaying normal text AND Interaction index = Component group index: display highlight text (i.e., change state).
Component is displaying normal text AND Interaction index /= Component group index: display normal text. (i.e., maintain state)
interval
The component retrieves the interval and replaces all occurrences of the interval variable ($interval or ${interval}) in the text with the interval text. If highlight text is defined a non-empty interval interaction will cause the component to highlight, while an empty interval interaction will cause the highlight to be removed.
limit
The component replaces all occurrences of the limit variable ($limit or ${limit}) in the text with the new limit from the Interaction. If highlight text is defined the component will highlight.
metric
The component retrieves the metric names and replaces all occurrences of the metric variable ($metric or ${metric}) in the text with the metric name list. If highlight text is defined a non-empty metric interaction will cause the component to highlight, while an empty metric interaction will cause the highlight to be removed.
index
The component checks the index that was transmitted and compares it to its highlight index. If the transmitted index matches the highlight index, the component will highlight, if the index is different the highlighting is removed.
text
replaces {text} in the label text.
Static Text Component
The Static Text Component displays static text or HTML. It is not limited to one or a few lines. The only variables it recognizes are the URL variables for embedding images, as described in Inserting Graphics into HTML. The banner at the top of the Insights supplied with AppResponse products is an example of a Static Text Component.
Edit the Static Text Component
The Static Text Component is edited using a simple text window.

To use the Static Text Component to display text, type in the desired text and click OK.
To use the Static Text Component with HTML there are a few options, all that require a basic understanding of HTML - some more than others. You can do the following:
1)
Type HTML directly into the text box and click OK.
2)
Click New Html Document to display a skeleton of an HTML document whereby you enter text between the tags.
Figure 72
Edit the Component Dialog for HTML with Template
 
3)
Select a tag from the drop-down list and click Add. Enter the text in between the start tag and the end tag.
4)
Check Auto-Complete. When checked, you can enter < symbol and the console gives you a list of tags to choose from. Auto-Complete is off by default.
Figure 73
Edit the Component Dialog for HTML with Tag Definitions
 
You can also use Insight variables. To see a list of Insight Variables enter $ into the window and press Enter.
You can insert graphics into the HTML. See Inserting Graphics into HTML.

Interactions
The Static Text Component does not produce or consume any Interactions.
