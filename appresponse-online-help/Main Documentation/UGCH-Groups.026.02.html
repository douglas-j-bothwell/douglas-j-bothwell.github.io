Built-in Groups
The appliance provides a set of built-in groups that are supported out of the box without no configuration required. These groups allow you to analyze traffic at an aggregate level instead of examining data for individual IP addresses.
The following table lists the built-in groups. Not all groups are applicable to AppResponse Standard.
 
Table 1
Built-In Groups 
Group Name
Description
Total Traffic
All traffic monitored by the appliance.
IP Protocol
All traffic for a single protocol encapsulated within IP (e.g., TCP, UDP or ICMP).
Port
All traffic for a single transport-layer port. (e.g., HTTP or UDP-457).
The Port group is only available as a drill-down from IP Protocol groups.
IP Address
All traffic for a single IP address.
Prefix/24
All traffic for IP addresses with a common 24-bit prefix (e.g., 192.168.10.0/24).
Dest CIDR 1
All traffic for IP addresses within a CIDR block. (e.g., 192.168.0.0/16).
The Dest CIDR group (short for Destination Classless Internet Domain Routing) is only available as a drill-down from ISP AS groups.
Dest AS
All traffic to IP addresses that are contained within a single autonomous system identified by Dest AS (see Figure 1 for an illustration). Dest AS, short for Destination Autonomous System, is the autonomous system number of the network that includes the remote IP address. IP addresses are mapped to a Dest AS number using the WHOIS database stored on the appliance.
ISP AS 1 2
All traffic that is routed over the service provider identified by ISP AS (see Figure 1 for an illustration). ISP AS is the autonomous system number of the service provider that routes data from the network in which the appliance is installed. This service provider (ISP AS) is identified using the BGP table. The name of ISP AS is displayed in the Information column for this group.
Peer AS 1 2
All traffic that is routed over a specific service provider through the network identified by Peer AS (see Figure 1 for an illustration). Outgoing traffic is sent to a service provider, identified by ISP AS. In most cases, the service provider must send this traffic to a peer network before it reaches the final destination. Peer AS is the autonomous system number of the peer network. In many cases, traffic is routed over multiple ASes before reaching the destination IP address. For the purpose of grouping, the appliance only records the first AS (ISP AS) and second AS (Peer AS) in the AS Path.
If the traffic terminates within the service provider (ISP AS) itself, the ISP AS number is the same as the Peer AS number. The Peer AS groups are only available as a drill-down from ISP AS groups.
1
These groups are only available if the appliance has been configured as a BGP peer. For more information on BGP and how to enable this functionality, see BGP and the AppResponse Appliance. If BGP is not configured, all ISP AS and Peer AS groups is set to ‘Unknown’.

2
Traffic to IP addresses that do not appear in the BGP table are allocated to ‘AS Unknown’. For information on configuring static mappings between IP addresses and AS numbers, see Setting Up Private-Address to AS-Number Maps .
