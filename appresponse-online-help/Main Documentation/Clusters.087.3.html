Domain Operations
This section describes the operations involved in creating, configuring and maintaining a domain. All domain operations are performed using the Domain Manager in the AppResponse Console.
Domain Manager
All domain operations are performed using the Domain Manager in the AppResponse Console. To invoke the Domain Manager, go to the Tools menu in the AppResponse Console and select Domain Manager.
Figure 2
Domain Manager on a Director
 
With one exception, the Domain Manager should always be run on the director. However, if the director fails, it may be necessary to forcibly delete an appliance from the domain. In this case the Domain Manager must be invoked from the appliance, as described in Forcibly Deleting an Appliance from a Domain.
The domain operations documented below must be executed using a user account with administrative privilege. See Account Privileges .
Creating a Domain
To create a domain, install and configure all appliances as normal. (See Installing the Appliance.) This installation procedure should be followed on the director and all AppResponse Appliances to be part of the domain.
After installation is complete, start and connect the AppResponse Console to the director. Use the Domain Manager to add the desired appliances to the domain.
Adding an Appliance to a Domain
To add an appliance to a domain, invoke the Domain Manager on the director and follow these steps:
Note: Appliances must be upgraded to 5.3 or later in order to be a part of a 6.0 Domain. Be aware that Recognized Applications are not distributed to appliances that are not 6.0. See Distributed Applications.
Procedure 1
Adding an Appliance to a Domain
1.
Click the New button.
2.
Enter the Name of the AppResponse Appliance. This Name is used to refer to the appliance when performing distributed operations within the Desktop Console connected to the director. It does not need to match the hostname of the appliance registered in DNS.
3.
In the IP/Hostname field, enter the IP Address or Hostname of the appliance. This field identifies the appliance and must be a valid IP address or hostname resolvable through DNS.
Note: If the director and the appliance are in a dual-stack environment, you should specify the IPv4 address (not the IPv6 address) of the management port on the appliance. For more information about IPv6 support, see What You Need to Know About IPv6 Support in AppResponse.
Note: Once an appliance is part of a director domain, you cannot change the appliance IP address or enable DHCP in the Administration > System > Setup page. (The DHCP settings on this page might appear to be available—ie, not grayed out—but changing them has no effect.) You must disconnect the appliance from the domain before you can change these settings.
4.
Enter the User Name and Password of a user account with Administrative privilege on the appliance. (See Account Privileges.)
5.
Click OK or Apply to save configuration changes. (Adding an appliance to a domain may take a few minutes.)
End of Procedure 1
 
 
When an appliance is added to the domain, the director distributes configuration information to the appliance. This information includes global account definitions and group definitions. See Distributing Configuration Information.
Deleting an appliance from the domain removes the distributed configuration information. See Deleting an Appliance from a Domain.
Add an Appliance to a Domain using Duplicate
In many cases, the same configuration information should be distributed to multiple appliances in a domain. To facilitate this operation, you should add one appliance to the domain and configure all desired distribution lists. Then, add a new appliance to the domain using the Duplicate feature. The new appliance adopts the same distribution configuration as the duplicated appliance.
Procedure 2
Adding an Appliance to a Domain using Duplicate
1.
Open the Domain Manager on the director.
2.
Right-click the appliance to be duplicated and choose Duplicate from the context menu.
Figure 3
Domain Manager—Duplicate
 
3.
Click OK in the confirmation dialog.
End of Procedure 2
 
 
After an appliance is added to the domain, you should configure the distribution list for each configuration item that should be pushed to the appliance. For more information, see Distributing Configuration Information.
Connection Status
The Domain Manager displays the connection status of an appliance at the bottom of the window.
Figure 4
Domain Connection Status
 
The status information contains the fields listed in the following table.
 
Table 1
Status Information Fields
Field
Description
Data Link
The Data Link is up if the appliance is connected to the domain. This means that the appliance and director have established communication and are ready to exchange data. If the appliance is disconnected from the domain, the Data Link field shows down.
Last Data Collection
The Last Data Collection field shows the timestamp of the most recent data that has been collected by the appliance.
Last Data Aggregation
The Last Data Aggregation field shows the timestamp of the most recent data from that appliance that has been aggregated on the director.
NTP Time Offset
The NTP Time Offset field shows the clock difference between the director and the appliance. The field value is zero if the clocks are synchronized exactly. The value is a positive number if the appliance clock is behind the director clock and negative if the appliance clock is ahead of the director clock.
An appliance configured in the Domain Manager that has not been manually disconnected automatically connects to the domain if it can establish communication with the director.
Deleting an Appliance from a Domain
When an appliance is deleted from a domain, the following actions take place:
1)
The director no longer distributes new configuration items to the appliance. (See Distributing Configuration Information.)
2)
All global user accounts are removed from the appliance.
3)
All other existing global configuration items are changed to local configuration items.
Deleting an Appliance from a Domain Using the Director
Follow this procedure to remove an appliance from a domain:
Procedure 3
Deleting Appliances from a domain using the director
1.
Open the Domain Manager on the director.
2.
Select one or more appliances in the table at the top of the Domain Manager window.
3.
Click the Delete button.
4.
Click OK or Apply to save changes.
End of Procedure 3
 
 
Forcibly Deleting an Appliance from a Domain
Under rare circumstances, it may be necessary to forcibly delete an appliance from a domain directly from the appliance itself. This is only necessary if the director is down or unreachable. An appliance should forcibly delete itself from the domain if:
The director machine has failed and must be replaced. Each appliance must be forcibly deleted from the domain before a new domain can be formed using the new director.
If the director machine is down or unreachable and the appliance should no longer be part of the domain.
Follow these steps to forcibly delete an appliance from a domain.

Emergency Delete
You can delete an appliance even if it is not currently connected to the Director. This operation is available only if the appliance is in one of the following states:
–
Connected
–
Disconnected
–
Unauthorized
–
Unavailable
Appliance List Export/Import
You can now export the names and IP addresses of all appliances to a CSV file. This makes it much easier to edit and copy large appliance lists between different Director domains.
To generate a appliance data file, open the Domain Manager and then choose File > Export. The resulting data file looks like this:
Name, IP
<appliance-name-1>, <appliance-IP>
<appliance-name-2>, <appliance-IP>
...
<appliance-name-n>, <appliance-IP>
You can also import a list of appliances from a CSV file into the Domain Manager. To import a data file, open the Domain Manager and then choose File > Import. The CSV file uses the following format:
Name, IP
<appliance-name>, <appliance-IP>, <username>, <password>
<appliance-name>, <appliance-IP>, <username>, <password>
...
<appliance-name-n>, <appliance-IP>, <username>, <password>
Note: The <username> and <password> fields are optional, and are never exported.
Disconnecting an Appliance from a Domain
An appliance that belongs to a domain receives configuration changes from the Domain Director. An appliance must be temporarily disconnected from a domain in order to perform certain administrative operations. Disconnecting the appliance from the domain ensures that the configuration changes do not interfere with the administrative operations. While the appliance is disconnected from the domain, the director maintains a list of pending configuration changes that must be applied when the appliance reconnects.
Follow the procedure outlined below to disconnect the appliance from the domain before performing the following operations:
Change the IP address of an appliance.
Backup or Recovery of an appliance
Change the port number assigned to the HTTP service on the director. All appliances must be disconnected from the domain before the port number can be changed.
An appliance should also be disconnected from the domain any time it is offline (e.g., maintenance or system replacement).
Follow these steps to disconnect an appliance from a domain.
Procedure 5
Disconnecting an Appliance from a Domain
1.
Invoke the Domain Manager on the director.
2.
Uncheck the Activate checkbox for the appliance(s) to be disconnected.
3.
Click OK or Apply to save changes.
The appliance should be manually reconnected to the domain after the operation is complete.
End of Procedure 5
