Custom Views
Custom Views provide a framework for creating targeted views that answer specific questions relevant to you and others in your organization: What is the status of Application X for users in my default enterprise network? How many Slow Pages have clients in Remote Office Y downloaded in the past hour? For each custom view, you can specify
The data sets displayed (Business Groups, Applications, Web Applications, and so on)
How the group data is displayed (charts, tables, and layout).
The Basic and Restricted users who cannot access the Custom View

Important Notes
Note the following:
This release does not support the following in Custom Views:
–
Top Tables
You cannot define a Custom View with a table that auto-updates to show (for example) the top x Business Groups by metric Y for the selected time. Only Favorites Tables with a defined set of groups are supported. Top Tables are included in all Navigator views, however.
–
Top Bar Charts
As a workaround, use Top Pie Charts instead.
–
Generating a Report from a Custom View
–
Clicking on a table or chart to view data in a separate Navigator
You cannot add a group to a table or chart unless it appears in the Select Groups table. If the group of interest does not appear, change the number of groups or the topping metric until you see the group you want to add. Then you can drag and drop it into the table or chart.
Adding data via drag-and-drop from the table is recommended in Tab Layout only. (In Card Layout, you might be unable to see the destination table or chart without scrolling down in the browser window.)
To change the order in which tables or charts appear in a Custom View, go to Tab Layout. You can then right-click in a tab and choose Move Left or Move Right to move it. When you switch to Card Layout, the cards appear as ordered in the tabs, from left (top) to right (bottom).
To open the pull-down menu for a tab, you can hover the cursor between the tab title and the close (x) button.
 
Create a New Custom View
Procedure 1
To create a new Custom View:
1.
Make sure that you are logged in to the appliance as a user with Administrator or Standard privileges.
2.
Home tab: Select Administration, and then “Custom Views” from the category table listing.
3.
Configure Custom Views tab: Click New.
4.
Edit: New Custom View tab: Specify the groups, charts, and tables you want to include in the Custom View.
 
To create a chart or table:
4.1.
In the Select Groups table (top), select the traffic data you want to display. Use Ctrl-click or Shift-click to select multiple entries.
4.2.
Send the data to a new table or chart. With the rows of interest selected in the table, click the Send To button in the table toolbar or one of the New Chart/Table toolbar buttons. You can also drag selected data into the table or chart, but this is recommended in Tab Layout mode only (because the destination table/chart is always visible—you never need to scroll down in the browser window).
 
4.3.
Edit the new table or chart as desired.
• 
To remove groups or edit the display format, right-click in the card and choose Settings.
• 
To add a group to a card:
Select the group in the table and either drag it into the card, or
Select the chart or table (in Tab Layout mode) and choose Send To > Add to Selected View in the table toolbar.
4.4.
Repeat step 4.1. through step 4.3. for each chart or table that you want to add to the Custom View.
5.
Edit the Custom View layout as desired.
You can choose either a Tab or Card layout. A Card layout can be useful because you can see multiple tables and charts at the same time.
Tip—When designing your layout, Tab Layout provides slightly more flexibility: you can move between charts and tables quickly (no need to scroll in the browser) and the middle tabs have both Move Left and Move Right context-menu options. Even if you are designing a Card Layout, you might want to use Tab Layout to configure and organize the individual components and then switch to Card Layout to preview the design.
To change the number of columns (Card Layout only), row height, or margins, click the New Chart/Table > Settings toolbar button.
 
To rename, move, or edit settings for a specific chart or table, go to Tab Layout. Then right-click on the tab, or hover the cursor between the tab title and the ‘x’ button. (You can also right-click in a chart/table to edit settings.)
 
6.
At the top of the page, click the Preview link and verify that all charts and tables in the page appear as desired. To edit the page further, click the Edit link.
 
7.
At the top of the page, choose File > Properties and specify the following:
• 
Name—Name of the Custom View as it appears in the tab and the Custom Views treeview in the Home tab.
• 
Description—A high-level description of the Custom View. This description appears when you click on the parent category in the Home tab > Custom Views treeview.
• 
Category—The parent category of the Custom View that appears in the Home tab > Custom Views treeview.
• 
Share with Other Users checkbox—This option is not selected by default, which means that the Custom View is accessible as follows:
• 
View Access—Administrators only
• 
Write Access—Administrators only
If Share with Other Users is selected, the Custom View is accessible as follows:
• 
View Access—All Administrators, all Standard users, and any Basic and Restricted Users assigned a role that includes access to this Custom View
• 
Write Access—All Administrators
8.
Choose File > Save to save the new custom view.
9.
Go to the Home tab. In the treeview, and select the Custom View category you assigned. Verify that the view appears in the Custom Views table.
End of Procedure 1
 
 
Edit a Custom View
Procedure 2
To edit a Custom View:
1.
Make sure that you are logged in to the appliance as a user with Administrator or Standard privileges.
2.
Home tab: Select Administration, and then “Custom Views” from the category table listing.
3.
Configure Custom Views tab: Click on the label in the Name column to open the view in Edit mode.
You can also open a view from the Custom Views treeview and then choose Settings > Edit (top-right corner of the page).
To edit a Custom View while it is open, click Settings > Edit in the top-right corner of the page.
4.
Edit the View as desired.
To create a chart or table:
4.1.
Select the data of interest in the Select Groups table (top).
4.2.
Send the data to a new table or chart. To do this, you can
• 
Click Send To in the table toolbar.
• 
Click one of the New Chart/Table toolbar buttons.
To edit a chart or table:
Right-click in the card and choose Settings.
To add data to a card:
Select the data in the table and drag it into the card or tab.
(Tab layout mode only) Select the data in the table, select the destination tab or card, and then choose Send To > Add to Selected View.
To change the number of columns row height, or margins (Card Layout only):
Click the New Chart/Table > Settings toolbar button.
 
To rename, move, or edit settings for a specific chart or table:
Go to Tab Layout and then right-click on the tab. (You can also right-click in a chart/table to edit settings.)
 
To edit the view name, description, category, or access settings:
4.1.
At the top of the page, choose File > Properties and specify the following:
• 
Name—Name of the Custom View as it appears in the tab and the Custom Views treeview in the Home tab.
• 
Description—A high-level description of the Custom View. This description appears when you click on the parent category in the Home tab > Custom Views treeview.
• 
Category—The parent category of the Custom View that appears in the Home tab > Custom Views treeview.
• 
Share with Other Users checkbox—This option is not selected by default, which means that the Custom View is visible to you and only you. Check this box to share with other users who log in to the appliance.
If Share with Other Users is selected, the Custom View is visible to the following users:
• 
All Administrators and Standard users
• 
All Basic and Restricted users who have a role assigned that allows them to view the specific Custom View. For more information, see XREF.
5.
Choose File > Save to save the new custom view.
6.
Go to the Home tab and verify that the view you save appears in the correct location in the Custom Views treeview.
End of Procedure 2
