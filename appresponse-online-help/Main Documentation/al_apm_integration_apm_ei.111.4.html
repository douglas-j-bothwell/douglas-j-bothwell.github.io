Cisco IP SLA Test Query Insight
The Cisco IP SLA Test Query Insight is useful when you want to investigate the performance of your Cisco IOS devices during periods of problematic performance. With this Insight, you can
Navigate through all Cisco devices for which NetSensor is collecting IP SLA data.
Drill down into SLA test classes to find metrics of interest.
View time-series charts that show the variation of specific metrics during the project time window.

Workflow Description
The following steps outline the general workflow for using this Insight:
1)
Open the Insight by choosing Insights > Element Polling > Cisco IP SLA Query.
➥ The window opens and the Polled Systems List populates with all Cisco devices discovered by NetSensor.
2)
Select a Cisco IOS device in the Polled System List.
➥ The System Information (top) is updated for the selected device. The IP SLA Metrics table (center) lists all the IP/SLA test classes (on the Jitter and Latency tabbed pages) for which NetSensor has collected data from the selected device.
Note—If “No Data” appears in the table, it could be that no tests are configured on the device or that there were no test results in the project time frame.
3)
Click the IP SLA Jitter or IP SLA Latency tab (as desired), then click a metric cell in the SLA Metrics table. (You can select multiple metric cells for comparison, if desired.)
➥ The time-series chart (bottom) shows the variation in the selected metrics over the current project time window.
