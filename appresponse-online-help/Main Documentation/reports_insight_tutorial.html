Tutorial: Creating an Insight Report
This tutorial shows one approach to creating an Insight report. It describes all steps, from start to finish, for creating a typical Insight report. The workflow followed here is the one given in Report Manager: Creating Insight Reports.

The build and test stages are likely to be iterative, as you modify your Insight report to provide exactly what you want to see.
State the Goal
The first step to creating an Insight report is determining what the report is intended to do. For this tutorial, let’s assume you want to develop a report that will answer the following questions:
How much bandwidth is the HTTPS application using?
What factors are affecting HTTPS response time?
Who are the biggest users of the HTTPS application?
To answer these questions, the Insight report should:
Display a time-series chart of throughput in and throughput out for the HTTPS application
Display a response time composition chart for the HTTPS application
Identify the top IPs using the HTTPS application (by throughput)
You want the report to show results for a 1-hour period.
Sketch the Report Layout
After defining the problem, it is time to determine which components are needed for the Insight report and how they should be arranged (both visually and functionally).
The first two things you want to display in the report are charts. To show throughput in and throughput out as separate traces, a Multi-Metric Time Series chart is needed. As already determined, the second chart will be a Response Time Composition (RTC) chart. These two charts will be stacked in the report, as shown in Figure 1. Because the HTTPS application is defined as a group, you can add it directly to each of these charts (no drilldown or selector is needed).
The third item for display—top IPs using the HTTPS application (by throughput)—calls for a Favorites Table. Because the top IPs are not predefined, you will need a Drilldown Analysis for this, linked to the table by a group interaction. This is an allowed use of the group interaction in Insight reports.

Build the Insight Report
Now that you know what components the Insight report will contain and how they will be arranged, you can build the Insight report.
Procedure 1
Building the Insight Report
1.
Set the project time to any 1-hour period.
2.
Open the Report Manager (choose Tools > Report Manager…).
3.
In the Report Manager, click New.
The Edit Report Content dialog box opens.
4.
Select Blank Insight as the report type.
The Edit Report Content dialog box expands to become a subset of the Insight Development Kit window. This version of IDK contains Components that are allowed in Insight reports, but otherwise works exactly like the full IDK.
5.
Add a Multi-Metric Time Series chart and a Response Time Composition chart to the Layout Grid.
6.
Switch to Actual view (Ctrl-A).
7.
Drag the Multi-Metric chart into the empty cell.
8.
Add another row to the Layout Grid and drag the RTC chart into the new cell.
9.
Add the HTTPS application to the charts:
9.1.
In the Desktop Console, open a Favorites Table.
9.2.
In the table, click the Add Group/Drilldown icon.
9.3.
In the Group/Drilldown Chooser, select Defined Apps: and choose HTTPS from the pull-down list. Click OK.
 
An HTTPS row is added to the Favorites Table.
9.4.
Click-and-drag the HTTPS row to each of the two charts in the Layout Grid.
Each chart displays data for the HTTPS application.
9.5.
Close the Favorites Table.
10.
Double-click on the Multi-Metric chart and edit it to display the Throughput (Inbound) and Throughput (Outbound) traces.
The Layout Grid should resemble the following figure.
 
11.
Return to the Workflow view.
12.
Add a Drilldown Analysis to the Layout Grid and configure it as follows:
• 
Select the Run Analysis on Startup checkbox.
• 
Set Drilldown to: to HTTPS>External IPs>1.1.1.1.
 
13.
Add a group interaction from the Drilldown to the Favorites Table.
14.
Switch to Actual view.
15.
Add another row to the Layout Grid and drag the Favorites Table into the new cell.
All components have now been added to the Insight report and the Layout Grid should resemble the following figure.
 
End of Procedure 1
 
 
Test the Insight Report
The Insight report is built and it is time to test it. As part of this, you can fine-tune the appearance of the report and make sure that everything is visible. Because each generated report will be static, the Insight report must look exactly as you want it before it is published.
Procedure 2
Testing the Insight Report
1.
Click the Test/Layout button.
The Preview Insight Content window opens.
 
All of the Components contain data, so the Insight report is working. However, it needs some changes to its appearance. Notice that some information is not visible (the RTC chart and several rows in the Favorites Table). Also, the table toolbar isn’t needed and can be removed.
Some changes can only be made from the Actual view, so make those before returning to the Preview for final resizing.
2.
Click OK to return to the Actual view.
3.
Edit the Favorites Table to hide the toolbar:
3.1.
Double-click the table.
The Edit the Component window opens.
3.2.
Right-click in the Edit the Component window and choose Hide Toolbar.
3.3.
Click OK.
4.
Best practices suggest removing chart borders. For each chart, unselect the Show Border checkbox (on the Format page of the Chart Settings dialog box).
 
5.
Make any other changes suggested by best practices or your own preferences.
6.
Click the Test/Layout button.
The Preview Insight Content window opens. Notice that the Preview now reflects the changes you just made.
7.
Resize the window horizontally and vertically to get the overall size you want. For example, if you plan to include the report image in a document, make sure it will fit where intended.
8.
Adjust each Component to a useful size. For example, make sure the charts are legible and that there are no scroll bars in the Favorites Table.
When you are finished, the Preview should look similar to the following figure.
 
9.
Click OK to close the Preview.
10.
Click OK to close the Edit Report Content window.
End of Procedure 2
 
 
Publish the Insight Report
The Insight report content is now complete. All that remains is to configure the report in the Report Manager and publish it.
Procedure 3
Publishing the Insight Report
1.
Click the Identify tab in the Report Manager.
2.
Change the report Name to <initials> HTTPS Report.
Note—Generally, you will want to use the Schedule tab to generate the report on a regular schedule. For this tutorial, you will keep the default settings and generate the report manually.
3.
Click Apply or OK.
4.
When the changes have been successfully committed, click OK.
End of Procedure 3
 
 
To see your new Insight report, go to the AppResponse web interface and click the Reports tab. You will find it on the Console Reports page under Unfiled (Public).
