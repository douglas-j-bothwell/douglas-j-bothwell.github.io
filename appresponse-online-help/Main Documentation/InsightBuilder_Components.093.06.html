Chart Components
All of the chart types available in the Desktop Console are also available as Visual Components for building Insights:
Bar Chart
Correlation Chart
Cumulative Chart
Distribution Chart
Multi-Group Time Series Chart
Multi-Group Metric Time Series Chart
Multi-Metric Chart
Pie Chart
Response Time Composition Chart
Route Latency Chart
Top Pie Chart
Top Time Series Chart
More information on charts can be found in the AppResponse Operations Guide.
Editing Chart Components
All Chart Components are edited using the Chart Settings Dialog or by directly setting controls, except for settings that pertain specifically to their use as Insight Components, as discussed below.
Reaction to Interval Interaction
Select Update Chart Time Interval to make the Chart reset its interval to the interval received in the Interaction.
Select—Select Time Region to make the Chart select the interval received in the Interaction, as though the user had done a left-click-sweep.
Composition
The Composition Tab of the Chart Settings Dialog in Chart Components gives you control over what features of a AppResponse Chart are available to the user of an Insight: only those features checked will be operational in the Insight for that particular Chart. Each type of Chart Component has its own set of features that may be checked. Each Chart in an Insight may have different settings. Detailed information on these features may be found in the Administrators Manual.

Chart Insight Links
The Insight Links tab of the Chart Settings Dialog in Chart Components gives you the ability to create a Chart that a user can launch other Insights from using the chart legend context menu (i.e. by right-clicking on the Chart legend.
Figure 56
Chart Settings Dialog > Insight Links Tab
 
This page has the following components:
Label - enter the name of the Insight as you want it to appear in the Launch drop-down menu in the Chart’s legend context menu.
Local - check this box if the Insight to be launched is in the local Insight directory.
Insight Path - if Local is not checked, you may select the Insight to be launched from a drop-down list with the Insights currently saved on the appliance. If Local is checked, you must enter the full filename and directory path of the Insight file.
Add - click Add to add a new line for entering an Insight link.
Remove - click Remove to delete an Insight link.
Default Insight - this is not functional in Chart Components.
Interaction Values Passed to Insights from Charts
The following Table lists the Interactions and values passed to the linked Insight for any chart that produces the listed Interaction.
 
Table 6
Interaction Values Passed to Insights from Charts
Interaction
Value(s) Passed to Linked Insight
group
For single group charts, the group is passed.
For multi-group charts, all groups are passed unless one or more groups are selected, in which case the selected groups are passed.
interval
Interval.
metric
In a single-metric chart, the metric is passed.
In a multi-metric chart, all metrics are passed unless one or more metrics are selected, in which case the selected metrics are passed. Metrics that have been removed by user action from the chart are not passed.
unit
Unit.
yAxisLimit
Y-axis limit values.
zoomInterval
In a chart configured with Selectable Time Region checked, if the user has done a horizontal sweep to narrow or expand the selected time region, the selected interval is passed, otherwise the chart interval is passed.
All other values needed by the linked Insight are obtained via that Insight’s Initializer Component.
Bar Chart
This is a Visual Component that displays a Bar Chart with multiple groups and one metric.
Interactions Produced
The Bar Chart Component produces the following Interactions:
 
Table 7
Interactions Produced by Bar Chart Components
Interaction
How Produced
group
User selects one or more groups in the chart.
metric
User selects a metric using the chart’s metric chooser.
unit
User changes the Bar Chart unit.
A group Interaction is sent out at startup to notify any Components that are listening to the chart's group selections of the initial group setting.
Interactions Consumed
The Bar Chart Component consumes the following Interactions:
 
Table 8
Interactions Consumed by Bar Chart Component
Interaction
Component Action
box
The appliance identified in the Interaction is saved. On a Director, whenever a group is queried the data is retrieved from the Appliance that was passed by the box Interaction.
clear
The chart displays the global Desktop Console time interval in the chart, requires the chart data, and sends out an interval Interaction with an empty interval.
group
All groups are removed from the chart and the groups in the Interaction are added to the chart.
initialization
When this component receives an initialization interaction with a start mode, it will hold off sending out any queries until an initialization interaction with an end mode is received.
interval
The chart displays that interval in the chart and re-queries the chart data.
metric
The metrics displayed by the chart are set to the metrics in the Interaction.
unit
The Bar Chart unit is updated with the unit in the Interaction if the metric in the Interaction matches the current Bar Chart metric. Otherwise, no action is taken.

Route Latency Chart
This is a Visual Component that displays the Route Latency Chart with trace route round trip time of the hops in the historic traces for a given IP.
Interactions Produced
The Route Latency Chart Component produces the following Interactions:
 
Table 23
Interactions Produced by Route Latency Chart Component
Interaction
How Produced
clear
User does a right-click and selects clear selected time region.
group
User selects the group(s) in the chart. The user has options to pass “All Groups” or “Selected Groups” in the interaction. The default option is to pass “Selected Groups”.
interval
User has options to “Pass Full Chart Interval” or “Pass Zoomed Chart Interval” in the Interaction.
yAxisLimit
User changes the minimum and/or maximum y axis limit value.
zoomInterval
User does a horizontal sweep to narrow or expand the selected time region.
The group and interval Interactions are sent out at startup to notify any Components that are listening to the Route Latency Chart's group and interval selections of the initial group and interval settings.
Interactions Consumed
The Route Latency Chart Component consumes the following Interactions:
 
Table 24
Interactions Consumed by Route Latency Chart Component
Interaction
Component Action
box
When a box Interaction is received, the appliance that was received in the Interaction is saved. On a Director whenever a group is queried the data is retrieved from the Appliance that was passed in by the box Interaction.
clear
The user's selected time region is cleared from the chart and an interval Interaction is sent out with an empty interval.
group
The current group is removed from the chart and the traceroute information is retrieved for the first group in the interaction and the hops in the traceroutes are added to the chart.
initialization
Initialization interaction with a start mode, it will hold off sending out any queries until an initialization interaction with an end mode is received.
interval
The option in the interaction is examined first. If the option is “Update Time Interval in Chart,” which is the default option if no option specified, the chart sets its interval from the interval in the interaction and re-queries the chart data. If the option is “Select Time Region in Chart” and the selectable time region feature is enabled, the chart highlights the time region from the interval in the interaction. If the highlight selected time region feature is enabled it sends out an interval Interaction with the newly selected time interval.
yAxisLimit
The minimum and maximum y axis limit values of the chart are updated to the minimum and maximum values in the Interaction if the current display unit matches the unit in the Interaction. Otherwise, no action is taken.
zoomInterval
Highlights the time region with the interval in the Interaction and send out an interval Interaction with the interval in the Interaction.
