Baseline Insights
Baseline Insights, as their name implies, help you establish a baseline for your network: what’s normal? They help you establish norms for latency, load, response time, and similar concerns, everywhere on your network, for any groups or applications of concern.
Baseline Insights include:
Group Application Baseline
Network Performance
Trending - Custom and Yesterday/Last Week/Last Month
User Response Dashboards - 24 x 7 and Custom
Group Application Baseline
Group Application Baseline is located on the Baseline tab. This Insight can answer questions such as “What’s the typical performance of my critical applications?”, “What’s normal and what’s not?”, and “Where might I start looking to improve performance?” The Insight defaults to using the Throughput metric. This can show you which Business Group applications are using the most bandwidth (high volume). You can change the metric and use the Response Time Composition Chart to see what kind of transaction activity is associated with this application. With this information you can answer the questions above.
Figure 1
Group Application Baseline Insight
 
Procedure 1
Using the Group Application Baseline Insight
1.
From the selector pane (top left) select the Business Group you wish to use by selecting the Top Business Groups tab or All Business Groups tab.
2.
From the selector pane (bottom left) select an application used by the members of the Business Group you selected above.
This shows server and client turn activity for the selected Business Group and application, which is a barometer of transition activity.
End of Procedure 1
 
 
Network Performance
The Network Performance Insight is located on the Baseline tab. This Insight helps isolate possible network latency issues by showing you round trip time and retransmission delay on inbound and outbound traffic. Note that it is normal for traffic to be slower on a public network as their bandwidth is usually less than a private network. The figure below illustrates this point.
Figure 2
Network Performance Insight
 
Procedure 2
Using the Network Performance Insight
1.
Select the Business Group you wish to use by selecting the Top Business Groups tab or All Business Groups tab from the selector pane (left) and wait for the graph to update.
2.
Double-click the selected Business Group to drill-down to Business Groups and IP addresses that are connected to the group.
End of Procedure 2
 
 
Trending
There are two Trending Insights (Custom and Yesterday/Last Week/Last Month). They are both located on the Baseline tab. Trending Insights show the current trend on any given metric as compared to a custom time frame or yesterday, last week, or last month. By looking at these time snapshots, you can learn what is normal for your network. You can also use the comparisons of the two times to figure out if you network is currently behaving within the normal range.
Figure 3
Trending (Yesterday/Last Week/Last Month) Insight
 
Procedure 3
Using the Trending Insight
1.
From the Insight index select one of the two Trending Insights.
2.
From the selector pane (left) select the Business Group you wish to use by selecting the Top Business Groups tab or All Business Groups tab.
By default the top time series chart shows the current hour for the project time. You can change the time period by using the project time controls.
The bottom time series chart shows three tabs (Yesterday/Last Week/Last Month) or another project time control that can be set to any time period (custom).
3.
Select the time period you wish to compare by either selecting one of the tabs (Yesterday/Last Week/Last Month) or setting a time period (Custom).
End of Procedure 3
 
 
User Response Dashboards
There are two User Response Dashboards (24 x 7 and Custom). They are both located on the Baseline tab. These Insights show a real-time and historical view of the application performance on the network. User Response Time Dashboard 24x7 runs 24 hours a day, seven days a week, independently of the current time selection. User Response Dashboard Custom runs to the time you configure using the Business Hours button on the toolbar.
Figure 4
User Response Dashboard (24 x 7) - Configure Tab
 
Procedure 4
Using the User Response Dashboards
1.
In the Configure tab select applications using All Applications or Worst Application tabs.
You can use the Control or Shift key to select more than one application.
2.
Select the My Dashboard tab.
The selector pane (left side) now shows only the applications you have chosen from the selector pane on the Configure tab.
3.
Select the application you want to monitor (only one application can be selected).
 
Every five minutes, the Response Time Composition Chart updates with real-time metrics against color-coded thresholds.
The severity indicators in the Severity column of My Dashboard represent the severity over the most recent five minute time interval.
The percentages in the Performance Summary column show a combined history of every five minutes an severity indicator value was received since the Insight was started until the current time. These percentages are reset on a daily basis.
Figure 5
User Response Dashboard (24 x 7) - My Dashboard tab
 
Default thresholds for User Response Time are 2000, 4000 and 6000 msec. Thresholds can be set by selecting the My Dashboard tab and right-clicking on the application and selecting Edit Threshold.
The Response Time Composition Chart only shows information for the application selected from the My Dashboard tab whether the view is from the Configure tab or My Dashboard tab. If nothing is selected in the My Dashboard tab, the Response Time Composition Chart is blank.
If the hours are set using the User Response Dashboard Custom, the Response Time Composition Chart shows the selected time period. If the User Response Dashboard 24x7 is used, the Response Time Composition Chart shows from the time the User Response Dashboard is opened to midnight that day. At midnight, the chart starts over.
The dashboard runs in the back ground for all applications on the My Dashboard tab. In other words, if you have the dashboard up for three hours with BGP selected, you can switch to HTTP and the chart shows HTTP metrics for the last three hours.
To know more about a certain application, from the My Dashboard tab, double-click an application from the selector pane (or right click > Launch > Application Drilldown).
The Application Drilldown window opens.
Figure 6
User Response Dashboard (24 x 7) - Application Drill-down window
 
From this window the Response Time Composition Chart is shown in a magnified view, metrics in the time series charts and which IP addresses are involved with certain metrics.
Procedure 5
Using the Application Drilldown Window
1.
Select the RTCC tab (upper right) to see a magnified view of the Response Time Composition Chart.
2.
To see a time series chart for a certain metric, select the metric (lower left) and click the Time Series tab (upper right).
This metric also drives the information shown in the Internal IP, External IP, IP Conversations and Business Group tabs
3.
To see the IP addresses based on the metric selected, click one of the Internal IP, External IP, IP Conversations and Business Group tabs (lower right).
End of Procedure 5
