Launch Components
The Insight Development Kit offers two Components for launching scripts, browsers, and other Insights:
Link and Script Launcher, which displays an HTML document or a textual representation of the results returned by a script.
Insight Launcher, which displays HTML text with embedded links for launching other Insights.
Link and Script Launcher
The Link and Script Launcher displays an HTML document or a textual representation of the results returned by a script. Using embedded links HTML documents can be retrieved, scripts can be run, and Insights can be launched.
Edit the Link and Script Launcher
The edit dialog for the Link and Script Launcher furnishes a simple text window for text, HTML, or script entry. The Link and Script Launcher shares the same Edit Component window as the Static Text Component. See Edit the Static Text Component for more information. Read below for information about Scripts.

The Link and Script Launcher Component can be used to execute multiple scripts. Each line is a separate script, the output of which is pipelined as input to the next script. The final output is displayed in the Link and Script Launcher Component pane in the Insight.
Output Format - choose “text/html” if the output of the script(s) is to be buffered until the </html> tag and then rendered in the Component display pane. Choose “text/plain” if the output of the script(s) is to be output immediately in the Component display pane.
Clear Behavior - choose “empty text” if the Component pane is to be completely cleared upon receipt of a clear Interaction. Choose “original text” to return to the text originally entered in the Component pane when the Insight was built. If the script type was chosen, the behavior of the Component is empty text regardless of the setting of this item, since it is never desirable to display a script to a user.
Embedded Link Format
The links are of the form
<a href=”http://www.networkphysics.com”>Network Physics</a>. If we refer to the portion of the URL with the “http:” as the protocol string, then the following protocol strings can be used in the links:
http: - launches an HTML document in this Components text area
ehttp: - launches an HTML document in an external browser
script: - launches a script in a separate window
iscript: - launches a script within this Component's text area
ascript: - launches Insight Extension (remote script) output to the component. See Insight Extensions and Scripts.
guide: - launches an Insight saved in the local Insight directory.
guideManager: - launches any Insight in the Insight Manager that is saved on the Appliance
guideManagerCategory: - generates a list of all Insights saved on the Appliance in the specified category using the special _name or _{name} variable.
When using the guideManagerCategory link, place the category name in curly braces, e.g., guideManagerCategory:{category}. Wherever you want the name to appear use the _name or _{name} variable to substitute in the Insight name. Both variables must be in the same line of HTML. For example, consider the following HTML:
<html>
<head><base href=”${localUrl}”></head>
<body>
<table width=”100%”>
<tr height=”50” valign=”middle”><td width=”50” height=”50” alignment” valign=”middle” >
<img src="ui/ide/resources/NetworkPhysics-64x64.gif" width="64" height="64" ></td>
<td width="100%" bgcolor="#c5caaf">&nbsp;&nbsp;
<b><font face="arial" size="+2">NetSensory Insights - Audit</font></b></td></tr>
</table>
<hr width="100%">
<table><tr><td>
<table>

<tr><td>&nbsp;&nbsp;&nbsp;&nbsp;<img src="ui/ide/resources/audit_large.gif" alt="" width="50" height="50"></td><td>&nbsp;<a href="guideManagerCategory:{Audit}"><font face="arial" size="+1">_{name}</font></a></td></tr>

</table>
</td></tr></table>
</body>
</html>
The line in bold shows the use of the _{name} variable, which in effect creates a recursive script that walks through the list of Insights in the Audit category and creates (in this case) a table entry for each one, along with the image (audit_large.gif) specified on that line. The resulting Link and Script Launcher pane, which is the entire Insight in this case, will be familiar as the Audit Category Insight supplied with the Desktop Console.

Link and Script Variables
There are several variables or keys that can be used within the href tag to use information passed by the group action. Like other Insight Development Kit variables, these variables require curly brackets around them if there is no space following them.
The following variables may be used to access members of a group that has been passed in by one of the following Interactions:
Group Interactions:
–
_ip or _{ip} will get you the IP address in any group that has been passed in with an IP address
–
_as or _{as} will get you the AS number of any group that has been passed in with an AS number.
–
_group or _{group} will get you the official group path String that is written in project files.
Interval Interactions:
–
_interval or _{interval} will get you the interval from the main time selection or the interval Interaction.
–
_start or _{start} will get you the start time of the interval.
–
_end or _{end} will get you the end time of the interval.
Metric Interaction:
–
_metrics or _{metrics} will get you the list of metrics from the metric Interaction.
Box Interaction:
–
_boxes or _{boxes} will get you the list of appliances from the box Interaction.
Alert Interaction
–
_tasks or _{tasks} will get you the list of alerts from the alert Interaction.
Value Interaction:
–
_limit or _{limit} will get you the limit that was passed in by the limit Interaction.
Miscellaneous:
–
_timezone or _{timezone} will get you the offset from GMT in hours.
Third-Party Integration Variables
The following are special variables that are used for third-party integrations and have no meaning outside of those integrations.
_timezoneIndex or _{timezoneIndex} will get you the timezone index.
_devices or _{devices} will get you the list of device IP addresses.
Embedding Images in HTML Pages
Images may be embedded within the HTML, and the base property may be set in the same way as with Static Text Components, as described in Edit the Label Component.

Insight Launcher
The Insight Launcher Component displays HTML text with embedded links. The links reference other Insights that can be launched from this Component. When an Insight is launched, this Component passes properties such as user selection of groups, metrics, top limit, time interval, and appliance as parameters so that the Insight can be initialized with these pieces of information. In this way, several Insights can be linked together to form a complete solution to the user.
