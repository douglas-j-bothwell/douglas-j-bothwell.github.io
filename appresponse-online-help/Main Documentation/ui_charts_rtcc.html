Response Time Composition Chart
The Response Time Composition Chart shows a stacked region chart with key metrics that contribute to end-to-end response time. This chart makes it easy to identify at a glance which metrics are responsible for Response Time issues. Note—This chart reflects the User Response Time as closely as possible. User Response Time is reflected by the metrics shown minus RTT (in) and RTT (out).
Figure 1
Response Time Composition Chart
 
This chart shows the following metrics:
Connection Setup Time [msec]
Server Response Time [msec]
Payload Transfer Time [sec]
Retransmission Delay [msec]
Round Trip Time [msec]
Round Trip Time [msec]
User Response Time [sec]
Switch to Clients/Servers
You can toggle the set of IPs for which the chart displays Response Time metrics: Servers (the default) or Clients. The “correct” setting depends on the question you want to answer:
Servers—What is the response time experienced by clients who request data from the Servers in this group?
Clients—What is the response time experienced by the Clients in this group?
RTCC charts in the Desktop Console include a “Switch to Clients/Servers” toolbar button. You can also right-click in the chart background and set the Traffic Direction or Reference setting in the Axes tabbed page.
