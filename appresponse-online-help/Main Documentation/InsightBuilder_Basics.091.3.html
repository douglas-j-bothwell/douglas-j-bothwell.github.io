How Insights Work
An Insight is a synergistic collection of tables and charts from the AppResponse user interface. These elements can be linked by various automatic analyses and supplemented with controls to provide interactivity. Once saved, an Insight can be opened as a distinct window within the Desktop Console. Insights provide a way of customizing how data is presented and how you interact with it to get specific summary information or to solve specific problems.
Insights can serve a variety of purposes, including:
Information summaries—predefined displays of commonly viewed information. The displays are automatically populated and generally require limited or no user input. Insights of this type provide a convenient way to monitor performance or provide high-level views to management.
Workflows—interactive “templates” that allow custom views of information, for which users can select data sources as needed. Workflow Insights can capture best practices and simplify the process of solving complex tasks by guiding the user through analyses appropriate to a particular task.
Five Building Blocks
Insights are created using five building blocks collectively called Elements. These building blocks are
Visual Components
Analysis Components
Controls
Interactions
Layers
Visual Components and Analysis Components communicate via Interactions. Controls enable users to set parameters for Visual Objects. Layers, which can be thought of as different “pages”, are a way of specifying the inheritance of Components and Interactions between different parts of an Insight.
Visual Components
Visual Components are just what the name implies: the parts of an Insight that a user can see and interact with, such as charts, topology maps, selectors, tables, text labels, and toolbars. Almost every element of the AppResponse user interface is available as a Visual Component for an Insight.

Analysis Components
Analysis Components are behind-the-scenes components that transform data in a variety of ways. For instance, the Drilldown Analysis Component takes a Business Group as an input and outputs the top members of a drilldown from that group.
Controls
Controls are the various elements of Visual Components (such as buttons, text fields, and combo boxes) that enable users to enter information and change settings. For instance, the “By” control on a Chart or Table lets the user specify the metric by which displayed groups are ranked.
Note: There are also controls within the IDK that you use to configure the appearance and behavior of the Insight Elements. When we discuss Controls (with an upper-case “C”), we mean the ones available to a user within an Insight.
Interactions
As noted above, Interactions enable Components to communicate among themselves. An Interaction automatically gets information of a specified type from one Component (its Producer) and transfers that information to another Component (its Consumer).
For instance, in Figure 2, selecting a new limit using the Top Control in the Toolbar causes a Value Interaction to get that limit and send it to the table (the Consumer of that Interaction). This allows the table to reflect the new limit in the number of items it displays.
Layers
Layers, which can be thought of as different “pages” in an Insight, offer a means of constructing multi-part Insights using a variety of inheritance methods that determine which elements of each Layer are available in other Layers. They are one of the primary means of preserving screen real estate (which is important because, for efficiency, all parts of an Insight should be visible without scrolling). Layers can be configured for sequential access or can be non-sequential.
