Domains
A domain is a set of AppResponse Appliances that collect and analyze data cooperatively. Domains provide the following benefits:
Centralized Management—Configuration changes can be applied to all appliances within a domain from a single location. This allows domains to be managed centrally.
Transparent Data Access and Automatic Data Aggregation—Each AppResponse Appliance within a domain collects data from a different network segment. Data can be retrieved from multiple appliances through a single AppResponse Console project transparently. Data can also be automatically aggregated to the Domain Director, providing a single location for data analysis.
Consolidated Reports and Alerts—Reports and alerts generated across the distributed domain can be defined, executed and examined in a single location.
Director-100 and Director-200 models support domains of up to 20 appliances; Director-300 models support domains of up to 40 appliances.
