Installing the Director
After you’ve determined where to install the director within your network, examine the shipping materials then follow the Director installation procedure to perform physical installation, wiring, and configuration of the director. This procedure is described in Director Installation Procedure.
Director Material Inventory
WARNING: Do not discard the original shipping carton. It is required for all returns and exchanges or warranty is void.
Before physically installing and wiring the director, examine the director back panel and the items that shipped with the director to familiarize yourself with the system. The following sections walk you through wiring the power, serial port and management network interface. The management interface is used for general purpose communication and must be connected to a network segment that allows it to communicate with end-user desktop machines and the appliances that make up the domain.

Director Installation Procedure
This procedure provides a general outline of the tasks required to wire, install and configure the director. Each step refers to a detailed procedure.
Step 1—Rackmount and Wire
Rackmount the director and wire the electrical, serial port and management network interface as described in Rackmount and Wire the Director.
Step 2—Initial setup
Use the command line interface to perform initial configuration of the director using the same procedure documented for the appliance (see Step 3: Initial Setup using the CLI).
Step 3—Complete setup
Use the web interface to complete the director configuration, as described in Complete Setup using the Web Interface.
After this procedure is complete, director configuration is finished. Install the AppResponse Console on a desktop machine to access the director (as described in Installing the Appliance); then follow the domain procedures documented in Domains.
Rackmount and Wire the Director
Procedure 1
Rackmount and Wire the Director
1.
Rackmount the director by following the instructions listed in the “Intel Server Chassis and Platform Bracket Kit Installation Guide”.
This document is in the small materials box included in the director shipping container.
2.
Connect the female plug of the supplied AC power cord to the AC input port on the back of the director next to the power-supply fan, and then connect the male plug of the power cord to a conditioned power outlet.
3.
Use a terminal emulator program (such as hyperterm on Windows or tip on Unix) to connect to the director’s serial port with the provided serial cable.
Use the following terminal-emulation settings: 115200 baud, 8 data bits, no parity, 1 stop bit, and no flow control.
Note: Nothing appears on the serial port console until the appliance is powered on, which occurs in the next procedure (see Step 3: Initial Setup using the CLI).
4.
Connect an RJ45 CAT 5 Ethernet patch cable between the director Management interface and a switch or router.
End of Procedure 1
 
 
With the rackmount and wiring complete, proceed to the next procedure.
Initial Setup using the CLI
Procedure 2
Initial Setup
1.
Press the power switch on the front of the director to turn it on.
Watch the serial port console for any error messages during the boot process.
The login prompt appears approximately 2 minutes after the appliance is turned on.
2.
Type admin and press Enter.
3.
At the password prompt, press Enter. (There is no default password.)
The appliance requires that a password be set on the admin account the first time the CLI is accessed.
4.
Use the passwd command to change the password.
4.1.
At the prompt, type passwd, then press Enter.
4.2.
At the Old Password prompt, press Enter for a null password.
4.3.
At the New Password prompt, enter a password and press Enter (you need to do this twice for confirmation).
Note: The initial CLI login is admin with a null password that you must change when you install the director. The admin login has administrative privileges and can be used to create additional CLI logins as well as web interface logins.

You can use the usermgr command to create a new user with read-only or administrative privileges.

While using the CLI, you can view the list of available commands by typing: commands
5.
To display the system configuration menu, at the prompt type setup, and then press Enter.
6.
To begin the interactive system configuration, at the setup prompt type config, and then press Enter.
7.
For each of the following parameters, type the appropriate value:
Caution—IP Address, Netmask, and Gateway changes may affect the visibility of this system on the network:
 
Table 2
CLI Parameters
Parameter
Value Description
Hostname
Type the host name for the system. Enter the hostname only, do not include the domain name. The hostname must be under 63 characters long, contain only letters, digits, or dashes, and start with a letter and end with either a letter or digit.
IP Address
Type the IP address for the management interface.
Netmask
Type the netmask for the management interface.
Default Gateway
Type the primary gateway IP address used by the management interface to reach other networks.
Domain
Type the default, fully qualified domain name for the system used during DNS resolution. Do not include the hostname. Each portion of the domain name must be under 63 characters long, contain only letters, digits, or dashes, and start with a letter and end with either a letter or digit.
The CLI should only be used to set the network parameters at install time. All subsequent changes should be performed using the web interface.
8.
Verify your new settings: At the setup prompt type showall, press Enter.
9.
To save the changes if the settings are correct, at the setup prompt type commit, and then press Enter.
10.
Because these changes require a restart to take effect, when you are asked if you want to reboot the system at the setup prompt type yes, and then press Enter.
If the login prompt appears approximately 2 to 3 minutes after you reboot the system, the reboot process is complete.
To quit the CLI without rebooting, type no at the reboot prompt, and then type quit and press Enter.
End of Procedure 2
 
 
With the initial setup of the director finished, proceed to Step 3 of the installation procedure (Complete Setup using the Web Interface). If this is a first time installation, be sure to read the next section for information on how to access the web interface.
Procedure 3
Access the Web Interface
Note: To successfully connect to the web interface you must be able to access the director from your desktop machine via TCP ports 8080 and 8443.
1.
Start a web browser and go to the director web interface by opening one of the following URLs:
• 
http://director-hostname:8080
• 
http://director-ip-address:8080
Note: It takes approximately 5 to 10 minutes after the director is rebooted for the web interface to be available.
This automatically redirects the browser to a secure (SSL) connection on TCP port 8443.
The browser may display the certificate validation popup window. Accept the certificate to proceed to the Login page.
2.
At the Login page, enter a valid username. During initial set-up, log into the director using the admin account.
3.
Type in the password for the user account.
4.
Click Login.
End of Procedure 3
 
 
Note: For full support of all features, use Internet Explorer version 5.x or Netscape 6.2 or later.
If the username and password supplied were valid, the main page of the web interface is displayed. This page provides a brief overview of the functionality available within the web interface. It also displays the name and access rights of the user account used to access the web interface.
Procedure 4
Complete Setup using the Web Interface
1.
From the director web interface, click the system menu, then click setup.
 
2.
Under Network Configuration Settings, verify the network parameters that were set during the initial CLI configuration (described in Step 3: Initial Setup using the CLI):
• 
Host Name
• 
Netmask
• 
Domain
• 
Gateway
• 
IP Address
Caution—IP Address, Netmask, and Gateway changes may affect the visibility of this system on the network.
3.
From the Management Interface Speed/Duplex Settings drop-down list, select the management network interface card’s mode of operation (or media type) used for communication.
The web interface displays the speed/duplex settings and current status of the management interface. The management interface speed/duplex is set to autoselect by default. However, the speed/duplex settings should be configured to the same value set on the router/switch port.
Caution—Changes to the Management Interface settings may greatly affect the responsiveness of the system.
If the switch port to which the management port is connected is forced to a specific speed or duplex setting, the management interface media type must be configured to the same settings; otherwise, select autoselect.
4.
(Optional) Under Domain Name Servers, type the DNS Server IP addresses used by the director to perform network IP address resolution (one server address per line).
5.
Under Time Settings, enter the following:
• 
Time Zone—Local Time Zone in which the director is used.
• 
Network Time Protocol (NTP) Servers—Servers used by the director to synchronize its clock (entered as one server address per line).
To prevent clock drift and to ensure that director time matches the time on other systems, it is strongly recommended that the AppResponse appliance clock be synchronized with a known time source using the NTP protocol. Public NTP servers (e.g., clock.isc.org) are available if your organization does not have internal servers. The IP address of the NTP server should always be used rather than its hostname. Refer to the 'Public NTP Server List' on http://www.ntp.org/ for a complete list of public NTP servers and access policies.
Note: The system automatically restarts the AppResponse appliance if necessary.
6.
Under history, the System Log displays details about all configuration changes, restarts, and shutdowns that have occurred on the system. Review the system log for any errors that may have occurred during the initial configuration.
Note: Whenever you change settings, consider backing up the system configuration (search for “backup” in the online help).
Scroll down the page (if necessary), and then click Apply to save the configuration settings.
7.
With the web interface running (as described in Procedure 3), click logout at the top of the page.
Note: You can log in again at any time, as described in Procedure 3. If you close the browser window without clicking logout, the web interface does not allow that user account to make administrative changes for 30 minutes. The admin user is exempt from this rule.
End of Procedure 4
