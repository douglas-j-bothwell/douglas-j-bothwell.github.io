Modifying a Default Report
This section discusses how to modify a default report using the Report Manager and IDK. The following topics are discussed:
Workflow: Modifying a Default Report in IDK
Tutorial: Modifying an Default Report in IDK
Workflow: Modifying a Default Report in IDK
Use this workflow to create a report by modifying one of the default (built-in) reports.
1)
In the Insight Manager, duplicate the Insight associated with the report you want to modify. (The Insight will have a name similar to that of the associated report and be listed in the Reports category of the Insight Manager.)
2)
After the duplicate Insight opens in IDK, edit it as desired. Be sure to follow the requirements and best practices given in Guidelines for Creating Insight Reports.
3)
Save the edited Insight to your local Insights directory and close IDK.
4)
Open the edited Insight (Insights > Local Insights > Insight_name). Right-click on any legend in the Insight and choose Publish > Publish Report… from the pop-up menu.
Note—If Publish Report does not appear in the pop-up menu, reopen the Insight in IDK and edit the legend’s Component to select the Publish as Report checkbox on the Composition tabbed page.
5)
In the Report Manager, configure the new report using the Identity, Actions, and Schedule tabbed pages.
6)
Apply the changes.
Tutorial: Modifying an Default Report in IDK
This topic is a tutorial showing how to modify one of the Insight reports supplied with AppResponse. It describes all steps, from start to finish, for creating the modified report.
For this tutorial, you will modify the “Detail Summary - Total Traffic for an Hour” report. The default report includes three charts, showing the top four applications, talkers, and conversation pairs on your network. You want to concentrate more on the applications, so you will:
increase the number of applications charted from 4 to 12
add a table with information about the top 12 applications
remove the conversations chart
Begin by making a copy of the Insight on which the “Detail Summary - Total Traffic for an Hour” report is based.
Procedure 1
Creating a Duplicate Insight
1.
In the Desktop Console, choose Insights > Insight Manager…
The Insight Manager opens, with a table listing all available Insights. The Insights used to create the default reports are listed in the “Report” category and are named similarly to the corresponding report.
Hint—To find an Insight more easily, click the dot in the Category table heading. This will sort the table by category, thus grouping the report Insights together.
2.
In the table, right-click on the row named Detailed - Total Traffic for an Hour and choose Duplicate from the pop-up menu.
The Insight Development Kit Dialog window opens, showing the duplicate Insight. The duplicate is added to the table in the Insight Manager under the name Copy of Detailed - Total Traffic for an Hour.
3.
In the Insight Development Kit Dialog window, choose File > Properties…
4.
Change the Insight Name to <your_initials> App Traffic Report, then click OK.
5.
Choose File > Save As… Save the duplicate in your local Insights directory as <your_initials> App Traffic Report.xml.
Note: You don’t need the original Insight copy any more. Later (after the IDK is closed), you can delete Copy of Detailed - Total Traffic for an Hour from the Insight Manager list.
End of Procedure 1
 
 
Now that you have a duplicate Insight, you can modify it to make the changes you want to see in the Insight report.
Procedure 2
Modifying the Insight
1.
Increase the number of charted applications:
1.1.
Double-click on the Chart:topMultigrp:Apps icon to open it for editing.
The Edit the Component window opens.
1.2.
Right-click on the legend (below the graph) and choose Settings…
The Chart Settings dialog box opens.
1.3.
Click on the Groups tab.
1.4.
Change the Top field from 4 to 12, then click OK.
The Chart Settings dialog box closes and the Edit the Component window updates to show 12 applications. It should appear similar to the following figure.
 
1.5.
Click OK.
2.
Create an application table:
2.1.
Drag a Top Group Table from the toolbar into the Layout Grid.
2.2.
Rename the table Table:top:Apps.
2.3.
Right-click on the icon and choose Edit Component.
2.4.
Click the Table Settings icon in the top right of the Edit the Component dialog box.
The Table Settings dialog box opens.
 
2.5.
Change the Group: to Application and Top: to 12.
2.6.
Click the Toolbar tab and deselect the Show Toolbar checkbox.
2.7.
Click OK.
The Table Settings dialog box closes and the Edit the Component window updates to show 12 applications.
2.8.
Click OK.
3.
Delete the conversations chart:
3.1.
Right-click on the Chart:topMultigrp:Conversations icon and choose Remove Component from the pop-up menu.
3.2.
Right-click on the Label:TopConversations icon and choose Remove zwComponent from the pop-up menu.
4.
Update the layout:
4.1.
Click the Layout Grid icon in the Layout Grid toolbar.
4.2.
Drag the Chart:topMultigrp:IPs icon down one cell (from 5:1 to 6:1).
4.3.
Drag the Label:TopIPs icon down one cell (from 4:1 to 5:1).
4.4.
Drag Table:top:Apps from the treeview into cell 4:1 (directly below the applications chart).
 
4.5.
Click OK.
5.
Change the Insight title:
5.1.
Double-click on the TextBox icon.
5.2.
Edit the HTML to change the title to Detailed Summary of Application Traffic for an Hour.
5.3.
Click OK.
6.
Test the layout:
6.1.
Click the Test/Layout button.
6.2.
If necessary, vertically resize the applications table to show all 12 rows.
The resized preview should resemble the following figure.
 
6.3.
Click OK.
7.
Choose File > Save.
8.
Click Cancel to close the IDK window.
End of Procedure 2
 
 
Now that the Insight is modified and shows the information you want, you can publish the Insight as a report.
Procedure 3
Publishing the Insight Report
1.
Set the Project Time to a 1-hour time range.
2.
In the Desktop Console, choose Insights > Local Insights > <your_initials> App Traffic Report.
The Insight opens.
3.
Right-click on any legend in the Insight (for example, the application names below the Top Applications chart) and choose Publish > Publish Report… from the pop-up menu.
Note: If Publish Report… does not appear in the pop-up menu, reopen the Insight in IDK and edit the legend’s Component to select the Publish as Report checkbox on the Composition tabbed page.
The Report Manager opens, listing the Insight report as New Report 1.
4.
Configure the report:
4.1.
Click the Identity tab and make the following settings:
• 
Set Name: to Detail Summary - App Traffic for an Hour
• 
Set Description: to Time history of apps and talkers, for throughput.
Note—Generally, you will want to use the Schedule tab to generate the report on a regular schedule. For this tutorial, you will keep the default settings and generate the report manually.
4.2.
Click OK.
4.3.
When the changes have been successfully committed, click OK.
5.
Close the Insight.
6.
In the Report Manager, delete the unneeded Copy of Detailed - Total Traffic for an Hour.
7.
Close the Report Manager.
End of Procedure 3
 
 
To see your new Insight report, go to the AppResponse web interface and click the Reports tab. You will find it on the Console Reports page under Unfiled (Public).
