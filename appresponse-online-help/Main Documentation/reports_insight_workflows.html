Creating Insight Reports
This section describes the following workflows:
Report Manager: Creating Insight Reports
IDK: Creating Insight Reports
In general, you should use the Report Manager to create Insight reports. Insight Development Kit is useful in certain situations, but should be avoided for routine use because it contains features beyond what is supported by Insight reports. These additional features let you create complex Insights, but are not supported in report publication and can lead to undesirable results if published.
Report Manager: Creating Insight Reports
When creating an Insight report from scratch, it is best to use the Report Manager. The following steps outline this workflow:
1)
Plan the report content and make a sketch of the layout.
2)
In the Report Manager, create a new report of type “Blank Insight”.
3)
Build and test the Insight report, being sure to meet the requirements and best practices given in Guidelines for Creating Insight Reports.
Note—The components provided by Report Manger for building the insight report are a subset of those available in IDK. You can refer to the Insight Development Kit User Guide for assistance with them.
4)
Save and close the Insight report.
5)
In the Report Manager, configure the new report using the Identity, Actions, and Schedule tabbed pages.
6)
Apply the changes.
You can find a step-by-step tutorial that follows this workflow in Creating an Insight Report.
IDK: Creating Insight Reports
Note: You must have Administrator privileges to use Insight Development Kit.
Generally, you should create Insight reports from the Report Manager using the Blank Insight report type (as described in Report Manager: Creating Insight Reports). However, there are situations in which you must use the Insight Development Kit (IDK). For example, you’ll need to use IDK if you want to:
Deploy an existing Insight as a report
Modify one of the default reports
Note: IDK allows you to create Insights containing elements that are not supported in reports. As a result, not all Insights can be published successfully as reports.
Workflow for Deploying a Custom Insight as a Report
Use this workflow to create a report based on an existing Insight.
1)
Using Insight Development Kit, open the Insight you want to publish as a report.
2)
Verify that the Insight meets the requirements and best practices given in Guidelines for Creating Insight Reports. If not, close the Insight and follow the Modifying a Default Report.
3)
Close IDK.
4)
Open the Insight (Insights > Local Insights > Insight_name). Right-click on any legend in the Insight and choose Publish > Publish Report… from the pop-up menu.
Note—If Publish Report does not appear in the pop-up menu, reopen the Insight in IDK and edit the legend’s Component to select the Publish as Report checkbox on the Composition tabbed page.
5)
In the Report Manager, configure the new report using the Identity, Actions, and Schedule tabbed pages (for details, see Report Manager: Creating Insight Reports).
6)
Apply the changes.
