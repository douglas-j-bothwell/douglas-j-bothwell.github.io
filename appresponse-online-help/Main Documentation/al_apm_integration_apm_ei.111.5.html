SNMP Query Insight
The SNMP Query Insight is useful when you identify a network host for which you want to view detailed performance metrics. This Insight lets you navigate through all MIB data collected in NetSensor, and is the only way to view metric classes that were user-defined in APM EI.

Workflow Description
The following steps outline the general workflow for this Insight:
1)
Identify the host that you want to analyze.
You can open this Insight from any individual IP address in the Desktop Console: right-click on the address and choose Analyze In > Element Polling > SNMP Query. This provides an end-to-end workflow in which you
• Identify a host with problematic performance (slow servers, for example) in the Desktop Console
• Adjust the project time (if needed) to the time window where you observe the problematic performance
• Send the IP address to the SNMP Query Insight
• Analyze performance on the host by selecting and viewing different performance metrics
If you already know the host you want to analyze, you can open the Insight directly (Insights > Element Polling > SNMP Query) and select the host from the Polled System List.
➥ After you select a host, the appliance queries NetSensor for all classes of metric data that might have been collected for that host within the project time window.
2)
Adjust the project time (if necessary) to focus on the time window of interest, if you have not already done so.
3)
If the Querying Metric Classes progress window appears, wait until this window closes and the Metric Classes on System table is populated.
4)
Select a metric class in the Metric Classes on System table.
➥ The Metric Class table is populated with all metrics for that class.
5)
Click on a metric cell of interest in the Metric Class table. You can Ctrl-click multiple metric cells to overlay their graphs in the chart.
➥ The time-series chart shows the variation in that metric over the project time window.
6)
Repeat this workflow as needed to view all metrics of interest on the host.
