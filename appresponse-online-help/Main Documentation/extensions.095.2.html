Creating an Insight Extension and Local Script
In general, Insight Extensions are deployed to an Appliance using a signed jar. At this time, all jars need to be signed by Riverbed. However, an SE or partner that has root access to an appliance can develop an Insight Extension by placing it directly on the box and editing it directly on the appliance. All Insight Extensions must be located in the following directory:
      /u1/cliext/insights/[ExtensionName]
where ExtensionName is the name of the extension. This name how the Extension is referred to in the Insight. In this directory you must place an executable file with the name run. This executable can then call other scripts and resources in the directory and on the appliance.
Local Scripts are placed on the user's hard drive and are referred to using the full path to the script on the native operating system. On a windows box scripts are often put into .bat files. An example:
      C:\Guides\InsightCommunity\Scripts\ping.bat
On a Windows box wherever a Script can be entered it is also valid to enter a DOS command.
