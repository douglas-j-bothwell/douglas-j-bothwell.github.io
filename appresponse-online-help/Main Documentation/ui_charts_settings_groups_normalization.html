Chart Settings: Groups: Normalization
Normalization is useful when you want to view the metric values as fractions or percentages of a higher (total or maximum) value. This option works differently, depending the window in which it appears:
–
Multi-Group Time Series Chart and Top Multi-Group Time Series Chart - With Normalization enabled, and a maximum value for the metric specified, the chart shows each data point as a percentage of the maximum value.
You should specify the maximum possible value for the metric shown in the chart.
–
Pie Chart and Top Pie Chart - With Normalization enabled, and a normalization group or value specified, the pie chart includes an “OTHER” slice that represents traffic that is part the total chart metric, but is not included in the selected groups.
This option is useful when you want to show a total chart metric that is greater than the sum for all selected groups. For example, you might have a chart that shows throughput for the five groups at HQ, and you also want to the chart to show the total throughput (for all groups) at HQ. In this example, “HQ” would be the normalization group. You should specify a normalization group that represents the total traffic such that <selected_groups_total> + <other_groups_total> = <normalization_group_total>.
To select the normalization group, click the Add Group button and select an IP address or group.
