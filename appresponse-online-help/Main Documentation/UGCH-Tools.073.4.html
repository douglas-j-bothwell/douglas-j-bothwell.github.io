Importing and Exporting Data
The data available in the appliance can be exported to a flat file and used in external applications. This includes configuration information, such as the list of Business Groups. This data can be edited externally and imported back into the Desktop Console.
Importing and Exporting Configuration Data
Many of the managers within the Desktop Console support import and export options. These options allow you to save configuration data in a CSV (comma-separated value) file for use in an external application. Similarly, data can be imported in CSV format, allowing configuration information to be readily imported from a database. The following managers support the import and export options:
Defined Application Manager
Port Information Manager
Alert Manager
Business Group Manager
Business Group Link Manager
AS Information Manager
Director Group Manager
Device/Interface Manager
Preferred IP Manager
To export data to a CSV file, launch the desired manager (e.g., Business Group Manager) and select File > Export. The Desktop Console prompts you for a name of a file to save the resulting CSV file. This file can be edited with Microsoft Excel or another external tool, then imported back into the Desktop Console by selecting File > Import when the appropriate manager window has the focus. Click OK or Apply to save the imported list into the database.
When a CSV file is imported, any lines in the imported file that do not match the expected format will generate a warning. This includes blank lines, and lines with blank fields in the CSV file. Be sure to check the end of the CSV file to ensure that there are no trailing blank lines (or lines with blank fields) to avoid these warnings.

Lines beginning with ‘#’ are comment lines. Comment lines provide the following information:
Version—The current software version (necessary for handling old file formats).
Timestamp—The time at which the export occurred.
Source—The name of the appliance from which the data was exported.
StartTime & EndTime—The start and end times of the project time selection.This is useful for data from tables, charts and topologies but can be safely ignored for configuration data.
Data Type—Which type of manager the data was exported from.
Data lines appear immediately following the comment lines. The first data line is a header. The header line lists the names of each column of data within the file. These columns correspond to the columns displayed for each row displayed in the Defined Application Manager.
The various managers use different CSV file formats, as described in the following sections. To familiarize yourself with the CSV file format of the manager you are working in, we recommend that you perform an export from that manager and examine the contents of the CSV file. It is best to edit a CSV file with an ASCII text editor such as Wordpad or Notepad, so that you do not accidentally insert any control characters into the file (as can happen with a program such as Microsoft Excel).
Some managers use CSV file formats that separate values with commas, while others use semicolons. The proper format is specified for each manager, below, where such applies (Application, Alert, Business Group, and AS Managers).
Defined Application Manager Import/Export File Format
The import/export file for the Defined Application Manager has the following columns:
Name—The name of the application that appears in the group column of a table.
Description—The description of the application that appears in the information column of a table.
Recognized App ID—An internal ID assigned to this Recognized Application based.
IP Protocol—The transport protocol for the application. Valid values are 1 for non-TCP-non-UDP, 6 for TCP and 17 for UDP.
Port Ranges—The transport port number(s) for the application.
ID—An internal ID assigned to this entry. This field is ignored during import, but a comma is still needed in the CSV file.
Type—1 = Standard Application, 2 = Server Application, 3 = Web Application.
Status—Shows whether the entry is Active or Inactive. “W” means the application is active and currently collecting data; “I” means the application is inactive and not currently collecting data.
Mode—Called “Track application turns” in the Desktop Console. S = TCP server-side turns enabled, C = TCP client-side turns enabled, N = Turns not enabled.
Servers—The IP address list of servers for a Server or web application.
URL—The list of URLs for a web application. The host name and path segments of the URL are separated by the “|” character.
Appliance—The list of appliances to which the application is distributed. This field is only included in files exported from a director. It does not appear in files exported from an appliance.
The column values within each data line are separated by commas. A literal comma that is part of the value of a column is translated to \. in the export file. Use the same character sequence if you need to include a comma as part of a column value.
Port Information Manager Import/Export File Format
The import/export file for the Port Information Manager has the following columns:
Protocol—Shows whether the application is TCP or UDP.
Port—The port number for the application.
Name—The name of the application.
Description—The description of the application.
Alert Manager Import/Export File Format
The import/export file for the Alert Manager has the following columns:
Name—The name of the alert.
Description—The description of the alert.
Version:Group—The group to which the alert applies. The group is identified using the group path notation and generally should not be modified by hand. The version number is included for backward compatibility.
Frequency—The frequency, specified in seconds, at which the alert condition is evaluated. The frequency is set to zero for adaptive alerts. Valid values for static alerts are 60, 300, 3600 and 86400.
Condition—The condition that is checked to determine whether an alert should be triggered. The condition is specified using an internal representation that should not be modified by hand.
Active—A boolean value which specifies whether the alert is active (True) or inactive (False).
Days—The Days parameter is a bitmask that specifies the days for which an alert is active. For example, a value of 127 is 1111111 in binary. This indicates that the alert is active all 7 days of the week. A value of 96 is 1100000 in binary indicating the alert is valid on Saturday and Friday. This parameter is only valid for static alerts. The value zero is used for adaptive alerts.
Start—The start time of the alert. It is expressed in minutes from 12:00am. This parameter is applicable to static alerts only.
End—The end time of the alert. It is expressed in minutes from 12:00am. This parameter is applicable to static alerts only.
Priority—The priority of the alert. Valid values are 2 (Minor), 3 (Major) and 4 (Critical). This parameter is applicable to static alerts only.
Action—The action to take when the alert is triggered. e.g., “email=verbosity:true\.anonymous@yahoo.com;snmp;”.
Trigger—This parameter identifies the alert triggering mechanism. Valid values are 0 (on crossing) and 1 (continuous). For adaptive alerts, the value is always zero.
Segment—The dashboard section in which the alert appears. Valid values are 1 (client), 2 (network) and 3 (server).
Type—The type of the alert. Valid values are 0 (static), 1 and 2 (adaptive alerts and adaptive alert components)
GroupList Type—T (Total) indicates a Standard Static Alert. For Static-Any Alerts, this field indicates the set type specified for the Group List in the Group tab of the Defined Application Manager. A = All, I = Set of, Inclusive,
E = Set of, Exclusive (Static Alerts). This field is blank for Adaptive Alerts.
GroupListData Type—The type of group in the alert. IP = IP Address, Business Group = Business Group, Client AS = Dest AS or ISP AS, Application = Defined Application, Other Application = Other Application, IP Conversation = IP Conversation.
GroupListData—The actual data in the first group field in the Group tab.
AppList Type—Blank for Standard Static and Adaptive Alerts. For Static-Any Alerts, this field indicates the set type specified for the Application List in the Group tab of the Defined Application Manager. T = Total (And not checked),
A = All, I = Set of, Inclusive, E = Set of, Exclusive.
AppListDataType—The type of application in the Application List of the Group tab. Application = Defined Application, Other Application = Other Application.
AppListData—The actual data in the Application List field.
Guide Links—The names of the Insight(s) to which the alert is linked in the Problem Management Dashboard.
Appliances—The list of appliances to which the application is distributed. This field is only included in files exported from a director. It does not appear in files exported from an appliance.
The column values within each data line are separated by semicolons.
Business Group Manager Import/Export File Format
The import/export file for the Business Group Manager has the following columns:
Name—The name of the Business Group.
Description—The description of the Business Group.
Members—This parameter lists the IP addresses and Autonomous Systems that are part of the Business Group. IP address ranges are denoted with a hyphen (e.g., 1.1.1.1-2.2.2.2). AS numbers are denoted with a single number (e.g., 13). The member list is delimited using the semi-colon.
Id—An internal representation of the Business Group. This field is ignored on import.
Appliances—The list of appliances to which the application is distributed. This field is only included in files exported from an director. It does not appear in files exported from an appliance.
Collect on Director—Controls the aggregration of Business Group data to the director for multiple Business Groups.
If this column is not in the import file, the import sets collect on director for existing groups to its existing value, for new groups it is set to none.
If this column is in the import file, the following rules are observed:
For new groups, the collect on director setting is set to exactly what is in the owners column entry for that group. If this entry is empty, the collect on director is set to “none.”
For existing groups when there is an owners column entry for that group, the collect on director setting is set to this value. If the owners column entry is empty for this group, the collect on director is set to its current value in the Business Group Manager, preserving all the data that was collected on the director for that group. For existing groups the only way to remove the collect on director setting in the import is to place the word None in the owners column for that group. This clears the collect on director setting for that group.
The column values within each data line are separated by semicolons.
Business Group Manager Importing
The order of entries in a CSV file is very important when importing into the Business Group Manager. Always follow these rules:
Business groups must appear before Business Group containers in the CSV file.
Business group containers must appear before their parent Business Group containers in the CSV file. i.e. a Business Group container cannot include to a Business Group container that appears below it in the CSV file.
Business Group Link Manager Import/Export File Format
The import/export file for the Business Group Link Manager has the following columns:
Name—The name of the Business Group link.
Description—The description of the Business Group link.
Group 1 Type—This value must always be 78 for legacy reasons.
Group 1 Name—The name of the first Business Group in the Business Group link definition.
Group 2 Type—This value must always be 78 for legacy reasons.
Group 2 Name—The name of the second Business Group in the Business Group link definition.
Id—An internal representation of the Business Group link definition. This field is ignored on import.
AS Information Import/Export File Format
The import/export file for the AS Information Manager has the following columns:
AS—The number of the Autonomous System.
Name—The name of the Autonomous System.
Description—The description of the Autonomous System.
Appliances—The list of appliances to which the application is distributed. This field is only included in files exported from a director. It does not appear in files exported from an appliance.
The column values within each data line are separated by commas. A literal comma that is part of the value of a column is translated to \. in the export file. Use the same character sequence if you need to include a comma as part of a column value.
Director Group Manager Import/Export File Format
The import/export file for the Director Group Manager has the following columns:
Name—The name of the director group.
Description—The description of the director group.
Type—The type of the director group. Valid values are “ISP” or “Total Traffic”.
Members—An internal representation of the members of the director group. The value of this field should not be modified by hand.
Preferred IP Manager Import/Export File Format
The import/export file for the Preferred IP Manager has the following column:
Member—An IP Address or CIDR block (e.g., 192.168.0.0/16) within the IP address list.
Device/Interface Manager Import/Export File Format

Display Name

SNMP Settings

Interface Description
Link Layer

Renaming Multiple Devices Using Data Import/Export
You can use the data export/import feature to specify custom names for multiple devices at once:
1)
In the Java console, open the Device/Interface Manager and choose File > Export. This exports all Device/Interface Manager data to a .csv file.
2)
Open the .csv file and edit the following fields for each device of interest:
–
Device/Interface Name - Enter the custom device name
–
Use Device Name - Set this to FALSE
3)
Save the file and choose File > Import.
Business Group Manager Importing
The order of entries in a CSV file is very important when importing into the Business Group Manager. Always follow these rules:
4)
Business groups must appear before Business Group containers in the CSV file.
5)
Business group containers must appear before their parent Business Group containers in the CSV file. i.e. a Business Group container cannot include to a Business Group container that appears below it in the CSV file.
Illegal Characters in Import Files
Importing configuration items definitions from a CSV file results in an error if certain characters appear in the CSV file. The following characters are not valid in the Name parameter for Business Groups or reports.
Backslash (\)
Comma (,)
Forwardslash (/)
In addition, letters that are not part of the English alphabet are not yet supported by the import mechanism.

Mark to Export
The Mark to Export feature can be used to identify specific cells in a table to save the corresponding time series metric data to a CSV file rather than exporting the entire table.
1)
Right-click the desired cells and click 'Mark to Export'. The cell turns gray to indicate that it has been marked for selective export.
2)
Select “Export” from the File menu.
Snapshots
Any view can be saved to a PNG image file by selecting Snapshot from the file menu. PNG images can be viewed in any recent browser. Before taking a snapshot, it is a good idea to enable the Information Panel for the view. The information panel includes the following information for the view:
Time selection
Metric granularity
Data source (appliance hostname)
Email Snapshots
You can email a snapshot from any table and chart (located in the right-click menu) or Insight (click the Email button in the toolbar).
Snapshots are emailed from the appliance or director to which the project is connected, not the computer hosting the Desktop Console.
