Network Performance Metric Definitions
The following table describes the Network Performance metrics available in AppResponse.
 
Table 4
Network Performance Metrics 
Metric
Definition
Packet Loss [%]
Inbound / Outbound
Percentage of TCP packets that were retransmitted. A high percentage of retransmissions often indicates packet loss in the network, but retransmissions may also be caused by other end-to-end TCP effects.
This metric is a measure of lost packets as a percentage of all packets observed for the Group of interest. Unlike packet loss reported on routers, which measures loss on one link only, this metric measures loss on all links where packets for the Group of interest were observed.
Packet Retransmission Rate [#/sec]
Inbound / Outbound
Average number of TCP packets retransmitted per second.
Retransmission Delay [msec]
Inbound / Outbound
Average delay per TCP Turn due to packet retransmissions.
Retransmission Delay is calculated as Data Transfer Time [sec] multiplied by an empirically-derived function of packet loss.

Round Trip Time [msec]
