Indicator
The Indicator Component creates a label with an image that displays a colored light depending on whether a specified condition is Critical, Major, Minor, or Normal. It may be considered a hybrid Component, as it is both a Visual Component and an Analysis Component that evaluates a condition.
Edit Indicator Component
The Edit Dialog for the Indication Component lets you set the conditions to be evaluated as Minor, Major, or Critical. You may combine up to three conditions using simple Boolean logic: “and” or “or”.
Figure 82
Edit Indicator Component
 
You may also set the following parameters:
Background Color—enter RGB values or click Select Color... to use the Set Background Color dialog, which operates in the same fashion as the Set Color Dialog.
Text Alignment—Left, Right, or Center
Icon Type - select Show Icon with Highest Severity to display only the icon representing the highest severity alert in the groups evaluated. Select Show Icon with All Severities to display all four icons in a row, with only the icon representing the highest severity alert in the groups evaluated lit up in color, and the rest gray.
Group Index—See Using the Group Index.
Use group with specified index - check this (default) to evaluate only the group received in the group Interaction for alert conditions. If unchecked, the Indicator Component evaluates all groups in the Interaction and displays the highest severity among the alerts detected.

Tab Container
The Tab Container Component displays a set of Hidden Visual Components in a tabbed pane.
Edit Tab Container
The Edit Dialog for the Tab Container lets you associate tab names and icons with the Component displayed in each tab.
1)
First add the Components you want in the Tab Container to the Insight as hidden Components
2)
Add the Tab Container to the Insight Development Kit grid.
3)
Right click on each hidden Component in the Element pane of the Insight Development Kit outline area and select Add to Tab for a menu of all the Tab Containers in that Layer of the Insight. Select the desired Component. Fill out the tab name in the Rename dialog that appears and click OK. Repeat until all desired hidden Components have been added. You may use any Interaction variable (e.g., $group) in a tab name Insight Variables.
4)
Edit the Tab Container.
Figure 83
Edit Tab Container
 
Right-click on a row to call up the context menu, which lets you Remove, Move Up, and Move Down a Component. Moving a Component up moves its tab to the left. You may also edit the tab name in this dialog, and select the desired tabs placement: Top, Bottom, Left, or Right.
Enter a URL variable combined with an icon filename (as described in Inserting Graphics into HTML) in the Icon File column to embed an Icon in the specified Tab Container tab. For example, entering the following string will embed the icon for a Multi-Group Time-Series Chart in the tab:
$npImageUrl/PieChart.png
Interaction Produced
The Tab Container Component is a producer of clear Interactions. Whenever a new tab is selected the clear Interaction is sent out. When this Component is created, it does not send out any Interactions.
Interactions Consumed
The Tab Container Component consumes group Interactions that the $group variable can be substituted.
Toolbar
The Toolbar Component provides Controls for the entire Insight or for any subset of Components. An Insight may have multiple Toolbars.
Edit Toolbar
Adding a Toolbar to the Insight Development Kit grid automatically calls up the Toolbar edit dialog.
1)
Click Add to call up the Add Controls dialog. Right-click to select the Controls you want in the Toolbar.
Figure 84
Add Controls Dialog
 
2)
Click OK to return to the Toolbar edit dialog, which is now populated by the Controls you chose.
Figure 85
Edit Toolbar
 
From this dialog you may perform the following configuration tasks.
Add - click to bring up the Add Controls dialog.
Move Up - select a Control and click this to move the Control up in the dialog, which moves it towards the left end of the Toolbar.
Move Down - select a Control and click this to move the Control down in the dialog, which moves it towards the right end of the Toolbar
Delete - select a Control and click this to remove the Control from the Toolbar.
Configure - if a configurable Control is selected, this button may be clicked to bring up an edit dialog for the Control.
Change Label - Select a Control and click this to change the default label for the Control.

Audio Clip Control
The Audio Clip Control lets you play .WAV files in an Insight and, if desired, link them to help pages controlled by the Help Pages Control, as described in Help Pages Control. The Audio Clip Control can be added from the Toolbar component. Audio files must be referenced to the local machine the appliance. To upload the .WAV files to the appliance, see Upload Images, HTML and Audio Files.
The Audio Clip Control comprises a drop-down pick list for selecting the clip desired, plus stop, pause, and play buttons. The list is order by how the files are arranged in the file folder where the .WAV files “live.” No matter where you start on the drop-down list, the audio files play in a continuous loop until you click stop.
Figure 86
Audio Clip Control
 
The Audio Clip Control configuration dialog allows you to add and name clips, set the index for coordinating with Label highlighting, and synchronize the clips with help pages. To configure it, select Audio Clips in the Toolbar Component edit dialog and click Configure.
Figure 87
Audio Clip Control Configuration Dialog
 
This dialog box has the following components:
Clips - this drop-down menu shows the clips already added. Click Add to add a new clip, which is appended to the end of the list. You may not change the order of clips once they are added. Click Remove to delete existing clips.
Clip - enter the directory path and filename of the audio clip, prefaced by either the ${local} or ${remote} URL variable (see Inserting Graphics into HTML). When you enter $, prompt appears to choose from, much like Figure 91.
Clip Name - enter a name for the clip that will be displayed in the drop-down pick list of the Audio Clip Control.
Highlighting - check this and enter a highlight index if you would like to use this with a Highlight Analysis (or directly) to control highlighting in one or more Labels within the Insight. You must also check this box in order for the audio file/help page sychronization to work.
Sync With Help - check this to synchronize the audio clips with help pages controlled by the Help Pages Control. If this box is checked, when the user selects an audio clip, the help page with the same position in the Help Pages list of the Help Page Control configuration dialog as that of the selected audio clip in the Clips list of the Audio Clips Control configuration dialog will be launched. For instance, the number-three help page will be launched whenever the number-three audio clip is played.
For a simplistic example, see Using Audio Clips and Help Pages.

Interactions Produced
The Toolbar Component produces the following Interactions:
 
Table 69
Interactions Produced by Toolbar Component 
interaction
How Produced
box
User selects an Appliance in the Appliance selector.
clear
The user clicks on the Clear button.
group
User selects a group path in the Add Traffic Group Control or a group category in the Group Categories selector.
index
The user changes the audio clip in the Audio Clip selector or the help page in the Help Control.
insight
The user selects an Insight in the Insight selector.
interval
The user changes the interval in the Interval selector.
limit
The user changes the limit in the Limit Control and hits Enter.
metric
The user selects a metric in the Metrics selector or a set of metrics in the Multi-Metric selector.
singleQuery
The toolbar is initialized (at startup).
text
The user enters text in the Text Entry control.
topMetric
The user selects a metric in the Metrics selector or a set of metrics in the Multi-Metric selector.
unit
The user changes the unit in the Unit Control.
The group, metric, topMetric, interval, box, limit, singleQuery, text, and insight Interactions are sent out at startup to notify any Components that are listening to the Toolbar's Controls of their initial settings. The singleQuery Interaction is used to make sure no queries are sent until all the Interactions have been sent during initialization.
If the group category Control is visible an initial group Interaction is sent out with the selected group category. If for some reason there is no group category, no Interactions are dispatched.
If the metric selector or multi-metric selector is visible in the toolbar the current set of metrics or an empty set of metrics is sent out as a metric Interaction and a topMetric Interaction depending on whether the selectors have a any metrics selected.
If the interval Control is visible, either the selected interval, or an empty interval Interaction is sent when the toolbar is initialized.
If the limit Control is visible a limit Interaction is sent out with the initial limit. If the Appliance selector is visible a box Interaction is sent with the initial appliance selection, or an empty Interaction if there is no selection.
Finally, if the Insight Selector is visible and insight Interaction is sent out with the initially selected Insight.
