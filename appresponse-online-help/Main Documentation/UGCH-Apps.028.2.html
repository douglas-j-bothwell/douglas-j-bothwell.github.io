Defined Applications: Overview
The Defined Application Manager (Desktop Console > Tools > Applications > Defined Application Manager) enables you to define the applications you want to monitor at all times, regardless of throughput levels and topping limits. You can define three different types of application, based on attributes in the Defined Application Manager:
Standard Application
Server Application
Web Application
Module Application
Standard Application
A Standard Application is defined based on either of the following attributes:
Protocol and port number(s)
You can define applications based on most IP protocols: TCP, UDP, ICMP, OSPF-IGP, ESP, and so on. Most (though not all) pre-defined applications are based on TCP or UDP.
Recognized Application
The appliance includes a set of pre-defined Recognized Applications and monitors these applications automatically, but imposes topping limits. If you want to monitor a Recognized Application regardless of topping limits, create a new Defined Application based on the Recognized Application.
Server Application
A Server Application is defined based on the following attributes:
Either of the following:
–
Protocol and port number(s)
–
Recognized Application
IP Addresses
Server Application definitions are useful when you want to monitor applications that assign ephemeral ports to connections across a range of port numbers. You can define a Standard Application with a range of port numbers, but this can cause inaccurate results if other monitored applications use the same ports. To avoid this ambiguity, you can define a Server Application that includes the server IPs as well as the port numbers that are specific to that application.
Web Application
A Web Application is defined based on the following attributes:
Protocol and port number(s)
IP Addresses
URLs
A Web Application defines an HTTP application that communicates over a specific set of TCP ports, using a specific set of server IPs and a specific set of URLs. Web Applications enable the appliance to distinguish between applications that use the same ports and servers but different URLs. For example, a web front end might use the same set of servers and ports to deliver a range of applications—ERP, CRM, Manufacturing—where the URL path indicates the specific application being delivered.
Note—Web Transaction Analysis significantly extends the ability of AppResponse to monitor web applications. You can use Web Transaction Analysis to
Track and analyze web page usage
Auto-discover downloads of web pages and related groups of web pages
Collect detailed metrics on web page usage
Auto-decode and analyze HTTPS web applications
Auto-detect users and identify web page downloads by detected user
Module Application
A Module Application is defined based on the following attribute:
UC Monitoring, which can be either voice or video
