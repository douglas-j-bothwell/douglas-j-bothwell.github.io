Table Components
The following AppResponse Table types are available for building an Insight:
Director Group Table
External Data Table
Favorites Table
Group Table
Single Group Table
Editing Table Components
All Table Components are edited using the Table Controls Dialog or by directly setting controls, except for settings that pertain specifically to their use as Insight Components, as discussed below.
The Table Controls Dialog for the Single Group Table is discussed in the section for that Component, as the Single Group Table is available only within an Insight.
Composition
The Composition Tab of the Table Controls Dialog in Table Components gives you control over what features of a AppResponse Table are available to the user of an Insight: only those features checked will be operational in the Insight for that particular Table. Each type of Table Component has its own set of features that may be checked. Each Table in an Insight may have different settings.

Table Insight Links
The Insight Links tab of the Table Controls dialog in Table Components gives you the ability to create a Table that a user can launch other Insights from using the table context menu (i.e. by right-clicking in a cell, row, or column).
Figure 58
Table Controls Dialog > Insight Links Tab
 
This dialog box has the following components:
Label - enter the name of the Insight as you want it to appear in the Launch drop-down menu in the Table’s legend context menu.
Local - check this box if the Insight to be launched is in the local Insight directory.
Insight Path - if Local is not checked, you may select the Insight to be launched from a drop-down list with the Insights currently saved on the appliance. If Local is checked, you must enter the full filename and directory path of the Insight file.
Add - click Add to add a new line for entering an Insight link.
Remove - click Remove to delete an Insight link.
Default Insight - if you have entered more than one Insight link, you may select the one that will be the default choice from a drop-down menu. The default Insight can be invoked by simply double-clicking on a Table cell. This does not work when multiple cells or rows are selected.

The following Table lists the Interactions and values passed to the linked Insight for any Table that produces the listed Interaction.

Interaction
Value(s) Passed to Linked Insight

If a single row or cell is selected, the corresponding group is passed.
If multiple rows or cells in a single column are selected, the corresponding groups are passed.

If a single row is selected, the first metric displayed (i.e., the one in the left-most metric column) is passed.
If a single cell is selected, that metric is passed.
If multiple rows are selected, for each group the first metric displayed (i.e., the one in the left-most metric column) is passed.

Director Group Table
This Component displays a Top Director Groups Table.
Interactions Produced
The Group Table Component produces the following Interactions
:
Table 30
Interactions Produced by Group Table Component 
Interaction
How Produced
group
User selects one or more groups in the table.
limit
User enters the top-by limit in the chart’s Top field. A new query may be issued.
metric
User selects a row in the table. The table sends out the metric selected by the user, or the first non-info metric when there is no column selection. Single Group Table
This is a Component that displays data for a single group in a tabular format. The metrics are arranged in rows and the metric values appear in a column. The metrics that initially appear in the table can be configured via the Table toolbar. The order of the metrics can be altered by dragging and dropping table rows. Metric names and units can be changed through the right click menu.
Figure 59
Single Group Table
 
The Single Group Table has a column that does not appear in other Tables: Severity. This is used in conjunction with the Analysis Aggregation Component and Threshold Analysis Components to create a kind of “alert” against the average value of the corresponding metric over the time period of the Insight’s project.
The Severity column displays four condition levels:
Normal (green)
Minor (yellow)
Major (orange)
Critical (red)
The thresholds for the Severity column may be set during Insight creation by editing the Threshold Analysis Component. The Insight user may change the thresholds by right-clicking on a row and selecting Edit Thresholds.
Edit Single Group Table
The Single Group Table edit dialog offers three tabs.
Figure 60
Single Group Table Controls Dialog
 
This dialog box has the following components:
Toolbar - Determines whether the Toolbar is displayed, and which Controls on it are displayed if it is: Group Label, Group Chooser, and Metric Chooser. For instance, in Figure 59, above, the Group Label and Metric Chooser are displayed.
Composition - controls the appearance of the Severity column.
–
Show Severity Text - adds the text description of the severity level (e.g., normal, minor, etc.) to the display.
–
Use Square Icons - makes the severity indicators square instead of round.
Insight Links - used exactly as with other Table Components, as described in Table Insight Links. Interactions Produced
Add the Severity Column to the Single Group Table
In Version 5.1.2 the Single Group Table edit dialog does not control the appearance of the Severity column. Instead, to add or remove the Severity column from a Single Group table, right click on the table to bring up the context menu and select Show/Hide Columns…, which will bring up the following dialog, where you may select or de-select any of the columns.
