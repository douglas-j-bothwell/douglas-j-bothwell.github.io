Working with the Insight Development Kit Window
This section briefly describes how to accomplish basic tasks when building an Insight. It covers the two main areas of the Insight Development Kit window: the Outline Area and the Layout Grid.
There are two distinct modes of working in the Layout Grid:
Grid Edit mode
Component Edit mode
The Insight Development Kit functions slightly different in each of these modes. See Selector Control (Edit Mode) for more information.
Understanding the Outline Area
The Outline Area of the Insight Development Kit window is used to work with Insight Elements (Layers, Components, and Interactions) in a non-graphical manner. It comprises three panes: the Element pane, the Description pane, and the Layout pane.
Working with the Element Pane
The Element pane displays the Layers, Components, and Interactions of the Insight in a hierarchical tree structure.
Figure 9
Element pane (with Layout Grid shown)
 
The figure above shows the Element pane displaying the Components of a simple Insight, comprising a Business Group Selector and a Multi-Group Time Series Chart. There are two Interactions: a Group Interaction from the Selector to the Chart and a Clear Interaction to clear the Insight. There is also a Drilldown Analysis. This Insight has one Layer.
Selecting a Visual Component in the Element pane outlines the Component, in blue in the Layout Grid, in both the Actual View and the Schematic View. If the selected Component is in a different Layer than the one currently displayed in the Layout Grid, the Layout Grid displays the Layer containing the selected Component.
Selecting any Element in the Element pane, and then right-clicking on it brings up the Element pane context menu, which enables a variety of operations on the selected Element. The operations shown in this menu are determined by the type of Element selected. .
Figure 10
Element Pane Context Menu
 
This context menu shares some selections with the Layout Grid context menu. The menu selections are discussed in the following sections as appropriate.
Working with the Description Pane
The Description pane is not interactive: it simply displays information about the Element selected in the Element pane.
Figure 11
Description Pane
 
However, you may set the Description pane to also display information about any Insight Development Kit toolbar icon you click on. To do so, choose Tools > Builder Settings to display the Insight Development Kit Settings dialog, select the Miscellaneous tab, and check Show Instructions.
You can remove the Description pane from the Outline Area by selecting Tools > Builder Settings to display the Insight Development Kit Settings dialog, select the Outline tab, and uncheck Description. Click OK.
Working with the Layout Pane
The Layout pane lets you set some of the size parameters of a Visual Component.
Figure 12
Layout Pane
 
The Java layout engine uses a complex algorithm to determine Component size, which you can control using the parameters in the Layout pane. These parameters are effective both in the Layout Grid (Actual View) while you are editing the Insight, and in the actual Insight when it starts up. However, interactions between these parameters, not to mention the additional parameters available in the Advanced mode of the Layout Options dialog (Edit Menu), may result in unexpected behavior. Experimentation is encouraged using the Preview Insight Content window. See Testing the Insight.
You must select the Component while in Selector control/Grid edit mode to set the parameter. See Selector Control (Edit Mode).
Minimum Size (x,y)
- enter the size in pixels here to specify the minimum size of the Component.
Preferred Size (x,y)
- enter the size in pixels here to specify the preferred size of the Component. These parameters are automatically entered when you resize a Component in the Layout Grid.
Maximum Size (x,y)
- enter the size in pixels here to specify the maximum size of the Component.
Resizable
- check either or both boxes to make the Component resizable in that dimension.
Expandable
- check either or both boxes to enable the Java layout engine to expand the Component as needed to satisfy the layout algorithm.
You can remove the Layout pane from the Outline Area by selecting Tools > Builder Settings to display the Insight Development Kit Settings dialog, select the Outline tab, and uncheck Layout. Click OK.
Understanding the Layout Grid
The Layout Grid comprises a set of panes into which Visual Components are placed when building an Insight.
Figure 13
Layout Grid
 
The lines between the panes are called dividers. With the Component Edit mode off (see Selector Control (Edit Mode), you can drag the bottom divider for a row to resize that row (and therefore all Visual Components in that row). Dragging the intersection of two dividers allows you to adjust both column width and row height at the same time.
The pane arrows at the right and bottom are used to add rows or columns of active panes, as described in Adding and Removing Active Panes.
A Layout Grid pane is “inactive” when it is blank. In this state, you cannot place a Visual Component in it. A pane is “active,” that is, ready to receive a Visual Component, when it displays either the text Add Visual Component or the text Blank. There are several different ways to add active panes to the Layout Grid. See Adding and Removing Active Panes and Working with Visual Components.
Layout Grid Operation
The operation of the Layout Grid is controlled by the Layout Grid sub-menu of the View Menu, which is divided into two sets of radio buttons.
Figure 14
View menu > Layout Grid
 
The top set of radio buttons controls the View of the Layout Grid.
Workflow View
The Workflow view displays the Insight in a workflow diagram.
Figure 15
Layout Grid - Workflow View
 
The Workflow view is best for viewing the Interactions between the Components and how information moves through the Insight. Many choose to use the workflow view to sketch the Insight, all though this is a personal preference. For a small exercise on building an Insight in Workflow view, see Using the Workflow View.
The Workflow view has a toolbar, which is an integral part of working in this view.
Figure 16
Layout Grid - Workflow View Toolbar
 
The toolbar buttons are, from left to right:
Move Selection Mode - use this button to highlight a node on the chart and move it.
Add Interaction Mode - to add Interactions while designing, select a node and drag the cursor to another node.
Pan Mode - to move the entire workflow diagram, select this button and move the diagram.
Interaction drop-down list - select an Interaction type from the drop-down list to display only this type of Interaction in the diagram.
Layout drop-down list - select a layout for the Insight Components and Interactions: hierarchical, circular, tree, organic, etc.
Note—The Layout drop-down list is not on by default. Click the Workflow control button and select Layout.
Options button - select to show the Interactions consumer and producer options on the diagram.
Set the display mode to show symbols - use this control to toggle between icons and shapes.
Select to show any Interactions produced by external sources and consumed by one or more Components (a common external source is the project time that implicitly sends internal Interactions to a various Components like charts and tables).
Zoom to - click and drag the cursor to select an area you want to zoom in on.
Zoom full - click to view entire workflow.
Zoom in 2x - use this to zoom in two times the shown magnification.
Zoom out 2x - use this to zoom out two times the shown magnification.
Layout Grid - brings up a separate window that allows you to drag and drop Visual Components on to the Layout Grid. You are allowed to span multiple “panes” with one Visual Component for proper layout.
Refresh Layout - cleans up the Layout area (much like how you “clean up” your desktop).
Workflow Controls - use this dialog box to customize the Workflow View toolbar.
Right-click on a node to display a context menu to perform operations such as add Interactions to all consumers (e.g., the Clear Interaction).
Figure 17
Interaction Mode Context Menu
 
Actual View
The Actual view displays the Insight as it appears to the user.
Figure 18
Layout Grid - Actual View
 
This is the view you work in to position, resize, edit and move Visual Components as described in Working with Visual Components. Its operation (mode) is controlled by the Layout Grid Toolbar, as discussed in the next section.
Schematic View
The Schematic view displays the Insight in a grid of panes that show the edit dialogs for the Visual Components in the Insight.
Figure 19
Layout Grid - Schematic View
 
You cannot resize, move, or position Components in the Schematic view; this view is intended for convenience when editing a number of similar Visual Components all at once, by avoiding the necessity of repeatedly using the Selector control in the Layout Grid Toolbar, as discussed below. For instance, you might use the Schematic view to enter text in all Label Components at once.
The Schematic view is useful for identifying active rows or columns that have no Visual Components. These “empty” rows or columns can lead to layout issues in the Actual view.
Size Change Mode
The bottom set of radio buttons in the Layout Grid sub-menu of the View Menu selects the Size Change mode of the Component Edit mode (see Selector Control (Edit Mode)) of the Layout Grid Actual View.
This selection does not govern size changes made by selecting a Visual Component using the Layout Grid context menu or the Element Pane context menu and entering the size parameters in the Layout Pane, or by spanning a Component (see Span Visual Components).
Size Changes Affect One Row
- select this to make size changes affect only the row in which the change is made. The adjacent row maintains its original size and moves down or up as the row operated on increases or decreases in height.
Size Changes Affect All Rows
- select this to make size changes affect all rows. The effect is greatest on the adjacent row, which will increase or decrease in height as the row in which the change is made decreases or increases in height.
Changing the height of a row in which there is a vertically-spanned Component (see Span Visual Components) always affects all rows, regardless of the size change mode selection.
Although this explanation may seem a bit opaque, a moment’s experimentation makes the effect of the Size Change mode quite apparent.
The Layout Grid Toolbar
The Layout Grid Toolbar controls the operation of the Layout Grid in Actual View. It has no effect in the Schematic view, Workflow view or during Test/Layout (see Testing the Insight). It comprises three controls: the Selector control, the Refresh control, and the Size Memory control.
Figure 20
The Layout Grid Toolbar
 
 
Selector Control (Edit Mode)
The Selector control toggles between the Grid Edit mode and Component Edit mode. It is on when highlighted (Grid Edit mode); the default is on. Click on the highlighted Selector button to go to Component Edit mode.
Selector on (Grid Edit Mode): in this mode, mouse actions are interpreted by the Layout Grid, not the Visual Component.
Click on a Visual Component to select it. A blue line will appear around the selected Component. The Component is highlighted in the Element pane, its description appears in the Description Pane, and its measurements appear in the Layout Pane, (if no measurements were previously entered, only the preferred size appears, which is set to whatever size the Component is in the Layout Grid).
Figure 21
Selected Visual Component in Layout Grid (Actual view)
 
Double-click on the Component to display its edit dialog. See Edit Visual Components.
Right-click on a Component or an Active pane to display the Layout Grid context menu, which allows you to add or remove Active panes and Visual Components, and to control the placement and positioning of Components.
Figure 22
Layout Grid Context Menu
 
You cannot resize a Component by dragging the divider in this mode. To resize a Component conveniently, turn off the Selector, resize the Component, and then turn the Selector back on.
Move a Component by click-dragging it.
Selector off (Component Edit Mode): in this mode, mouse actions are interpreted by the Component.
Right-click on an Active pane to display the Active Pane Context Menu, which enables you to “span” grid positions with a single Visual Component or to leave a grid position blank.
Click on any control in a Visual Component to use that control; e.g. the Top Metric Control (By) in a Group Table Component.
Right-click on a Component to display its context menu; e.g., right-click on a time-series Chart Component to display the Set Y-Axis Marker menu or on its legend to display the Chart Legend context menu.
Double-click on a Component to evoke the behavior associated with a double-click; e.g., the Chart Settings Dialog for a Chart Component.
Refresh Control
The Refresh control applies any settings that have been entered using controls or panes outside the Layout Grid. For instance, if you want the Toolbar Component to span two active panes, and use the Fill to Right command (in the right-click menu), you do not see the changes until you click refresh.
Note: You may not see changes associated with your actions until you click Refresh.
Size Memory Control
The Size Memory control “memorizes” the size settings selected for Components using either click-drag in the Layout Grid, the Layout Pane (see Working with the Layout Pane), or the Tools > Set Component Size menu. In effect, it saves the sizes of Components without saving any other parameters.
Adding and Removing Active Panes
There are many different ways to add Active panes (panes displaying the text Add Visual Component) to the Layout Grid so that you may add Visual Components, or to remove them.
The grid’s initial state is one Active pane. Until at least one Visual Component has been added, you may only use the pane arrows to add Active panes.
The following table lists the methods of adding and removing Active panes.

Click on any Visual Component in the Element pane to select it. Right-click on the Component to display the Element Pane context menu (page 5), and select Remove Row Above or Remove Column to the Left to remove Active panes.

Select a Visual Component in the Layout Grid, right-click on it and select Remove from the Active Pane context menu.
You may choose to leave an active pane blank in the finished Insight. To do so, in either the Actual or Schematic view, right-click on the active pane to display the Active Pane Context Menu and select Leave This Grid Location Empty. The Active pane now displays the word Blank, and displays as empty in the finished Insight. You may place a Component into a blank Active pane, but you may not change a blank Active pane back to an empty Active pane.
