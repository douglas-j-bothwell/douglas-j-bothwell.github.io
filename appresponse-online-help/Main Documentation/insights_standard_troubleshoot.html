Troubleshooting Insights
Troubleshooting Insights are designed to help you quickly isolate response time issues: where’s the problem, and what’s causing it? Using them you can drill down to the server, network, application, provider, or hop that’s at fault.
These insights include a “Drill into:” pull-down menu that enables you to
Drill into Apps (Selected Business Group > Top Applications >
Top IPs > Top Connected IPs)
This view is more useful for viewing and analyzing top applications within a particular Business Group.
Drill into IPs (Selected Business Group > Top IPs >
Top Connected IPs > Top Applications)
This view is more useful for viewing and analyzing top individual IP addresses within a selected Business Group.
Troubleshooting Insights include:
Bandwidth Hog
High Network Latency
High Packet Loss
Slow Servers
It is important to note that the Troubleshooting Insights essentially all work the same with different metrics (including Worm Hunt in Security Insights).
Bandwidth Hog
This Insight is useful for identifying the Business Groups, applications, hosts, and conversations that are consuming the most bandwidth. You can use this Insight to answer questions such as:
Which applications consume the most network resources in a specific location or Business Group?
Who is using the bandwidth for those applications? Where in the network are the top-talking clients and servers for applications that consume a lot of bandwidth?

Procedure 1
Using the Bandwidth Hog Insight
1.
In the Insight Index window > Troubleshoot tabbed page, double-click on the “Bandwidth Hogs” link.
The Bandwidth Hogs Insight appears. The Top Business Groups tabbed page (top left) lists the Business Groups with the highest bidirectional throughput.
2.
Select one of the Business Groups in this tabbed page.
The Time Series graph (top right) shows the total bandwidth consumption for the selected Business Group.
The Top 10 Apps table (bottom left) shows the applications with the highest throughput within the selected Business Group.
3.
(Optional) If the Time Series Chart shows a significant “spike” in throughput, drag horizontally (left to right or right to left) in the chart to highlight this spike. This selects the time window of interest.
When you select a time window in this manner, all three bottom tables show data for this selected period only. If no time window is selected in the Time Series Chart, the three panes show data for the entire project time window.
Figure 2
Horizontal Dragging to Select a Time Period of Interest (EXAMPLE)
 
Key Concept: Select “Spikes” in a Time Series Chart To Pinpoint Applications and Hosts of Interest
When you are using the Troubleshooting Insights, it is often useful to drag horizontally to select significant “spikes” in a Time Series Chart. This can help you find applications, hosts, and conversations that might be contributing to—or are significantly affected by—conditions such as excessive bandwidth consumption, network congestion, high-latency connections, and long application response times.
4.
Select a row in the Top 10 Apps table.
The Time Series graph shows throughput for the selected application (as well as for the entire Business Group).
The Top 10 IPs table shows the IPs with the highest throughput for the selected application (bottom left) within the selected business group (top left).
5.
Select a row in the Top 10 IPs table.
The Time Series graph shows throughput for the selected IP.
The Top 10 Connected IPs table shows the IPs with the highest throughput with the selected IP (Top IPs).
6.
Select a row in the Top Connected 10 IPs table.
The Time Series graph shows throughput for the connected IP.
End of Procedure 1
 
 
High Network Latency
This Insight identifies the Business Groups, applications, hosts, and conversations with the longest round trip times, and is useful for answering questions such as
Which Business Groups have the longest network round trip times?
In the Business Groups with the longest round trip times, which applications are most affected by network delays?
Which servers and clients have the longest round trip times for specific applications?
How AppResponse Identifies Latency
AppResponse identifies latency as Network Round trip Time (RTT), and reports both inbound and outbound latency. (“Inbound” and “outbound” directions are determined using the Internal Address List specified in the Administration > System > Setup page) AppResponse assumes that outbound latency (internal IPs --> external IPs) will be higher than inbound latency. This Insight shows the Business Groups with the highest Outbound Round Trip Time.
Figure 3
High Network Latency Insight
 
Procedure 2
Using the High Network Latency Insight
1.
In the Insight Index window > Troubleshoot tabbed page, click on the “High Network Latency” link.
The High Network Latency Insight opens. The Top Business Groups tabbed page (top left) lists the Business Groups that have the highest Outbound Goodput during the selected time period.
The Goodput metric measures the transfer rate of TCP packet payload data for all connections over the selected time interval, but excludes retransmissions and dataless TCP packets. A high-goodput Business Group should have a large sample of RTT (round trip time) measurements. Applications, hosts, or conversations with high round-trip times, taken from a large sample of RTT measurements, are possible indications of one or more high-latency connections.
2.
Select one of the Business Groups in this tabbed page.
The Time Series graph (top right) shows the outbound Round Trip Times for the selected Business Group.
The Top 10 Apps table shows the applications with the longest outbound Round Trip Times within the selected Business Group.
3.
(Optional) If the Time Series Chart shows a significant “spike” in throughput, drag horizontally in the chart to select the time window of this spike.
4.
Select a row in the Top 10 Apps table.
The Time Series graph shows round trip times for the selected application (as well as for the entire Business Group).
The Top 10 IPs table shows the IPs with the longest outbound Round Trip Times for the selected application (bottom left) within the selected business group (top left).
5.
Select a row in the Top 10 IPs table.
The Time Series graph shows the Round Trip Times for the selected IP.
The Top 10 Connected IPs table shows the IPs with the longest outbound Round Trip Times for the selected IP (Top IPs table).
6.
Select a row in the Top 10 Connected IPs table.
The Time Series graph shows Round Trip Times for the selected conversation.
End of Procedure 2

Slow Servers
This Insight is useful for identifying applications and hosts that have long response times.
How AppResponse Identifies Slow Servers
AppResponse identifies “clients” and “servers” based on the host that initiates a TCP Connection Setup request: the host that sends the setup request is the client, and the host that receives it is the server. Using this definition, AppResponse identifies those “servers” that have a long Server Response Time. This metric is defined as the time between the server acknowledgement (server sends acknowledgement to TCP Connection Setup request) and the server reply (server sends first packet of requested data).

Procedure 4
Using the Slow Servers Insight
1.
In the Insight Index window, click the “Slow Servers” link.
The Slow Servers Insight opens and lists the Business Groups that have the most server-side application turns during the selected time window. The “chattier” an application is on the server side, the greater effect a slow server can have on that application’s performance.
2.
Select one of the Business Groups in this tabbed page.
The Time Series graph (top right) shows the server-side response time for the selected Business Group.
The Top 10 Apps table shows the applications with the highest server-side response times within the selected Business Group.
3.
Optionally, if the Time Series Chart shows a significant “spike” in throughput, drag horizontally in the chart to select the time window of this spike.
4.
Select a row in the Top 10 Apps table.
The Time Series graph shows server-side response times for the selected application (as well as for the entire Business Group).
The Top 10 IPs table shows the IPs with the highest server-side response times for the selected application (bottom left) within the selected business group (top left).
5.
Select a row in the Top 10 IPs table.
The Time Series graph shows the server-side response times for the selected IP.
The Top 10 Connected IPs table shows the conversations with the highest server-side response times for the selected IP (Top IPs table).
6.
Select a row in the Top 10 Connected IPs table.
The Time Series graph shows the server-side response times for the selected conversation.
End of Procedure 4
