Transaction Analyzer / Wireshark
Integration
SteelCentral Transaction Analyzer enables you to analyze captured packets, traffic flows, and user-level transactions from AppResponse appliances. Transaction Analyzer greatly extends AppResponse functionality: when you find a problematic traffic window in AppResponse, you can immediately analyze and troubleshoot that traffic using AppDoctor, QuickPredict, the Data Exchange Chart, and all the other features in Transaction Analyzer.
If you do not have a solution that includes Transaction Analyzer functionality, you can send packets to Wireshark instead.

SteelCentral AppResponse and Transaction Analyzer: Important Notes
Note the following:
SteelCentral AppResponse supports SSL Level-1 (anonymous) encryption only between the SteelCentral AppResponse appliance and the Transaction Analyzer host. Level-2 (certificate) encryption is not supported.
This means that, when the Transaction Analyzer capture service is enabled on an SteelCentral AppResponse appliance, any Transaction Analyzer user with connectivity to that appliance can perform captures.
You can set up an SteelCentral AppResponse appliance to capture in either single-arm mode (one interface) or dual-arm mode (both interfaces). If you reboot the appliance and change the capture mode while a continuous capture is running, you must stop/restart the continuous capture in the Application Capture Manager in Transaction Analyzer. Otherwise this capture will not run correctly.
The SteelCentral AppResponse appliance and the Transaction Analyzer capture agent each has its own separate traffic filter. If these two filters are configured differently, you might see different traffic patterns during the same time window.
Best Practices for Selecting and Analyzing Captured Packets
Riverbed recommends the following best practices:
1)
Filter as much irrelevant traffic as possible before you send packets to the Transaction Analyzer window in Transaction Analyzer.
When you view packets in the Transaction Analyzer, Transaction Analyzer assumes that all traffic in that window defines one and only one user-level transaction (such as a complete web-based query or FTP request). If the selected console object includes additional traffic beyond the transaction of interest, you should filter out this traffic in the Packet Download Manager or Trace Explorer.
2)
If you see packets that you want to analyze, send them to Transaction Analyzer or Wireshark immediately.
The continuous capture process stores traffic data in a rolling buffer; when the buffer is full, the capture process overwrites old data. This means that, if you wait, the traffic you want to view might get overwritten.
Set-Up Requirements
Before you can send traffic from AppResponse to Transaction Analyzer, you must do the following:
1)
If you have not already done so, install Transaction Analyzer on the same host where you plan to run the Desktop Console.
2)
Go to the Administration > System > Capture page and enable the packet capture process on the appliance.
NOTE—If your appliance supports high speed captures, you can specify additional options on this page. For more information, see Starting, Stopping, and Configuring High-Speed Captures.

Previewing/Downloading Packets for IPs, IP Conversations, and Business Groups
Note: If you see traffic that you want to analyze in Transaction Analyzer, send the packets to Transaction Analyzer immediately. If you wait, the packets might get overwritten.
The general workflow is as follows:
1)
Perform the steps described in Set-Up Requirements if you have not already done so.
If your appliance supports high-speed captures, this page has additional options which are described in Starting, Stopping, and Configuring High-Speed Captures.
2)
In the Desktop Console, set the project time to focus on the time window of interest.
Before you send packets to Transaction Analyzer, it is good practice to filter out as much extraneous traffic as possible. For this reason, you should always set the project time to the general time window when the packets were captured.
3)
Select one of the following, based on the packets you want to analyze in Transaction Analyzer:
–
IP (in an insight, table, or Topology window)
–
IP Conversation (in an insight or table)
–
Business Group (in an insight or table)
–
Alert (in the View > Dashboard window or the View > Log > Alerts window)
Your goal is to select the object that includes all the packets you want to analyze in detail while excluding as much irrelevant traffic as possible.
4)
Right-click on the selected item and choose one of the following:
–
Analyze in > Transaction Analyzer
Preview, download, and analyze the packets using Transaction Analyzer. This option is available only if Transaction Analyzer is installed locally.
–
Analyze in > Wireshark:
Download the captured packets for analysis in Wireshark. This option is available only if Wireshark is installed, and Transaction Analyzer is not installed, on the local host.
–
Analyze in > Packet Download Manager:
Preview and filter the selected packets in AppResponse, then download and open in Wireshark or a specific Transaction Analyzer window. This option is available only on appliances that support High Speed Captures.

Previewing/Downloading Packets using the Packet Download Manager
The Packet Download Manager (Tools > Packet Download Manager) is available on appliances that support High-Speed Captures. You can use this manager to
Preview all captured packets in the rolling buffer
Select packets based on time intervals and IP conversation pairs
Send packets to a specific Transaction Analyzer window (or to Wireshark) for in-depth analysis
Download and save the selected packets to a capture file for further analysis
Preferences for Launching Transaction Analyzer
AppResponse includes a set of preferences that enable you to configure how AppResponse invokes Transaction Analyzer (for example, when you right-click on an IP conversation and choose Send To > Transaction Analyzer). The Transaction Analyzer tabbed page in the Tools > Preferences dialog box enables you to specify three different modes for invoking Transaction Analyzer automatically:
Use Registry (run Transaction Analyzer in "default" mode)
Run Transaction Analyzer in “default” mode by running the Transaction Analyzer executable associated with Transaction Analyzer capture files in the OS registry on the local host.
Select Application
Invoke a specific Transaction Analyzer solution and executable. This option is useful when you have multiple Riverbed solutions and/or releases installed and you want to invoke a specific solution/release which might not be the default as defined in the OS registry.
Custom
Invoke the Transaction Analyzer solution using a text command as entered at a command prompt. This is an advanced option that is useful for users who want to invoke a specific Transaction Analyzer executable with a custom set of command-line arguments.
Wireshark Integration
You can download packets directly to Wireshark from the Packet Download Manager and the “Individual Page Views” insight.
The Wireshark tabbed page in the Tools > Preferences dialog box enables you to specify two different modes for invoking Wireshark automatically:
Use Registry (run Wireshark in "default" mode)
Run Wireshark in “default” mode by running the Wireshark executable associated with capture files in the OS registry on the local host.
Select Application
Invoke a specific Wireshark solution and executable. This option is useful when you have Wireshark releases installed and you want to invoke a specific release which might not be the default as defined in the OS registry.
Note: This functionality is available only on appliances that support High-Speed Captures, and requires Wireshark release 1.2.1 or higher.
