Setup and Configuration for Element Polling Insights
To fully use the Element Polling Insights, you must have an instance of NetSensor running on a host that can communicate with the SteelCentral AppResponse appliance. In addition, SteelCentral AppResponse must be configured to obtain data from NetSensor.
Procedure 1
Using NetSensor as a Data Source
1.
If not already done, install an instance of NetSensor on your network, as described in the NetSensor Installation Guide.
2.
Verify that NetSensor is set to run in the correct polling mode, as determined by which Insights you intend to use:
You must use AppPath Analysis mode for the following Insight:
• 
AppNetwork Path Analysis Insight
You can use either Simple or AppPath Analysis mode for the following Insights:
• 
Cisco IP SLA Test Query Insight
• 
SNMP Query Insight
• 
SNMP System Monitor Insight
For help with this step, contact an APM EI administrator or see the “polling mode” information in the NetSensor User Guide.
3.
Verify that NetSensor is configured to collect the desired data from your network. Only the devices and metrics specified in active APM EI polling profiles will be available to SteelCentral AppResponse.
Note: Server polling in APM EI is off by default. However, this information can be very useful in AppResponse and we recommend that you include servers in your polling profiles.
For help with this step, contact an APM EI administrator or see the “polling profiles” information in the NetSensor User Guide.
4.
Configure AppResponse to use data from NetSensor:
4.1.
Go to the Administration > System > Integration > Element Polling web page (Figure 1).
4.2.
Select the “Obtain data from” checkbox.
4.3.
Select the “NetSensor” radio button.
4.4.
Enter the host where NetSensor is installed in the “Hostname/IP” field.
4.5.
If desired, click Configure to verify that NetSensor is installed and running. If the APM EI interface login does not appear, contact an APM EI administrator.
4.6.
Click Apply.
5.
Launch the Desktop Console.
End of Procedure 1
 
 
Limited SNMP polling is available using the built-in Basic SNMP Collector. This data source can be used only with the SNMP System Monitor Insight.
Procedure 2
Using the Basic SNMP Collector as a Data Source
1.
Configure AppResponse to use data from the Basic SNMP Collector:
1.1.
Go to the Administration > System > Integration > Element Polling web page (Figure 1).
1.2.
Select the “Obtain data from” checkbox.
1.3.
Select the “Basic SNMP Collector (built-in)” radio button.
1.4.
Click Apply.
2.
Launch the Desktop Console.
End of Procedure 2
 
 
Selection of the data source is done on the Administration > System > Integration > Element Polling web page, as shown in Figure 1.

Communication over Secure HTTPS Connections
AppResponse can now communicate with NetSensor over secure HTTPS connections. This functionality requires the following:
You must configure NetSensor to use secure mode. For more information, see the AppSensor User Guide and search on the string "secure mode".
In AppResponse, you must configure the Administration > System > Integration > NetSensor page as follows:
–
The Web Console Port field must specify the HTTPS port specified in NetSensor.
Note that NetSensor 1.1 has different default ports for HTTP (default = 9190) and HTTPS (default = 8543).
–
The “Port for servicing AppResponse data requests” field must specify the NetSensor client connectivity port.
The default client connectivity port is 3100 in NetSensor 1.1. This port is the same regardless of whether NetSensor is using secure mode or not. For more information, see the NetSensor Installation Instructions.
–
The "Use secure communications" checkbox must be selected.
