Defined Application Manager
The Defined Application Manager (Desktop Console > Tools > Defined Application Manager) enables you to define the applications of interest in your network—that is, the applications for which you want to collect metrics at all times, regardless of topping limits. You can define applications based on ports, protocols, IPs, and URLs.

Adding URLs to Web Applications
When you enter a string in the URL field, and click the Add button. The host (before the first single ‘/’) and path appear in the table, as shown in the following figure.
Figure 2
Adding a URL to a Web Application
 
Note the following:
It is good practice to verify that the IP(s) in the Servers tab accurately correspond to the hostname in the URL. If the server IP for "www.hostname.com" (for example) is not included in the Server tab, the appliance will not identify the application.
You cannot define more than 100 URL paths (for all Web Applications combined) per appliance. If you have two different web applications that include the same URL path, this counts as two paths.
AppResponse matches URLs in a case-sensitive manner. The following URLs are considered unique:
–
www.hostname.com
–
www.Hostname.com
–
www.HostName.com
When you add a string to the table, the Defined Application Manager adds a wildcard '*' to the start of the path. Some proxy server deployments can result in GET messages that include the web server hostname—for example:
GET http://www.webapp.com/clientserv/login HTTP/1.1
Adding a wildcard to the start of the URL path (*/clientserv/*) ensures that the appliance matches these types of GET messages to the web application.
You can include '*' wildcards in the URL string. No other wildcard characters are supported.
UTF encoding is not supported, so “~string” and “%7string” are different strings. For such cases, use two URLs in the Web Application definition.
